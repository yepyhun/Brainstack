# Phase 16.1 Context

## Why This Phase Exists

Phase `16` made Layer 2 safer and clearer, but it did not solve the deeper strategic problem.

Brainstack now works as a real system, but it is no longer close enough to the original intended shape:

- `Hindsight` was supposed to make the agent feel noticeably smarter.
- `Graphiti` was supposed to provide a serious temporal knowledge graph.
- `MemPalace` was supposed to provide strong corpus storage, retrieval, and packing.

Instead, the current Brainstack stack contains too much middle glue relative to the donor strength that was actually recovered.

The new hard baseline proves this is not just a feeling:

- Brainstack-specific LongMemEval subset
- `3/15`
- `0.20`
- `392.4s`

That result is bad enough that the project cannot honestly continue with normal feature work.

## Strategic Diagnosis

The current system has real value in these areas:

- single memory owner
- installer-carried integration
- doctor / rebuild / runtime safety
- temporal correctness
- provenance handling
- host ownership boundaries

But those are not the original reason the donor stack was chosen.

The original ambition was:

- donor-first memory intelligence
- thin Brainstack glue
- lower complexity by letting the strong donor kernels solve their own strong problem

The project drifted away from that.

## Confirmed Donor Drift

### Hindsight / L1

Expected:

- rich retain from full context
- structured fact extraction
- proactive recall
- cross-session smartening

Current Brainstack reality:

- transcript shelf
- continuity summaries
- profile rows
- some fast preference carry-through

That is useful, but it is still materially below the original “learn, not just remember” reason for choosing Hindsight.

### Graphiti / L2

Expected:

- temporal context graph
- entities, relations, states, conflicts
- provenance
- point-in-time truth handling
- meaningful graph retrieval advantage

Current Brainstack reality:

- explicit graph stores exist
- temporal/current/prior/conflict logic exists
- inferred links now exist in bounded form

This is structurally respectable, but still below the intended graph power.

### MemPalace / L3

Expected:

- strong long-horizon corpus memory
- strong retrieval hierarchy
- strong bounded packing
- real large-corpus advantage

Current Brainstack reality:

- corpus shelf exists
- document sectioning exists
- bounded packing exists

But the donor-level corpus advantage is not yet clearly restored.

## Hard Constraints For This Recovery Window

- no new feature drift during recovery
- no benchmark-gaming
- no new regex or keyword expansion as substitute for donor strength
- no donor-strength gap may be closed by “temporary” custom glue and then left in place
- every retained Brainstack-owned module must justify itself as:
  - ownership
  - orchestration
  - safety
  - packaging
  - host integration

## Phase 16.1 Deliverables

This phase must produce:

- a donor-by-donor audit table
- a feature-freeze rule for the recovery window
- a concrete recovery scope that can drive `16.2`
- a clear split between:
  - justified Brainstack-owned value
  - diluted donor strength
  - accidental glue

## Success Criteria

Phase `16.1` succeeds only if the project ends up with:

- an honest donor-strength audit
- a hard written reason for why `16.2/17/18/19` are necessary
- a recovery scope strong enough to prevent another loop of local patching
