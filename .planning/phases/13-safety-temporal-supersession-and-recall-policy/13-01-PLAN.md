## Objective
Add Brainstack-owned temporal normalization, provenance merge, and bounded recall-safety policy so the existing Tier-2 write path can safely represent corrections, expose basis when needed, and stay token-disciplined.

## Deliverables
1. `temporal.py`
- adopt the small donor shape for:
  - temporal field normalization
  - point-in-time effectiveness checks
- fit it to Brainstack's existing graph/profile/continuity records instead of creating a second temporal subsystem

2. `provenance.py`
- normalize provenance records before durable write / recall formatting
- merge multiple evidence inputs without duplicate growth
- keep the representation deterministic and bounded

3. write-path hardening
- extend reconciler/store integration so new and updated state writes carry consistent temporal/provenance metadata
- preserve supersession history instead of destructive overwrite
- ensure conflict rows and current/prior rows remain machine-distinguishable

4. recall-safety policy
- prefer current truth by default
- surface prior truth only when it materially helps
- surface conflicts explicitly
- surface provenance only on important or low-confidence cases
- keep the recall block bounded so token discipline is not regressed

5. targeted verification
- deterministic unit/integration tests for:
  - supersession correctness
  - point-in-time queries
  - provenance merge
  - conflict recall behavior
  - bounded provenance rendering
- installed-runtime sanity proof that the real Bestie path uses the new recall safety rules

## Non-Negotiables
- no new memory layer
- no second runtime
- no donor-owned subsystem transplant
- no adaptive usefulness scoring in this phase
- no provenance spam on ordinary recall
- no decorative helper modules that the live provider path does not use

## Verification
- local tests prove temporal/provenance helpers independently
- integration tests prove the reconciler/retrieval path uses them in real writes and real recall formatting
- installed Bestie runtime proof confirms the new policy is present after rebuild/reinstall
- summary explicitly states what Phase `13` solved versus what remains for Phase `14` and `15`
