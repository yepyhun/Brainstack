# Phase 77 Plan

## purpose

Convert the multilingual/multimodal principle from aspiration into a recurring proof gate across capture, semantic indexing, graph recall, corpus recall, tool surface, and final packet rendering.

## core question

How can Brainstack prove it is not English-only or text-only while avoiding language-specific hacks and premature multimodal feature claims?

## GSD planning enforcement check

This phase must preserve:

- multilingual and multimodal by design
- no heuristic sprawl: no locale phrase lists as main intelligence
- truth-first: unsupported modality support must be named honestly
- quality priority: accuracy > token efficiency > speed
- no benchmaxing: evals must reflect ordinary real-world memory use

## shared north star and real-world proof gate

North star for Phase 70-77: Brainstack must become easier and safer for Hermes agents to use as a memory kernel without expanding Brainstack into runtime execution, workflow governance, or fake autonomy.

Every phase closeout must prove one of these:

- Hermes can use Brainstack more reliably in a real memory workflow
- or the phase is prerequisite work and proves no regression while naming the exact later phase that must deliver the user-visible improvement

Minimum proof shape:

- targeted tests for the changed surface
- at least one end-to-end Hermes/Brainstack interaction path or Brainstack packet/inspect proof using real provider/store code
- evidence ID/source/provenance proof where memory is involved, not only generated answer text
- no-regression check against Phase 66 golden gates once they exist
- bounded token/output sanity for any new tool, MCP, inspect, corpus, or trace surface

## explicit scope model

### Brainstack-owned scope

- multilingual golden fixtures
- typed non-text evidence contract
- cross-language capture/recall assertions
- packet rendering checks
- scorecard against product quality targets

### out of scope

- full image/audio model implementation
- language-specific phrase farm
- benchmark-only optimization
- claiming multimodal recall before contracts exist

## architecture decisions to freeze before execution

### 1. language set

Minimum proof languages:

- Hungarian
- English
- German
- one non-Latin-script language

### 2. modality contract

Non-text-derived records must be representable as typed evidence with source modality, extraction provenance, and final evidence projection.

### 3. scorecard rule

Competitive readiness must compare Brainstack to Mnemosyne, CerebroCortex, neural-memory, and Brainstack's own principles without letting benchmark scores override product truth.

Every scorecard claim must cite one of:

- a local Brainstack test/eval/inspect proof
- a local code path with file reference
- a cloned or primary-source competitor file used only for comparison
- a clearly marked unsupported/deferred gap

README-only competitor claims are allowed as leads, not proof.

## phase-specific real-world proof examples

Closeout must include at least:

- cross-language recall proof across Hungarian, English, German, and one non-Latin-script language without phrase farms
- cross-language explicit capture proof through typed contract/schema, not wording cues
- non-text typed evidence fixture proof with source modality and extraction provenance
- honest unsupported-modality proof where unsupported behavior is named and does not pretend to work
- scorecard proof that accuracy, token efficiency, and latency are measured in that priority order
- proof that competitor parity claims are backed by local evidence, not README marketing

## acceptance criteria

- Brainstack proves cross-language recall and capture without phrase farms
- non-text evidence has a stable typed contract
- unsupported modality behavior is named honestly
- product scorecard is evidence-based, not aspirational
