# Phase 62 Hard Gates

Phase 62 is not complete unless all of the following are true.

## 1. authoritative-state gate

- the final design clearly states where current autonomous-system truth now lives
- that state is Brainstack-owned and recallable across restart/session boundaries
- the closeout does not pretend transcript residue is an acceptable substitute

## 2. live-vs-historical gate

- the system can distinguish:
  - currently live
  - previously established but currently absent
  - merely discussed / claimed in older transcript
- the proof shows that old narration no longer silently becomes current truth

## 3. no-transcript-authority gate

- questions about heartbeat / pulse / evolver current status do not get answered from transcript residue first
- transcript and continuity, if used, are explicitly secondary evidence
- the closeout names what precedence change enforces that

## 4. no-heuristic-sprawl gate

- no keyword farm
- no locale-specific rescue
- no exact-thread or exact-phrase routing
- no new hidden phrase tables for "is it working"

## 5. recap-separation gate

- the implementation does not collapse Phase 61 recap routing and Phase 62 live-state authority into one mixed lane
- the closeout explains the separation clearly

## 6. cron-capability-truth gate

- the cron file-write incident receives a concrete verdict:
  - real missing tool surface
  - real permission denial after tool attempt
  - or false capability claim without tool attempt
- this verdict must be backed by session/log/code evidence, not speculation

## 7. boundary-discipline gate

- the phase does not drift into generic Hermes cron maintenance
- any host seam change is narrow and explicitly justified by Brainstack correctness
- if no host seam is needed, the phase says so explicitly
