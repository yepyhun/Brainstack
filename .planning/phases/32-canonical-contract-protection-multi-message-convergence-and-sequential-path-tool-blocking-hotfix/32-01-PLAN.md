# Phase 32 Plan

## goal

Hotfix the proven live breakpoints so canonical operating-rule truth can no longer degrade into partial Tier-2 revisions, explicit multi-message rule teaching converges correctly, and Brainstack-only tool blocking applies on every live tool path.

## scope

### in

- canonical behavior-contract source precedence and supersession protection
- explicit multi-message rule-teaching convergence for user-authored operating rules
- sequential-path Brainstack-only tool blocking parity
- demotion of `session_search` as a personal-memory fallback for full style-contract recall

### out

- broad behavior-policy redesign
- new side-memory systems
- general prompt or personality tuning unrelated to the proven seams

## core principles this phase must preserve

- follow the canonical immutable project principles:
  - [IMMUTABLE-PRINCIPLES.md](../../IMMUTABLE-PRINCIPLES.md)
- especially:
  - donor-first
  - fail-closed upstream compatibility
  - truth-first / no "good enough"
  - zero heuristic sprawl

## architecture decision

### accepted target model

- explicit user-authored canonical behavior-contract truth outranks weaker background/Tier-2 derivations
- multi-message rule teaching converges into one committed canonical revision instead of relying on assistant echoes or session-local reinforcement
- Brainstack-only guardrails are runtime-enforced uniformly across execution modes
- full rule recall comes from committed canonical contract truth or explicit fail-closed state, not from `session_search` or fragment stitching

### rejected model

- letting a shorter or weaker `tier2_llm` contract supersede a stronger explicit revision because it still compiles
- relying on guidance text alone to stop side-memory tools
- treating session-local reinforcement as a substitute for durable canonical update

## workstreams

### 1. canonical contract protection

- harden `behavior_contract` write admission so weaker partial `tier2_llm` revisions cannot supersede better explicit truth
- preserve compatibility with the first-class storage seam landed in `30.5`

### 2. multi-message convergence

- add an explicit convergence path for user-authored operating-rule teaching that spans multiple consecutive messages
- converge to one canonical committed revision instead of partial per-message capture

### 3. sequential-path tool blocking

- wire the Brainstack-only blocking seam into the sequential tool path, not only the concurrent one
- make `skill_manage` and similar side-memory writes fail closed on every live execution path

### 4. style-contract recall hardening

- ensure full-rule recall uses committed canonical contract truth
- prevent `session_search` from standing in as if it were a personal-memory owner for rule recall

### 5. observability and proof

- add targeted traces/tests proving:
  - the stronger explicit revision survives
  - multi-message teaching produces one canonical revision
  - both tool paths block side-memory writes identically

## validation ladder

1. explicit multi-message rule pack lands as one good canonical revision
2. weaker Tier-2 contract cannot supersede it
3. full rule recall comes from canonical truth, not `session_search`
4. `skill_manage` personal-memory writes are blocked in both sequential and concurrent execution paths
5. rebuilt runtime reproduces the fix on the same live-style scenario

## anti-goals

- do not reintroduce profile-lane shadow ownership for style contracts
- do not add new heuristic keyword farms as the main fix
- do not leave sequential and concurrent execution paths diverged
