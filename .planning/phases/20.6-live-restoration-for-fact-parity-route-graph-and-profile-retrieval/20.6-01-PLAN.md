# Phase 20.6 Plan

## Goal

Restore the live benchmark-path kernel after `20.5` by fixing the remaining restoration blockers rather than building new measurement infrastructure.

## Scope

### In

- fact-path parity against the earlier stronger decomp-off path
- route activation recovery
- sparse graph-yield recovery
- profile / identity retrieval misses
- bounded grounding leak audit for retrieval-correct / answer-wrong rows
- comparable answer-only and split reruns at the end

### Out

- new decomposition experiments
- new bonus tuning loops
- new evidence-priority wording loops
- donor replacement or donor bypass logic
- broad judge redesign beyond what already landed in `20.5`

## Planning Assumption

`20.5` already established the measurement baseline well enough.

`20.6` should therefore treat proof instrumentation as mostly frozen and spend its effort on live restoration, with only bounded audit work where the live answer still discards retrieved evidence.

## Execution Steps

1. Restore fact-path parity first.
   - document the exact old stronger baseline path again before touching code
   - compare packet shape, rendering, budgeting, and runtime assumptions against the current live fact path
   - identify what drifted inside the live fact path itself, not only in structural routes

2. Recover route activation and graph yield together.
   - treat route activation and graph yield as one interdependent restoration problem
   - widen route activation only in bounded, language-agnostic ways
   - recover graph yield in the same pass so newly activated routes do not just activate empty channels

3. Audit profile / identity retrieval misses explicitly.
   - treat `5d3d2817`-type misses as their own bounded class
   - inspect corpus retrieval / ranking / packing on identity-like fact questions
   - if the class does not reproduce, say so explicitly instead of speculating

4. Audit bounded grounding leaks explicitly.
   - inspect only proven retrieval-correct / answer-wrong rows such as `d682f1a2`
   - trace where the retrieved evidence is lost between memory context and final answer
   - if the class does not reproduce beyond the current known sample, say so explicitly

5. Run comparable reruns.
   - answer-only
   - split-proof
   - phase output must distinguish restoration improvement from better measurement

## Implementation Rules

- Do not let fact-path parity drift to the third or fourth workstream again.
- Do not treat route activation as success if graph yield remains empty.
- Do not treat graph yield as success if route activation still leaves almost every row on default fact mode.
- Do not let the profile / identity miss class disappear into generic “misc retrieval failure”.
- Do not let the grounding leak audit expand into a new proof phase.
- Do not let the phase claim success merely because split-proof remains more honest than raw answer score.

## Output Shape Required From Phase 20.6

The final summary must explicitly say:

- what changed on the live fact path
- whether structural route activation materially increased
- whether graph yield materially increased
- whether profile / identity retrieval improved
- whether the known grounding leak class reproduced and produced an actionable fix
- whether live restoration actually improved or remained insufficient

## Exit Gate

Phase `20.6` is done only when:

- fact-path parity has been explicitly audited and either improved or sharply explained
- route activation and graph yield have both been re-measured together
- profile / identity retrieval misses have been explicitly audited as a named class
- retrieval-correct / answer-wrong leakage has been explicitly audited as a named class
- final answer-only and split reruns are comparable to `20.5`
- if restoration remains weak, the summary says so directly instead of describing the phase as a partial win
