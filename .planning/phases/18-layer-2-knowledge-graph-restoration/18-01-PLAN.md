# Phase 18 Plan

## Goal

Restore Layer 2 so Brainstack can honestly claim Graphiti-shaped graph strength on an embedded backend, with `Kuzu` as the default target, without regressing back into local SQLite graph approximation or packaging-only wins.

## Scope

### In

- introduce a real L2 backend seam with `Kuzu` as the default implementation target
- move active graph storage and retrieval strength out of the current SQLite-only path
- preserve explicit current truth as primary while materially improving:
  - historical truth usefulness
  - conflict usefulness
  - inferred-link usefulness
  - relationship traversal usefulness
- keep L1’s graph channel contract stable while making the graph results behind it meaningfully stronger
- preserve cross-store ingest consistency and non-blocking provider behavior
- define a bounded graph eval ladder that proves graph usefulness rather than nicer packaging

### Out

- L3 corpus backend migration
- full donor-backed semantic/vector restoration for L1
- large benchmark chasing as the main driver
- host/runtime rebuild loops unless a bounded runtime smoke is actually needed

## Planning Assumption

`18` comes after `17`.

That means this phase must do two things at once:

1. replace the local SQLite graph center with a donor-aligned embedded graph path
2. do it without forcing another L1 redesign or smuggling L3 restoration into the same phase

This is not a cosmetic graph cleanup.
It is the real L2 restoration step.

## Execution Steps

1. Define the active L2 backend contract.
   - Introduce an explicit graph backend seam that can host `Kuzu`.
   - The contract must cover:
     - entity upsert
     - explicit relation upsert
     - inferred relation upsert
     - state upsert with supersession
     - conflict surfacing
     - graph search / traversal for L1 retrieval consumption
   - The seam must support later donor refresh work without another architecture reset.

2. Move graph write ownership behind the backend seam.
   - Re-route the current active graph writes away from direct SQLite centrality.
   - Keep Brainstack shell ownership only for:
     - publish / retry / partial-failure coordination
     - deterministic reconciliation rules that remain shell work
   - Do not let `graph.py` remain the place where donor graph strength is simulated.

3. Replace the active graph retrieval center.
   - The graph results returned to L1 must become materially richer than the current SQLite sort-and-package path.
   - Improve:
     - multi-hop usefulness
     - relationship surfacing
     - temporal graph recall
     - bounded inferred-link usefulness
   - Maintain clear class separation:
     - current explicit truth
     - historical truth
     - conflicts
     - inferred links

4. Thin down the local graph approximations.
   - `brainstack/graph.py` must stop being the effective graph-intelligence core.
   - `brainstack/db.py` must stop being the effective graph-retrieval engine.
   - `brainstack/retrieval.py` must not become the place where weak graph recall is cosmetically rescued.
   - `brainstack/usefulness.py` may retain telemetry only; it must not become the hidden graph relevance engine.

5. Preserve deterministic local reconciliation only where justified.
   - `reconciler.py` may stay local for deterministic state publication and conflict-safe writes.
   - It must not be used as a substitute for missing Graphiti-shaped graph intelligence.
   - If a rule belongs to graph reasoning rather than deterministic publication, it belongs on the donor-shaped L2 side.

6. Preserve non-blocking ingest and cross-store consistency.
   - Keep heavy graph extraction / graph publication out of the synchronous turn path.
   - Implement the first concrete cross-store ingest journal for shell state + `Kuzu`.
   - It must cover:
     - pending vs published graph-write state
     - idempotent replay
     - partial-failure visibility
     - resumable retry after interruption
   - Plan for:
     - idempotent graph publish
     - resumable graph ingest
     - partial-failure visibility
     - retry without duplicate truth publication
   - This remains Brainstack shell responsibility, not a new subsystem.
   - `19` may extend this mechanism to `Chroma`, but may not be the phase where the mechanism first appears.

7. Build the bounded Phase 18 graph eval ladder.
   - Gate A:
     - fast graph contract tests
     - class separation tests
   - Gate B:
     - graph usefulness scenarios
     - must prove better connected recall and relation usefulness
   - Gate C:
     - small graph-heavy real-world or benchmark-derived subset as a regression guard
   - Gate D:
     - bounded live/runtime smoke only if source-side proof leaves a real doubt

8. Prove L1 compatibility without another rewrite.
   - The L1 graph channel must consume stronger graph results through the existing executive retrieval contract.
   - `executive_retrieval.py` must not need another conceptual redesign to accept restored L2 power.

## Implementation Rules

- donor-first means Graphiti-shaped graph behavior, not Graphiti-flavored naming
- `Kuzu` is the default target unless execution evidence clearly disproves it
- no SQLite-only fallback as the accepted end state
- no shallow local graph hacks to imitate missing donor behavior
- no packaging-only success criteria
- no benchmark-gaming
- no hidden migration of L3 scope into this phase
- no synchronous heavy graph work in the turn path
- no new handwritten multilingual heuristics as a substitute for graph restoration

## Output Shape Required From Phase 18

The implementation must leave behind:

1. a real L2 backend seam with a `Kuzu`-default implementation path
2. thinner local graph modules
3. stronger graph retrieval results for the existing L1 graph channel
4. donor-faithful graph tests
5. a bounded graph eval ladder
6. the first concrete shell↔`Kuzu` cross-store ingest journal / publish-state mechanism
7. explicit proof that graph usefulness improved in substance, not only in packaging

## Exit Gate

- L2 is materially stronger in real graph behavior, not just tidier in presentation
- explicit current truth remains primary
- historical truth, conflicts, and inferred links are still clearly separated but now more useful
- relationship reasoning is materially better in realistic follow-up scenarios
- the active graph center is no longer the old SQLite local path
- the result is strong enough that calling the phase “Graphiti-recentered” is defensible
