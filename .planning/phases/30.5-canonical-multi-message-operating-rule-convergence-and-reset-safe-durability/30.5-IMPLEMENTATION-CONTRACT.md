# Phase 30.5 Implementation Contract

## invariant

Operating-rule truth must remain canonical, durable, inspectable, and user-authored.

## required implementation properties

- one canonical durable operating-rule revision exists per principal scope
- that canonical revision is backed by a dedicated first-class behavior-contract storage seam
- explicit multi-message user teaching can update that revision without requiring a single-message perfect paste
- assistant restatement is not canonical truth
- session-local reinforcement is clearly marked as session-local
- a successful durable update yields a committed receipt
- reset after a committed update preserves the same canonical revision
- exact recall after reset reads from the canonical revision, not from assistant echo text
- full-contract recall must not be synthesized from compiled policy summaries or slot fragments
- explicit operating-rule teaching must use a dedicated admission path rather than generic durable-admission rules

## prohibited outcomes

- assistant says “updated” without a committed canonical revision
- assistant-generated rule summary becomes the source of truth
- corrections only modify session-local reinforcement while pretending the durable rules changed
- reset silently discards an acknowledged operating-rule update
- generic stable-memory guardrails reject the user’s operating-rule teaching while the runtime continues as if the rules had been durably learned
- partial operational fragments are presented as if they were the authoritative complete rule set

## likely implementation seams

- provider pre-answer write path
- dedicated behavior-contract storage / revision record
- sync-turn durable update path
- canonical style-contract storage / revision metadata
- correction-to-canonical-update routing
- reset barrier and write receipt handling
- recall surface and inspectability helpers

## verify contract

- targeted source regressions
- shim parity smoke
- `hermes-final` reinstall and smoke
- live runtime reproduction of the real failure pattern
