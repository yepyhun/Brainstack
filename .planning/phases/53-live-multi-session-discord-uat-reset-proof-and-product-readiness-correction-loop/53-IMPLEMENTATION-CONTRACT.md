# Phase 53 Implementation Contract

## purpose

This phase exists to prove the rebuilt product in real use and correct any remaining live defects until the system is genuinely product-ready.

## non-negotiable constraints

- no new feature work
- no benchmark chasing
- no fake proof via selective scenarios
- no declaring success on a partial matrix
- no reopening architecture design during this phase except to delete, demote, or revert code that violates the current contract
- no host-level Brainstack behavior-governor regression
- no new micro-phase creation as a substitute for running the ladder
- no heuristic prompt-tuning whose only purpose is to make one rung pass
- no leaving dead or half-wired governance code in place “for safety”
- no accepting intermittent truth-state behavior as “good enough”
- no conflating wording variance with authority-state variance
- no ambiguous `fresh state` definition between ladder runs
- no using proactive continuity or personality polish as a reason to keep Phase 53 open after the product-ready gate is met
- no narrowing the native explicit profile seam into a hardcoded personal-field schema
- no locale-specific or language-specific profile parsing as a substitute for a generic memory-kernel integration
- no turning the Brainstack mirror lane into a text-only profile normalizer
- no new schema, enum, or validation layer whose purpose is to define what profile truth is “valid” during Phase 53 execution
- no implicit style/profile slot mining retained as a hidden product requirement
- no ordinary-turn Brainstack communication-policy injection used to mask weak explicit recall

## execution contract

1. Run the live six-step ladder on fresh mutable state.
2. Record every failure concretely.
3. Fix the narrowest real defect family.
4. Rerun the whole ladder, not only the failing rung.
5. If the failure comes from leftover governance, duplicate authority, or obsolete compatibility wiring, delete or sharply reduce that code instead of wrapping it.
6. Repeat each rung on fresh mutable state until truth-state behavior is stable across at least `3` runs.
7. Include adversarial reset, contradiction burst, and controlled dual-principal checks before closure.
8. Repeat until the matrix is green.
9. Wipe mutable state again and rerun the entire ladder from scratch.
10. Only treat the phase as complete after two consecutive all-green fresh-state runs.

Fresh-state contract:
- every ladder run must explicitly record what was wiped
- mutable session/runtime state must be wiped before each full run
- durable explicit truth must only be wiped when the rung is intentionally testing first-teach-from-nothing
- reset-durability rungs must preserve the durable explicit truth they are supposed to verify
- explicit rule packs are valid durable truths when taught explicitly, but free-chat residue is not an allowed substitute for explicit teaching

## architectural guardrails

- Hermes remains the owner of the native explicit write seam.
- Brainstack remains a mirrored memory kernel and recall layer.
- Delivery logic stays at the host seam.
- Brainstack must not reacquire ordinary-turn behavior control.
- Native explicit profile writes must remain generic durable truth, not a narrow `name/age/language/emoji` contract imposed by the kernel.
- Multimodal and generic durability must not be reduced to text-only personal-field normalization.
- Brainstack may preserve and recall an explicit archival rule pack, but must not synthesize communication-policy truth from ordinary chat unless that teaching was explicit.
- Ordinary turns must not carry Brainstack-generated communication-policy sections, native-preference prose sections, or equivalent hidden behavior-governor payloads.
- Phase 53 may remove bad architecture, but it must not introduce a new architecture direction.
- Corrections may demote or remove old Brainstack-governor remnants, but must not amputate valid bounded archival, mirror, or explicit recall roles.
- Native explicit write -> Brainstack mirror precedence should be enforced with code-level proof, not only plan intent.
- Native explicit write -> Brainstack mirror must be one-way and non-bouncing:
  - mirror records must remain mirror-class records
  - Brainstack must not promote a mirrored native explicit write into its own first-class explicit profile authority
- Owner-first routing should outrank cue-first keyword heuristics whenever an authority owner is available.
- Inspectability is part of correctness here: module responsibility must remain explainable after each fix.
- The shipped product-ready gate is explicit truth + correct recall + clean surface + stable reset behavior; it is not “automatic style obedience from free chat”.

## proof standard

The phase is only complete when:
- the full ladder passes end-to-end
- the ladder passes again on fresh mutable state
- failures, fixes, and residual risks are all explicitly documented
- each ladder rung has an explicit pass/fail verdict with evidence
- no known dead or half-wired shipped-path governance fallback remains unresolved
- authority precedence is proven by dedicated code-level and live evidence
- native-write-no-bounce invariant is proven by dedicated code-level evidence
- no rung remains intermittent at the truth-state level across `3` fresh-state repetitions
- latency and leak hygiene are explicitly measured, not assumed
- product-ready scope is met without quietly expanding the gate into proactive continuity or personality-polish work
- no remaining shipped-path dependency assumes ordinary-turn style/profile projection as part of the success criteria

## output artifacts required

- live test matrix
- iteration-by-iteration defect log
- per-rung pass/fail ledger for both fresh-state runs
- authority-precedence proof artifact
- native-write-no-bounce proof artifact
- deterministic truth-state ledger for the `3x` repeated rung runs
- latency ledger and leak-check artifact
- inspectability / dependency-responsibility artifact
- explicit rule-pack recall proof artifact
- ordinary-turn no-policy-projection proof artifact
- final readiness verdict
- explicit residual-risk note if anything remains
