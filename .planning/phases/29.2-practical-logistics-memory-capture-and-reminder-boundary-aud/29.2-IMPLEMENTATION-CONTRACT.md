# phase 29.2 implementation contract

## objective

Decide and harden the correct Brainstack ownership boundary for practical logistics memory:

- address-like facts
- appointment-like facts
- reminder handoff facts
- short todo facts

without turning the system into a noisy task dump or a second reminder database.

## system doctrine this phase must preserve

- Brainstack owns durable memory truth
- execution systems may own reminder delivery, not canonical memory
- durable memory should stay queryable and semantically clean
- ephemeral operational noise must not flood the durable shelves

## core principles this phase must preserve exactly

1. donor-first
   - reuse existing Brainstack shelves and settled temporal/entity patterns before adding anything new
2. modular, thin integration
   - keep the change at the smallest correct Brainstack seam
   - do not blur Chron, Brainstack, host wrapper, and runtime responsibilities
3. no heuristic sprawl
   - no brittle text-guessing engine for reminders or errands
   - only bounded class-level logic with explicit evidence is allowed
4. single durable owner
   - Brainstack may own only the durable truth that truly belongs in memory
   - Chron must not become a hidden memory owner
   - Brainstack must not become a second scheduler owner
5. native-system synergy, not replacement
   - Chron keeps execution scheduling
   - Brainstack adds durable recall truth only where that creates real later value
6. durable only where durable value is real
   - stable provider/place truth can deserve durability
   - transient same-day clutter should stay out unless explicitly promoted or repeatedly proven durable

## non-negotiables

- no blanket promotion of all logistics text into profile rows
- no reminder shadow database hidden in cron metadata
- no leaving obviously useful stable real-world facts as transcript-only forever
- no broad product pivot into a planner or task app

## acceptable outputs

- a class-by-class ownership map
- a bounded repair for the classes that clearly deserve durable capture
- an explicit no-persist decision for classes that should stay ephemeral

## unacceptable outputs

- "store everything"
- "store nothing"
- "cron already knows it, so Brainstack does not need any truth"
- any design where future recall still depends mainly on transcript luck for stable logistics facts

## protected boundaries

### durable-owner boundary

- Brainstack remains the only durable owner

### scheduling boundary

- reminder scheduling remains an execution concern, not the durable truth store

### explicit Chron boundary

- Chron owns:
  - reminder creation
  - reminder timing
  - reminder triggering
  - reminder delivery
- Brainstack may own:
  - stable provider/place facts
  - bounded appointment truth with later recall value
  - minimum reminder residue only when later coherent recall requires it
- Brainstack must not own:
  - full scheduler state
  - execution metadata as a shadow reminder database
  - every short-lived todo by default

### noise boundary

- transient errands and same-day clutter should not become a permanent profile trail by default

## minimum evidence required before calling phase 29.2 done

- one live inventory of current shelf placement for the observed logistics facts
- one explicit ownership map for each fact class
- one bounded decision on what gets durable capture now
- one bounded decision on what explicitly stays non-durable
- one regression plan covering reminder behavior and durable recall

## current best repair shape to prove or reject

- stable provider/place facts:
  - likely deserve durable capture
- appointment instances:
  - likely belong in temporal or typed event lanes rather than plain profile rows
- reminder handoff:
  - needs minimal durable residue, not full scheduler-state duplication
- transient short todos:
  - likely explicit non-durable by default unless repeated or promoted

## recommended model level

- `high`
- this phase is primarily a boundary and data-model decision, so it needs careful reasoning but not the wider live forensic spread of Phase 29
