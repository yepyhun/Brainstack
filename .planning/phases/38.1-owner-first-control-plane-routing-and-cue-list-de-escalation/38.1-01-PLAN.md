# Phase 38.1 Plan

## goal

Move the control plane from cue-first packet shaping toward owner-first routing:

- use owner-derived signals as the primary routing and budget inputs
- demote cue lists to narrow fallback only
- preserve existing packet quality gains

## scope

### in

- `control_plane` routing and packet-shaping cleanup
- owner-signal-first policy shaping
- cue-list de-escalation
- regressions for preference, temporal, operating, and task surfaces

### out

- broad retrieval rewrite
- new parser layer inside the control plane
- cue farm replacement with another local heuristic engine
- packet or prompt redesign

## architecture decision

### accepted target model

- control-plane policy is driven first by:
  - owner parser outputs
  - route-resolver outputs
  - retrieved channel support and conflicts
- cue lists remain only as narrow boundary fallback when no stronger signal exists

### rejected model

- query-shape cue lists as the main routing brain
- hidden heuristic scoring layers
- broad local classification logic inside the control plane

## workstreams

### 1. routing audit

- map where cue-derived booleans currently drive policy
- identify which paths can switch to owner signals immediately

### 2. owner-first refactor

- demote cue lists
- promote owner parser outputs and route results
- preserve high-stakes boundary safety

### 3. proof

- add regressions for non-starved preference queries
- preserve operating and task retrieval quality
- show cue lists are no longer the primary packet shaper

## validation ladder

- source:
  - targeted control-plane regressions
  - broader packet / retrieval guards
- `hermes-final`:
  - payload parity and direct interpreter proof
- live runtime:
  - rebuilt runtime smoke on preference / operating / temporal / task queries
