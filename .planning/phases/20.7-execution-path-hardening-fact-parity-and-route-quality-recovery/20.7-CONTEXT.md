# Phase 20.7 Context

## Why This Phase Exists

Phase `20.6` removed two false explanations:

- the installed Bestie plugin really had a stale `executive_retrieval.py`
- the live route-hint path really was failing on MiniMax response shape

Both were fixed.

After those fixes, live structural routing actually activated and backend population was no longer mostly empty.

But the real live result got worse:

- answer-only: `3 / 15`
- split raw answer score: `2 / 15`
- split `retrieval_correct`: `2 / 15`
- split `both_correct`: `1 / 15`

That means the next phase is no longer about proving that the route path is wired.

It is about fixing the live retrieval-quality path itself, while also closing a separate execution-path correctness bug.

## What This Phase Is

This is a live retrieval-quality recovery phase with five bounded targets:

1. execution-path anomaly hardening
2. fact-path parity restoration
3. route-quality recovery, explicitly split into:
   - route selection
   - candidate quality
   - evidence packing
4. profile / identity retrieval recovery
5. bounded grounding leak carry-through

## What This Phase Is Not

This phase is **not**:

- a new proof-calibration phase
- a new wiring-fix phase
- a new decomposition phase
- a new bonus / prompt wording tuning loop
- a donor bypass phase
- a benchmark-gaming phase

## Entry Conditions

Phase `20.6` ended with these explicit truths:

- live route activation is now real, not dead
- backend population is now present in both graph and corpus stores for most rows
- fact-path parity is still badly below the old stronger decomp-off path
- `5d3d2817` remains a named profile / identity retrieval miss
- `d682f1a2` remains a named retrieval-correct / answer-wrong grounding leak
- `6a1eabeb` and `gpt4_7f6b06db` exposed a separate execution-path anomaly:
  - blank route metadata
  - missing `<memory-context>` block

## Target End State

At the end of `20.7`, the system must be able to say clearly:

- whether the silent execution-path anomaly is fixed
- whether live fact-path parity materially improved
- whether route-quality failure is primarily:
  - selection
  - candidate quality
  - packing
- whether the profile / identity miss class improved
- whether the bounded grounding leak class still reproduces after the retrieval fixes

If live restoration remains insufficient, the summary must name the dominant blocker directly rather than describing the result as mixed or promising.
