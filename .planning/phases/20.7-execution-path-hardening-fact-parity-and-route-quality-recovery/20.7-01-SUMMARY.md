## Verdict
- `20.7` did not restore live parity.
- The old `20.6` execution-path anomaly was real and is now fixed on the reproduced path:
  - no blank route metadata
  - no missing memory block
- But after hardening the route parser, the full comparable reruns collapsed back to total fact fallback:
  - `route_source=fallback`: `15 / 15`
  - `requested_mode=fact`: `15 / 15`
  - `applied_mode=fact`: `15 / 15`
- The next blocker is therefore no longer silent execution failure.
- The next blocker is that live route-hint output is too ambiguous / unusable under the current strict parser contract, so structural routing is not activating safely.
- Fact-path parity also remains badly unresolved versus the older `20.2` decomp-off `10 / 15` baseline.
- Retrieval remains the dominant failure layer on the split run.
- The agreed next-scope guard for `20.8` is now explicit:
  - do not assume the next route-hint fix is just more prompt tightening
  - begin with a raw-output route-hint audit on the benchmark questions
  - explicitly compare two options:
    - fix the current LLM route-hint contract strongly enough to emit only `fact` / `temporal` / `aggregate` / `unknown`
    - replace the LLM route-hint call with a simpler, more deterministic classifier if the contract cannot be made reliable
  - this `replace` option is considered donor-first-consistent, not donor drift
