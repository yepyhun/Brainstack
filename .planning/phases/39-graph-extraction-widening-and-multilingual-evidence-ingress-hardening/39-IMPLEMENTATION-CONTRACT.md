# Phase 39 Implementation Contract

## invariant

Brainstack must strengthen typed graph ingress without expanding local heuristic text extraction into the graph layer's main intelligence.

## required implementation properties

- typed graph evidence remains the only graph write path
- malformed or unsupported graph evidence fails closed
- ingress receipts or traces are explicit enough for runtime inspection
- the implementation stays compatible with future non-text producers of `GraphEvidenceItem`
- packet or prompt noise does not increase

## explicit anti-goals

- language-specific regex farms
- phrase-table multilingual harvest
- broad semantic widening hidden inside boundary code
- new graph prompt-steering layers

## likely implementation seams

- `brainstack/graph_evidence.py`
- `brainstack/graph.py`
- `brainstack/extraction_pipeline.py`
- graph-related tests only where needed

## verify contract

- prove valid typed graph evidence still persists cleanly
- prove malformed or unsupported graph evidence fails closed
- prove improved trace or receipt observability where applicable
- prove parity on source, `hermes-final`, and live runtime
