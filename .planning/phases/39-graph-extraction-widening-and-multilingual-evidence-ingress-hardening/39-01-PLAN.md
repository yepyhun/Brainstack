# Phase 39 Plan

## goal

Strengthen the typed graph-evidence boundary after the packet and control-plane cleanup:

- keep the existing typed graph-evidence write seam
- make graph ingress receipts, provenance, and failure behavior easier to inspect
- avoid turning text extraction into the graph layer's primary intelligence

This phase should make graph truth:

- more auditable
- more fail-closed
- more runtime-parity-safe
- more ready for future multimodal typed producers

## scope

### in

- typed graph ingress contract hardening:
  - tighten what counts as valid typed graph evidence at the boundary
  - make unsupported or malformed graph evidence fail closed with explicit proof
- graph observability:
  - improve receipts, traceability, or runtime proof around typed graph ingress
  - make source, install, and live runtime parity easier to verify
- boundary-safe implementation only:
  - stay inside existing typed evidence discipline
  - preserve prompt and packet quietness

### out

- graph extraction widening by adding more phrase families
- multilingual phrase harvest
- broad ontology or schema expansion
- prompt-packet redesign
- graph-first runtime steering

## architecture decision

### accepted target model

- typed graph evidence remains the only graph write path
- graph correctness comes from contract hardening and observability first
- any future semantic widening must come from donor-aligned producers, not local text regex expansion

### rejected model

- local regex-family growth as the main graph intelligence
- text-only widening sold as multimodal progress
- hiding malformed ingress behind quiet degradation

## workstreams

### 1. boundary contract audit

- map the current typed graph ingress assumptions
- freeze the accepted fail-closed contract

### 2. observability hardening

- make graph ingress receipts and trace surfaces clearer
- ensure source/install/runtime all expose enough proof for inspection

### 3. proof

- add regressions for malformed or unsupported typed graph ingress
- preserve typed fail-closed writes
- prove no packet-noise regression

## validation ladder

- source:
  - targeted graph-boundary regressions
  - typed fail-closed and receipt proof
- `hermes-final`:
  - payload parity and direct interpreter proof
- live runtime:
  - rebuilt runtime smoke on typed graph ingress and failure cases
