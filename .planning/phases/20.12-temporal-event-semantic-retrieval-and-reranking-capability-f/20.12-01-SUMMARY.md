Phase `20.12` is execution-complete at gate.

It delivered a real temporal-event semantic retrieval / reranking capability layer, proved it locally, and evaluated both the bounded internal path and a stronger external candidate. The residual `gpt4_7f6b06db` gap did not collapse under either scorer, so the phase should be read as a **capability decision and bounded implementation win**, not as a full temporal-answer recovery.

## Capability Decision

### Bounded internal route

The bounded hybrid over the current Chroma scorer was worth doing first and is now proven.

But the live evidence also showed its limit:

- the scorer path no longer crashes
- yet the internal semantic signal still favors irrelevant `temporal_event` rows in the hard temporal case
- therefore the current local bounded hybrid is **not sufficient** as the sole temporal-event relevance layer

### External scorer route

The Jina v5 TEI path is now also evidence-backed rather than just plan text:

- a local TEI service was already available
- a direct query/document probe gave a stronger ranking signal for the trip examples than the current Chroma scorer
- an env-gated external scorer path was wired into the bounded temporal-event rerank layer

But the targeted live proof showed the harder truth:

- even under the Jina-gated scorer, the relevant trip temporal-event rows still did not become the final selected temporal continuity evidence
- candidate-debug showed that the temporal pool was still dominated by unrelated recent events
- in the Jina proof, the fused candidate set only contained one trip-related row, and it was a transcript row rather than the intended temporal-event chain

So the external scorer is a valid capability layer, but it is **not sufficient by itself** for the residual `gpt4_7f6b06db` miss.

## What 20.12 Does Claim

- a bounded temporal semantic rerank capability now exists in code
- a stronger external scorer can now be evaluated without a full backend migration
- Jina v5 TEI has earned “preferred external evaluation candidate” status through local evidence, not only through planning preference
- the phase proved that scorer strength alone does not resolve the current temporal miss

## What 20.12 Does Not Claim

- it does **not** claim that Jina v5 TEI solved the residual temporal case
- it does **not** claim that the remaining miss is just another ranking tweak away
- it does **not** justify a full corpus-wide embedder swap
- it does **not** reopen benchmark-specific lexical or route hacks

## Carry-Forward Reading

The next blocker is now more precise than before `20.12`:

- not route-hint
- not cheap temporal shortlist/query shaping
- not simple packing-only
- not “no semantic reranking exists”

The next blocker is:

- **upstream temporal-event candidate availability / generation reliability**
- specifically, the desired trip chain is still not reliably present in the bounded temporal-event pool that the reranker receives

This means the next phase should target:

- why the relevant trip events are not consistently present as temporal-event continuity rows in the live bounded pool
- and only secondarily whether a stronger scorer should stay enabled after that upstream truth is fixed
