# Phase 20.22 Context

## Why this phase exists

`20.21` crossed a real production threshold:

- deployed Bestie now preserves the warm Brainstack provider across `/reset` finalize
- live reset now writes typed graph state
- live aggregate recall now sees graph candidates
- the isolated deployed-live reset eval recovered to `5 / 5`

That means the current stack is no longer blocked on the earlier graph/corpus availability seam.

The honest next question is no longer:

- “what is the next aggregate micro-fix?”

It is:

- “how good is the deployed product now across broader live conversation quality targets?”

## Product targets now in play

This phase is the first one after `20.21` where the leading questions should come directly from the project doctrine:

- coherent continuous conversation
- proactive stateful continuity
- long-range accurate recall and relation-tracking
- usable storage of large bodies of knowledge
- meaningful token savings

Right now that would be premature.

`20.21` already proved:

- the live graph/corpus stack is not just “enabled on paper”
- the deployed Bestie runtime can now produce graph-native state and live graph candidates

Without a broader deployed-live read, another feature phase would risk:

- benchmaxing drift
- solving the wrong next problem
- piling new capability on top of unmeasured product behavior
