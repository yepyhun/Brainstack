## workstreams

### 1. classify typed invariant enforcement modes

- separate:
  - deterministic repairable invariants
  - non-repairable but blockable invariants
  - advisory-only invariants that do not belong in the typed final-output layer
- ensure the typed output contract expresses that distinction explicitly

### 2. harden final-output enforcement semantics

- if a typed invariant violation remains after deterministic repair, do not let the runtime silently ship the answer
- make the outcome explicit:
  - repaired
  - blocked / regenerated
  - fail-closed with deterministic reason

### 3. remove trace-only obedience gaps

- eliminate the state where:
  - violation is known
  - trace records it
  - user still receives the violating content
- keep the proof surface, but do not let proof substitute for enforcement

### 4. keep enforcement generic

- fix the whole typed invariant bug class, not only dash punctuation
- do not add punctuation-specific hacks where a general typed-invariant outcome model is the correct seam

### 5. add deterministic proof

- replay and unit coverage for:
  - repairable typed invariants
  - non-repairable typed invariants
  - remaining-violation handling
  - runtime send-path obedience
  - trace parity between validator result and final shipped content
