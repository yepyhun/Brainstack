# Phase 40 Plan

## goal

Close the remaining memory-integrity gap by making Brainstack memory authority converge across write, read, repair, and debug surfaces without adding new heuristic routing or text-only packet tricks.

This phase is not a style-only patch.

This phase should establish:

- a global memory transaction boundary
- anti-regeneration guards for style authority
- generation-linked authority convergence across canonical contract, compiled policy, and ordinary-turn active lane
- deterministic out-of-band proof surfaces for operators and tests

## scope

### in

- global transaction boundary hardening:
  - nominal read surfaces do not create, patch, supersede, or otherwise mutate any durable truth class
  - this must hold for:
    - style-contract truth
    - task memory truth
    - operating truth
  - explicit writes remain available only behind deliberate write paths
- historical repair:
  - define auditable quarantine, supersede, and tombstone semantics for already-polluted canonical behavior-contract generations
  - support dirty-store repair without hiding that repair occurred
- anti-regeneration:
  - prevent transcript-derived or profile-derived style fragments from reconstituting an active communication authority when a canonical style generation already exists
  - keep profile and graph rows in their own owner roles instead of letting them become a shadow style-contract source
- authority convergence:
  - make compiled policy and ordinary-turn active lane converge on the same canonical authority lineage
  - self-heal compiled policy from clean canonical truth when possible
  - otherwise fail closed with explicit status rather than drifting onto pseudo-contract fallback
- deterministic out-of-band debug and proof:
  - expose machine-readable Brainstack authority state separately from the model-facing packet
  - prove both clean-store and dirty-store flows
  - prove read-side effect count is zero on nominal lookup surfaces

### out

- broad owner-first routing redesign
- route-resolver policy redesign
- broad transcript/profile heuristic cleanup outside the anti-regeneration seam
- graph semantic widening or multilingual phrase expansion
- prompt cosmetics or packet-sludge work not required for authority convergence

## architecture decision

### accepted target model

- one principal scope may have many historical behavior-contract generations, but only one active canonical generation
- exact canonical recall and ordinary-turn active lane may remain different renderings, but they must share one authority lineage
- nominal read surfaces do not mutate durable truth
- repair is explicit, auditable, and non-destructive by default
- debug truth is out-of-band and machine-readable
- Brainstack-owned packet truth is separated from host/runtime prompt layers in operator-facing debug output

### rejected model

- fixing the issue by wiping memory as the product solution
- fixing only style-contract writes while leaving task/operating read-side mutation alive
- letting profile or graph residue re-synthesize a contract-like authority when canonical style truth already exists
- relying on model self-report as the debug surface
- solving the gap with new cue farms or broader prompt steering

## workstreams

### 1. transaction boundary inventory

- map every nominal read surface that can currently trigger durable mutation
- classify each mutation path by owner:
  - style-contract
  - task
  - operating
- freeze the accepted write entrypoints

### 2. read-side mutation removal

- remove durable mutation from read-only prefetch and recall paths
- keep explicit write APIs and explicit teaching/correction seams intact
- separate model-facing packet assembly from operator-facing receipts and traces

### 3. historical repair and generation control

- add auditable repair operations for polluted canonical generations
- keep lineage visible:
  - active generation
  - superseded generation
  - quarantined generation
  - tombstoned generation

### 4. anti-regeneration guard

- stop transcript-derived and profile-derived style atoms from becoming active communication authority when canonical style truth already exists
- keep fallback profile rows visible only as profile evidence, not as contract synthesis material

### 5. authority convergence and self-heal

- bind compiled policy and ordinary-turn active lane to the same canonical authority lineage
- rebuild compiled policy from clean canonical truth when the compiled row is missing or stale
- if rebuild is impossible, expose explicit fail-closed state instead of shadow fallback authority

### 6. deterministic proof surface

- add out-of-band snapshots that report:
  - active canonical generation
  - compiled policy status and source lineage
  - active-lane source lineage
  - Brainstack-owned rendered sections
  - suppressed rows or residues
  - read-side effect count
  - repair events
  - fallback reason when present
- explicitly distinguish:
  - Brainstack-owned packet state
  - host/runtime prompt layers

### 7. proof

- clean-store proof:
  - teach canonical contract
  - reset
  - exact recall and ordinary-turn lane converge on the same lineage
- dirty-store proof:
  - polluted store is repaired audibly
  - stale residues stop re-synthesizing authority
- transaction proof:
  - recall/debug questions do not write
- packet/debug proof:
  - debug surface stays out-of-band
  - model-facing packet no longer carries write receipts or operator traces

## validation ladder

1. source unit and integration gates for transaction boundary, repair, lineage convergence, and proof surfaces
2. dirty-store replay proving repair without wipe as product solution
3. fresh-store replay proving clean teaching and reset recall
4. installed `hermes-final` parity and doctor proof
5. live runtime smoke proving:
   - no read-side writes
   - convergent generation ids
   - deterministic out-of-band debug snapshot
