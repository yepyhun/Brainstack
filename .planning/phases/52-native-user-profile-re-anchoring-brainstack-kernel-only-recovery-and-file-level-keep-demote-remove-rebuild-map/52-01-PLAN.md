# Phase 52 Plan

## goal

Re-anchor explicit user/profile truth to the Hermes native explicit-memory seam, while demoting Brainstack back to a donor-first memory kernel that mirrors and augments native memory instead of acting like a parallel profile governor.

The target product is:
- chat-first host
- donor-first memory shell
- one explicit profile authority
- Brainstack as memory kernel, not behavior engine

## workstreams

1. Native authority re-anchor
- treat the Hermes native explicit-memory seam as the first-class explicit profile path
- require user identity, durable preferences, and explicit user-profile facts to land there first
- keep Brainstack behind the provider seam as a mirror/augment layer, not a rival authority

2. Authority precedence contract
- define one explicit precedence table for every truth class:
  - explicit user identity
  - explicit user preference
  - explicit communication/style rule packs
  - operating truth
  - task truth
  - transcript evidence
  - graph/corpus evidence
- state for each class whether it is:
  - native-authoritative
  - Brainstack-authoritative
  - mirrored
  - supporting evidence only
- reject any dual-primary or ambiguous truth class

3. File-level keep / demote / remove / rebuild map
- produce one file-level matrix that classifies:
  - host-native anchors
  - Brainstack kernel files to keep
  - Brainstack governance files to demote
  - reply-path or profile-primacy logic to remove or sharply reduce
  - bridge seams to rebuild around native writes

4. Native-to-Brainstack migration path
- define how current explicit Brainstack-held profile/style truth re-anchors to native Hermes storage
- classify migration decisions:
  - move to native path and mirror back into Brainstack
  - keep only as archival Brainstack truth
  - supersede/tombstone in Brainstack where native profile becomes canonical
- require one-way convergence:
  - native explicit writes may update Brainstack
  - Brainstack inference must not silently overtake native explicit truth
- define native-unavailable semantics:
  - if native explicit authority is unavailable, surface `stale mirrored snapshot` or `no native explicit authority available`
  - never silently rebuild first-class explicit truth from transcript/profile residue as a substitute

5. Brainstack kernel-only shape
- keep Brainstack strong in:
  - transcript continuity
  - recall augmentation
  - task / operating truth
  - graph / corpus
  - cross-store consistency
- demote Brainstack in:
  - profile primacy
  - style-governance pressure
  - ordinary-turn behavior shaping

6. Native-write mirror contract
- define the exact rule for built-in writes:
  - native host writes first
  - Brainstack mirrors through `on_memory_write()`
- ensure no second explicit profile authority path is needed for the common case
- keep archival style-contract recall explicit and bounded
- require one-way idempotent mirroring:
  - every mirror record carries `native_write_id`, `source_generation`, and `mirrored_from`
  - mirrored native truth must never be emitted back upstream as a new explicit host write
  - replaying the same native write must converge, not fork duplicate truth

7. Anti-regression guard
- prevent Brainstack from regrowing a shadow explicit profile authority from:
  - ordinary chat inference
  - transcript-derived style fragments
  - tier1/tier2 extracted preference candidates
- candidate or extracted state must remain supporting until confirmed by the native explicit path
- explicitly close the known rebuild seams:
  - transcript-derived style atom creation in `plugins/memory/brainstack/profile_contract.py`
  - profile+graph communication-contract reconstruction in `plugins/memory/brainstack/retrieval.py`
  - ordinary-turn query-shape escalation in `plugins/memory/brainstack/control_plane.py`

8. Tooling and payload parity
- rebase the shipped and verified surfaces in the same phase:
  - `README.md`
  - `install_into_hermes.py`
  - `scripts/brainstack_doctor.py`
  - host payload trees
  - tests
  - proof/docs surfaces
- remove or rewrite any brainstack-first instruction that conflicts with native explicit-profile primacy

9. Execution order
- sequence the recovery so we do not break product behavior:
  - lock native anchors
  - lock authority precedence
  - lock mirror contract
  - demote Brainstack custom profile primacy
  - migrate/re-anchor current explicit state
  - align installer/doctor/docs/payload parity
  - keep kernel retrieval intact
  - reroute explicit profile writes through native memory
  - then do end-to-end proving

10. Proof contract
- define proof for the desired end state:
  - user tells the bot their name and preference
  - the native explicit-memory seam records it
  - Brainstack mirrors it
  - ordinary chat stays natural
  - explicit recall returns the right truth
  - post-reset recall still works
  - no shadow Brainstack-only explicit profile authority reappears
  - doctor/installer/docs/tests all reflect the same architecture
  - native-unavailable mode fails explicitly instead of residue reconstruction
