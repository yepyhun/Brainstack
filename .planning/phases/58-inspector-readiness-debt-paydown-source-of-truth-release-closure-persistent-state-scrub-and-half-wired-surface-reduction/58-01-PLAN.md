  - `host_payload/agent/brainstack_mode.py`
  - behavior/style migration helpers in `brainstack/db.py`

- keep the new `croniter` dependency fix in the source repo
- ensure doctor catches every dependency or source/runtime drift needed for the cleaned state
- ensure release/install proof is one-command reproducible onto the target checkout

### 5. planning and audit debt closure

- update `STATE.md`, `ROADMAP.md`, and phase artifacts so they no longer imply open issues that are already closed in code
- where an old issue is still real, carry it forward explicitly into Phase 58 instead of leaving it as vague historical residue

### 6. final inspector-readiness proof and release

- run targeted tests for every touched debt family
- run doctor on the installed runtime
- verify persistent-state cleanliness on the installed runtime
- produce a short inspector-facing execution result that says:
  - what was cleaned
  - what is now guaranteed
  - what residual risk remains, if any
- cut a clean source-of-truth release only after the repo and proof story are coherent

## acceptance criteria

- source-of-truth repo is clean and releasable
- `finafina` can be brought to the corrected state from the source repo without hand edits
- live persistent state no longer contains:
  - interrupt/status transcript contamination
  - active or superseded behavior-authority residue that contradicts the native-truth / mirror-only model
- doctor proves all required runtime dependencies and invariants, including cron scheduling support
- shipped half-wired surfaces have explicit keep/remove decisions and the deadest residue is removed or quarantined

- if inspector-ready is claimed, the release boundary and proof boundary are both explicit and coherent

## anti-goals

- no new capability build
- no broad architectural rewrite
- no heuristic patching to rescue one Discord example
- no target-only cleanup that cannot be reproduced from source of truth
- no pretending that dead-code tooling alone proves safe deletion
- no declaring inspector readiness from runtime health alone while the repo, planning corpus, or persistent state remain dirty
