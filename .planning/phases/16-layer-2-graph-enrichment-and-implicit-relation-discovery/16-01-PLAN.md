# Phase 16 Plan

## Goal

Turn Brainstack Layer 2 from a correct but relatively shallow graph into a stronger, more useful knowledge graph with:

- explicit truth classes
- bounded implicit relation discovery
- stronger graph retrieval
- cleaner graph recall packaging

This phase is broader than “improve L2 extraction”.
It is an L2 architecture phase.

## Scope

### In

- define the target L2 truth model
- add explicit separation for:
  - explicit truth
  - inferred links
  - stale/retired records
  - conflicts
- strengthen graph retrieval/ranking
- strengthen graph recall packaging
- add bounded implicit relation discovery
- preserve temporal/provenance correctness
- keep the solution Brainstack-owned and installer-carried

### Out

- no new memory runtime
- no extra user-facing memory tools
- no scratchpad
- no benchmark-driven roadmap drift
- no keep-inspired skill+notes expansion
- no Tier-1 heuristic growth to fake graph intelligence

## Execution Steps

1. Lock the target L2 model before coding.
   - define the exact graph fact classes
   - define what is current truth vs historical vs inferred vs conflict
   - define what may replace current internals and what must stay stable

2. Design the new L2 retrieval contract.
   - define graph ranking inputs
   - define current-first behavior
   - define when inferred links may surface
   - define when historical truth is included
   - define conflict escalation behavior

3. Implement explicit graph fact-class separation.
   - extend the internal L2 model so inferred and stale records are not flattened into the same surface as explicit current truth
   - preserve Phase 13 temporal/provenance guarantees

4. Implement bounded implicit relation discovery.
   - build a candidate-link pipeline inside Brainstack
   - keep it bounded, confidence-aware, and non-magical
   - do not create a second graph engine
   - do not rely on broad lexical heuristics

5. Replace the weak L2 retrieval kernel where needed.
   - improve graph ranking
   - improve graph packing
   - use Mnemosyne only as narrow inspiration for internal retrieval discipline
   - do not import its tool-heavy architecture

6. Add focused validation.
   - explicit truth still wins over inferred link
   - historical truth still behaves correctly
   - conflicts still surface correctly
   - inferred links do not flood recall
   - live recall feels stronger without becoming noisier

7. Carry the phase through the installer and live runtime.
   - no live-only patch
   - doctor/smoke should prove the new L2 behavior in installed Bestie

## Design Rules

- Brainstack remains the only memory owner
- correctness before cleverness
- retrieval engine improvements over heuristic growth
- multilingual-safe by architecture, not by regex expansion
- no benchmark chasing
- no god-object growth
- keep modules narrow and replaceable

## Exit Gate

- Brainstack L2 has a clear internal truth-class model
- implicit relation discovery exists and is bounded
- graph retrieval is materially stronger than the current lexical row-priority path
- recall packaging clearly separates current truth, history, inferred links, and conflicts
- temporal/provenance guarantees remain intact
- the solution stays modular, Brainstack-owned, and installer-carried
