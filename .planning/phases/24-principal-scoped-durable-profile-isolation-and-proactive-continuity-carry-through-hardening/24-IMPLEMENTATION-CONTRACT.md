# Phase 24 Implementation Contract

## Objective

Close the two post-Phase-23 residuals in the right order:

- principal-scoped durable profile isolation
- proactive continuity carry-through hardening

without reopening the settled Brainstack/native boundary or smuggling a SHIBA-style capability expansion into a correctness phase.

## System doctrine this phase must preserve

- Brainstack remains the owner of durable personal-memory truth.
- Phase `22` boundary decisions remain the baseline.
- Phase `23` remains the broader live-quality baseline.
- Correctness on the durable profile axis outranks new extraction sophistication.
- The project guardrails remain active and explicit:
  - donor-first
  - modularity / upstream updateability
  - truth-first
  - fail-closed on the owned axis
  - no benchmaxing
  - no overengineering

## Workstream A: Principal-scope isolation diagnosis

- Reproduce the cross-principal durable profile bleed.
- Start with a cheap canary before any broader rerun:
  - principal A gets a few strong profile/style markers
  - principal B gets none
  - B must stay clean after reset
- Inspect:
  - principal scope keys
  - durable profile writes
  - retrieval filtering
  - contract injection inputs
  - session-finalize / flush behavior

Required artifact:

- one isolation truth note with the first proven failing seam
- one canary result showing whether the bleed is still alive before and after the repair

Protected rule:

- do not patch carry-through first while scope hygiene is still uncertain

## Workstream B: Proactive continuity miss diagnosis

- Reproduce the dietary carry-through miss.
- Include one negative control with no dietary constraint so the repair cannot win by over-carrying invented constraints.
- Inspect:
  - extracted/profiled truth
  - persisted truth
  - retrieval rows
  - packet composition
  - final answer behavior

Required artifact:

- one carry-through truth note with the first proven drop point
- one negative-control result showing whether the fix remains selective

Protected rule:

- do not assume the problem is “memory forgot it” without layer evidence

## Workstream C: Shared-seam decision

- Explicitly test whether one lower seam contributes to both residuals.

Required artifact:

- one shared-seam verdict:
  - yes, shared seam
  - or no, separate seams

Protected rule:

- if a real architecture fork appears here, bring it back to the user in simple language before locking the repair path

## Workstream D: Bounded repair

- Repair the deepest proven seam first.
- If the residuals are separate, still keep the order:
  - isolation repair
  - then carry-through repair

Protected rule:

- no broad SHIBA-style uplift
- no broad retrieval/scoring rewrite
- no host-monolith redesign

## Workstream E: Live proof and residual refresh

- Rerun the residual families on the live path.
- Run the cheap principal canary and the carry-through negative control as the first recheck before broader reruns.
- Reclassify the outcome honestly:
  - fixed
  - improved but residual remains
  - or deeper issue still present

Protected rule:

- no green closeout without explicit rerun truth

## Protected boundaries

### Anti-overengineering boundary

- no new large subsystem
- no broad host cleanup
- no generic “make Tier-2 smarter” detour

### Boundary-stability / donor-risk boundary

- do not casually reopen the Phase `22` coexistence model
- do not disable legitimate native features unless the evidence directly proves they cause the residual

### Capability-deferral boundary

- SHIBA-style missing pieces remain valid later work
- they are not the primary fix vehicle here unless the narrow path is proven insufficient

### Truth boundary

- persisted-state evidence outranks self-explanation
- packet evidence outranks intuition about “what the model probably forgot”
- scope bleed must be fixed at the real ownership/filtering seam, not hidden behind smarter extraction

## Minimum evidence required before calling Phase 24 done

- one reproduced or explicitly falsified principal-scope bleed path
- one reproduced or explicitly falsified carry-through miss path
- one principal-bleed canary result
- one carry-through negative-control result
- one localization verdict for each residual
- one shared-seam verdict
- live rerun truth after the bounded repair
- one honest next-thread recommendation
