# phase 29.3 summary

## verdict

Phase `29.3` closed with a non-heuristic replacement.

The final model is:

- compact communication slots stay as the ordinary-turn lane
- one canonical principal-scoped `preference:style_contract` row owns the detailed Humanizer contract
- explicit style/rules recall uses the existing model-backed route seam with `style_contract` mode
- ordinary generic retrieval and prompt projection exclude the long-form style contract

## implemented surfaces

- `brainstack/style_contract.py`
  - canonical long-form style-contract payload normalization and rendering
- `brainstack/tier2_extractor.py`
  - structured `style_contract` output
- `brainstack/reconciler.py`
  - canonical upsert into `preference:style_contract`
- `brainstack/db.py`
  - one-shot compatibility migration from old corpus-backed style artifacts into the canonical profile lane
- `brainstack/executive_retrieval.py`
  - model-backed `style_contract` route and exact slot targeting
- `brainstack/retrieval.py`
  - explicit ordinary-turn exclusion of `preference:style_contract`
- installer/doctor hardening
  - `auxiliary.flush_memories.provider: main`

## why this is the accepted design

- no regex routing
- no transcript section parser
- no second durable owner
- no ordinary-turn prompt bloat
- principal-scoped user memory remains on the personal-memory axis
