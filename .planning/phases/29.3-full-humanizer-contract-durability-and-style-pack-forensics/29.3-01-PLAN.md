# phase 29.3 plan

## goal

Replace the reverted heuristic 29.3 approach with a clean, non-heuristic design that gives the full Humanizer rule pack one durable, personal-memory owner without bloating ordinary turns.

## scope

### in

- detailed style-contract durable storage model
- Tier-2 structured ingest for the detailed style contract
- explicit style/rules recall routing
- ordinary-turn exclusion rules for the detailed style contract
- one-shot migration from the rejected corpus-backed artifact if it exists

### out

- regex-based style-query routing
- transcript section parsing or transcript reconstruction as a durable path
- corpus-backed long-form style memory as the default final owner for this use case
- prompt-only obedience tuning
- 29 separate durable micro-slots by default

## north star

Brainstack should be able to do both of these cleanly:

- silently apply the compact active communication contract on ordinary turns
- answer explicit questions about the full Humanizer rule pack from durable personal memory

without transcript luck, regex gates, or prompt bloat.

## core principles this phase must preserve

1. donor-first
   - stay inside existing Brainstack seams:
     - Tier-2 structured extraction
     - canonical personal-memory storage
     - retrieval routing

2. modular, thin integration
   - the fix must live in:
     - ingest
     - storage
     - retrieval intent
     - ordinary-turn exclusion
   - do not smear it across unrelated runtime layers

3. no heuristic sprawl
   - no regex style detection
   - no transcript section parser
   - no language-bound special cases

4. single durable owner
   - one canonical owner for the detailed style contract
   - no second owner in corpus, persona, skill, or transcript

5. native-system synergy, not replacement
   - compact contract stays compact
   - the detailed lane is additive for explicit recall only

6. durable only where durable value is real
   - the full Humanizer pack is durable user preference truth
   - meta-discussion about files, loading, or prompt mechanics is not

## lane decision matrix

| memory class | principal-scoped personal truth | needed on ordinary turns | correct lane verdict |
| :--- | :--- | :--- | :--- |
| compact communication rule | yes | yes | canonical compact communication/profile slot |
| detailed Humanizer contract | yes | no | one canonical `preference:style_contract` row |
| graph/entity truth | sometimes | maybe | graph lane |
| persona/skill/file-loading mechanics | no | no | non-authoritative residue, must not own style memory |

Interpretation rules:

- do not move detailed user style truth to corpus by default
- do not let a long-form style row leak into the ordinary prompt
- do not allow transcript history to remain an authoritative fallback once the durable row exists

## architecture decision

### accepted target model

- compact operational contract:
  - existing canonical communication slots
- detailed contract carrier:
  - one principal-scoped `preference:style_contract`
- explicit recall:
  - route mode `style_contract`, resolved through the existing model-backed routing seam
- ordinary-turn behavior:
  - generic profile retrieval and prompt projection must exclude `preference:style_contract` unless explicitly targeted

### rejected model

- regex/direct-query detector for Humanizer questions
- transcript parser that rebuilds the style pack from section headers
- corpus document as the final authoritative owner for this memory class

## file-level execution order

1. `brainstack/tier2_extractor.py`
   - extend structured JSON schema so the model can emit one bounded detailed style-contract artifact directly
   - no transcript reconstruction helper

2. `brainstack/reconciler.py`
   - upsert one canonical `preference:style_contract` row
   - no corpus write for the final path

3. `brainstack/db.py`
   - one-shot compatibility migration:
     - if a prior `style_contract` corpus artifact exists, move its content into canonical `preference:style_contract`
     - retire the old artifact as authoritative owner

4. `brainstack/control_plane.py` and `brainstack/executive_retrieval.py`
   - add a non-heuristic explicit `style_contract` route mode
   - exact-target `preference:style_contract` on that route
   - exclude the long-form style row from ordinary generic retrieval

5. `brainstack/retrieval.py`
   - explicitly keep `preference:style_contract` out of the compact active contract projection

## workstreams

### 1. durable carrier correction

- define the canonical detailed style slot
- prove it is principal-scoped and personal-memory owned

exit gate:

- one detailed style carrier exists, and it is not corpus-backed in the final design

### 2. structured ingest correction

- make Tier-2 emit the detailed contract directly in structured output
- remove transcript-derived reconstruction from the runtime path

exit gate:

- the detailed style contract can be written without transcript heuristics

### 3. explicit recall routing

- route explicit style/rules questions through a model-backed route mode
- exact-target the canonical detailed slot

exit gate:

- explicit recall works without regexes

### 4. ordinary-turn exclusion

- prove that normal prompts and normal retrieval do not leak the long-form style row

exit gate:

- compact prompt remains compact
- ordinary queries do not drag the detailed style contract in

### 5. cleanup gate

- remove every rejected-path artifact from active runtime code:
  - regex style routing
  - transcript section parsing
  - direct style-contract corpus retrieval
  - corpus-backed authoritative style lane

exit gate:

- no dead code remains from the rejected 29.3 design
- no half-wired alternate path remains callable

## validation ladder

1. source tests for structured ingest and exact targeted recall
2. compatibility migration test from old corpus-backed artifact to canonical profile slot
3. live runtime proof that:
   - normal prompt stays compact
   - explicit style query finds the detailed slot
   - ordinary query does not
4. source-of-truth parity and deploy parity check

## anti-goals

- no prompt stuffing
- no transcript-luck pseudo-durability
- no multilingual regex pack
- no dual owner for detailed style truth
- no dead code from the rejected attempt
