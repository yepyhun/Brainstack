# Phase 37 Plan

## goal

Close the newly proven canonical style-contract failure:

- the live Brainstack state can persist a polluted canonical contract
- the polluted contract can include user-speech framing and stale headings
- the compiled hot-path policy can then rebuild itself from that polluted contract
- reset behavior will faithfully recall the wrong canonical truth

This phase should make style-contract persistence:

- bounded
- auditable
- reset-proof
- promotion-safe
- resistant to conversational contamination
- semantically faithful for punctuation and surface-form invariants

The target is not “harder style enforcement”.

The target is:

- a clean canonical contract lane
- a bounded patch lane
- an explicit replace / supersede lane for full user-authored rule-pack replacement
- promotion guards that refuse polluted authority
- and a repair path for principals already carrying bad canonical state

This phase is not allowed to solve the issue by inventing a broader style-intent heuristic layer.

## scope

### in

- canonical capture sanitation:
  - inspect the full style-contract candidate path from user text to canonical commit
  - reject candidate titles, sections, and rule lines that are clearly chat framing rather than contract truth
  - keep exact style-contract recall semantics intact once a clean canonical contract exists
- explicit full-contract replacement:
  - add a first-class replace / supersede path for cases where the user is clearly replacing the previous canonical pack with a new final set
  - stop forcing full-pack replacement through:
    - raw full-parse promotion
    - or narrow small-patch semantics
  - preserve auditable supersession rather than silent overwrite
- fragment discipline:
  - re-evaluate multi-message contract capture so prior user fragments cannot poison the canonical candidate
  - preserve legitimate multi-message rule-pack teaching only when the merged payload is structurally clean
  - stop rewarding longer, more conversationally contaminated candidates just because they contain more text
  - keep multi-message capture bounded to explicit contract-bearing material, not adjacency-based chat synthesis
- patch-lane hardening:
  - keep patch mode bounded to patch-like inputs
  - prevent patch mode from inheriting surrounding dialogue framing
  - prevent patch mode from preserving stale headings or stale count claims when the user explicitly replaced them
- promotion and compiler guards:
  - block compiled policy promotion from obviously polluted canonical contracts
  - require canonical metadata and rendered content to agree on clean title/section/rule identity
  - ensure the hot path cannot be rebuilt from user-utterance-like clause content
- punctuation semantic fidelity:
  - keep distinct invariant meaning for:
    - forbid em dash only
    - forbid all dash-like punctuation
  - prevent compiler and validator logic from collapsing both rules into the same hyphen-repair behavior
- polluted-state recovery:
  - define a safe repair or quarantine path for already-polluted style contracts in live stores
  - ensure recovery is auditable and does not silently destroy exact user-authored rule content
  - keep repair bounded to the style-contract surface, not a broad migration sweep
- proof:
  - prove the exact live incident no longer reproduces
  - prove reset behavior recalls the final cleaned contract rather than stale or conversational state
  - prove multi-message explicit rule teaching still works after sanitization

### out

- broad behavior-policy redesign
- broad retrieval or packet-assembly redesign
- broad operating-truth redesign
- new user-specific marker farms
- language-specific phrase harvesting as the primary style-contract safety mechanism
- replacing canonical style contracts with transcript search
- weakening exact recall or fail-closed honesty to hide the bug
- silent destructive cleanup of live memory without auditable recovery logic

## architecture decision

### accepted target model

- canonical style-contract authority must come only from structurally valid contract content
- conversational framing and rule content must not share the same canonical promotion surface
- multi-message capture may remain, but only after a bounded extraction step that isolates contract content from nearby chat
- structural cleanliness checks may use narrow grammar hygiene, but must not expand into cue-farm intent routing
- explicit full-pack replacement must become a first-class, auditable supersede operation
- patch mode must stay a narrow rule-replacement seam, not a backdoor full-contract writer
- punctuation-oriented rules must preserve their true semantic distinction through:
  - canonical storage
  - compiler projection
  - mechanical repair
- compiled behavior policy must refuse obviously polluted canonical inputs
- a live polluted contract must be:
  - detected
  - prevented from continuing to poison the hot path
  - recoverable through an auditable repair path
- reset behavior should become reliable because the persisted authority is reliable, not because the runtime “works around” bad truth

### rejected model

- allowing full-contract candidates to start with user handles, commands, or turn framing
- treating longer merged candidates as better by default
- fixing polluted canonical truth by layering more prompt-time overrides on top
- relying on keyword deny lists as the primary safety mechanism
- building a new free-form style-intent detector instead of fixing the canonical capture seam
- preserving stale count headings like `27 rules` when the user has explicitly converged to a newer set

## workstreams

### 1. incident freeze and seam audit

- map the exact style-contract path:
  - candidate generation
  - full contract parsing
  - patch parsing
  - canonical commit
  - compiled policy promotion
- freeze the accepted incident model against the live evidence:
  - polluted canonical revision
  - polluted compiled policy title and clauses
  - reset reproducing the bad canonical state
- identify the minimum structural cleanliness checks needed at each seam

### 2. bounded contract extraction

- separate explicit contract extraction from surrounding conversational scaffolding
- ensure multi-message teaching captures only the contract block, not adjacent prompts or debugging text
- remove scoring incentives that favor polluted candidates because they are longer or have more merged fragments
- keep legitimate explicit rule-pack capture working for:
  - single-message full contracts
  - bounded multi-message full contracts
- add a distinct full-replace / supersede path for “this is the new final set, forget the rest” style user intent

### 3. patch-lane hardening and promotion guard

- keep patch mode constrained to short, patch-like rule updates
- prevent patch mode from preserving polluted titles, stale headings, or non-rule lines
- add promotion guards so a canonical contract that still looks like user dialogue cannot become the source of the compiled hot-path policy
- ensure compiler degradation does not quietly normalize polluted clause content into authoritative short forms
- split punctuation semantic handling so “no dash at all” cannot be rewritten into “use hyphen-minus instead”

### 4. polluted-state repair and quarantine

- define a deterministic repair path for canonical style contracts already carrying chat framing or stale headings
- quarantine rather than trust polluted canonical rows where automatic repair is not safe
- keep the repair logic auditable through metadata, source, and revision transitions
- ensure live stores can be healed without broad manual DB surgery

### 5. proof

- add source regressions for:
  - polluted full-candidate rejection
  - bounded multi-message explicit rule teaching
  - explicit full-pack replacement / supersession
  - patch-mode rejection of conversational framing
  - compiler refusal to promote polluted canonical content
  - punctuation semantic fidelity for dash-related rules
  - repair or quarantine of already-polluted contract rows
- add `hermes-final` proof that the installed runtime uses the same sanitized capture and promotion logic
- add rebuilt live runtime proof against the concrete incident shape:
  - explicit 25-rule convergence
  - reset
  - exact 25-rule recall without stale `27` heading bleed-through

## validation ladder

- source:
  - targeted tests for canonical candidate sanitation, patch boundedness, polluted-promotion rejection, and repair/quarantine behavior
  - tests that prove clean multi-message rule teaching still lands one canonical revision
  - tests that prove user-speech framing cannot become the canonical title or hot-path clause text
- `hermes-final`:
  - payload parity plus direct interpreter proof that the installed capture path rejects the old polluted incident shape
  - proof that compiled policy source rows are clean after canonical repair or re-commit
- live runtime:
  - rebuilt runtime smoke showing the incident flow no longer reproduces
  - proof that reset recalls the cleaned final contract
  - proof that no new heuristic farm was introduced to fake the result
