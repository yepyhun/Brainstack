# Phase 15.1 Plan

## Goal

Close the remaining host-side ownership leaks so personal identity/style/project memory stays Brainstack-owned in Brainstack-only mode, while procedural skill workflows remain available.

## Scope

### In

- tighten Brainstack-only host guidance around personal-memory capture
- reduce or block clearly wrong detours from personal preference capture into:
  - `skill_view`
  - `skill_manage`
  - `read_file`
  - `write_file`
- keep reusable procedural skill usage intact
- carry the behavior through the installer and doctor
- prove the behavior on the live Discord runtime

### Out

- no full skill shutdown
- no new memory runtime
- no expansion of Tier-1 heuristics
- no benchmark tuning
- no broad file-tool amputation for unrelated tasks

## Execution Steps

1. Map the exact host surfaces that still nudge the model the wrong way.
   - inspect Brainstack-only guidance, tool exposure, and any remaining prompt/path leaks
   - separate:
     - legitimate procedural skill usage
     - illegitimate personal-memory detours

2. Implement the smallest durable ownership boundary.
   - prefer guidance/routing changes over language-specific content rules
   - keep the fix focused on personal-memory capture in Brainstack-only mode
   - avoid broad tool removal unless a narrower boundary cannot work

3. Keep procedural skills explicitly alive.
   - personal profile/style/identity stays Brainstack-owned
   - reusable procedures/workflows may still use skills
   - the plan must not turn “fixing ownership” into “crippling Hermes”

4. Carry the fix through installer and doctor.
   - no live-only runtime patch
   - installer must reproduce the boundary on fresh Hermes checkouts
   - doctor should verify the carried host behavior where practical

5. Add focused tests.
   - personal preference capture does not route into generic notes/skill detours
   - procedural skill flows still work
   - the fix remains valid in Brainstack-only mode
   - no regression in existing host-memory displacement behavior

6. Reinstall into live Bestie and run bounded Discord smoke.
   - clean reset if needed
   - verify that personal style capture stays inside Brainstack behavior
   - verify no notes-file or skill detour appears for personal preferences
   - verify procedural skill usage still works when actually requested

## Design Rules

- ownership first
- routing first
- minimal complexity
- multilingual-safe where possible
- no new heuristic sprawl
- no benchmark-chasing

## Exit Gate

- in Brainstack-only mode, personal preference capture no longer detours into generic note/file flows
- personal style/profile capture no longer detours into skill inspection/mutation flows
- procedural skills still function for real workflow reuse
- the fix survives reinstall through the Brainstack integration kit
- live Discord smoke confirms the new ownership behavior without creating a larger or more fragile host design

## Recommended Effort

- `high`
