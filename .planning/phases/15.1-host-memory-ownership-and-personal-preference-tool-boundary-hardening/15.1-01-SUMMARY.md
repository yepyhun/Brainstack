# Phase 15.1 Summary

## Outcome

Phase 15.1 closed the remaining host-side ownership leak where Brainstack-only mode could still drift personal preference capture into Hermes side-memory files or adjacent host tool flows.

The fix stayed bounded:

- no skill shutdown
- no new runtime
- no new Tier-1 heuristics
- no benchmark-driven rewrite

Instead, the phase hardened the existing host boundary in two places:

1. Brainstack-only mode now blocks file-tool detours into Hermes side-memory locations.
   - `read_file`, `write_file`, and `patch` are rejected when they target:
     - `~/.hermes/notes/...`
     - Hermes-root `MEMORY.md`
     - Hermes-root `USER.md`

2. `run_agent.py` now injects explicit ownership guidance in Brainstack-only mode.
   - personal identity, preferences, communication style, and project context belong to Brainstack
   - notes files and skill records must not be used for that class of memory
   - `skill_manage` remains available for reusable procedural workflows
