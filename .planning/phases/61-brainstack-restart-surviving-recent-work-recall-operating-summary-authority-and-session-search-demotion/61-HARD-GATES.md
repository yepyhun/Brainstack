## 1. Memory-kernel gate

- the closeout explicitly states that the fix was made in Brainstack recent-work recall
- the closeout does not present `session_search` tuning as the main solution
- the final answer names what Brainstack now remembers across restart that it failed to remember before

## 2. Restart recap gate

- after restart or auto-reset, a broad recap question about immediately preceding work produces a non-empty Brainstack recall packet
- this proof must not rely on exact transcript search first
- this proof must not be tied to one exact sentence

## 3. Durable operating-truth gate

- major recent work is represented in durable Brainstack operating truth, not only in continuity/transcript residue
- the proof distinguishes:
  - active work
  - completed work
  - discarded/superseded work when relevant

## 4. Cross-session gate

- the recall path does not silently fail just because the relevant work lives in a previous session id
- the phase explains how principal scope and cross-session retrieval are handled for recap

## 4.1 Route-separation gate

- recap recall is implemented as a distinct route from `task_like`
- the final design does not blur obligation/task lookup with "what were we doing / what landed" recap recall

## 5. No-transcript-dump gate

- the fix does not solve recap by projecting large raw transcript blocks into ordinary memory
- token discipline remains explicit
- the closeout states the bounded packet consequences

## 6. No-sebtapasz gate

- there is no exact-phrase hack for `phase 60`
