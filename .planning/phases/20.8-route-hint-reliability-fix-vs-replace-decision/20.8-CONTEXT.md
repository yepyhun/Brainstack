## What This Phase Is

This phase is a **narrow route-hint reliability phase**.

It exists to answer one bounded question cleanly:

- can the current LLM route-hint path be made reliably machine-usable
- or should it be replaced with a simpler deterministic classifier

The phase starts with raw-output audit, not with more blind prompt tweaking.

## What This Phase Is Not

This phase is **not**:

- a broad fact-path parity phase
- a profile / identity retrieval recovery phase
- a graph-yield phase
- a new proof-calibration phase
- a decomposition comeback phase
- a multilingual keyword-table routing phase
- a benchmark-gaming phase

If the route-hint path cannot be made reliable here, the phase must say so plainly instead of smearing the problem across unrelated retrieval work.
