# Phase 20.8 Summary

## Verdict
- `replace` won over `fix`
- the old LLM route-hint path stayed unreliable for two independent reasons:
  - historical raw outputs were explanatory prose instead of machine-usable route tokens
  - the current auxiliary-client LLM route-hint path still hit `401` in the target runtime path
- the bounded deterministic route-hint classifier is now the chosen `20.8` implementation path

## What Changed
- route-hint default behavior now uses the bounded deterministic classifier instead of the old LLM route-hint call
- the benchmark runner no longer forces direct route-resolver injection by default
- a dedicated route-hint audit script was added to compare deterministic behavior against the old LLM path
- punctuation-robust cue matching was added so route cues like `in total?` and `first,` are not silently missed

## Validation
- local validation: `39 passed`
- target runtime payload reinstalled into Bestie before live reruns

## Live Comparable Results
- answer-only: `2 / 15`
- split raw answer: `5 / 15`
- split `retrieval_correct`: `3 / 15`
- split `both_correct`: `3 / 15`

## Route Activation Result
- route-hint reliability is no longer the main blocker
- split route-source distribution:
  - `deterministic_route_hint`: `14 / 15`
  - `default`: `1 / 15`
- split requested/applied mode distribution:
  - `fact`: `7`
  - `temporal`: `6`
  - `aggregate`: `2`

## Channel Yield Result
- route activation and channel yield are now explicitly separated
- activated `temporal` routes: `6`
  - temporal channel nonzero: `2`
  - temporal channel zero: `4`
- activated `aggregate` routes: `2`
  - graph channel nonzero: `1`
  - graph channel zero: `1`

## Interpretation
- `20.8` succeeded at its narrow purpose:
  - route-hint reliability was restored by replacing the old LLM route-hint path
  - structural routing is live again in the comparable benchmark path
- `20.8` did **not** improve live benchmark quality
  - answer-only fell to `2 / 15`
  - split remained weak at `5 / 15`
- this means the dominant blocker is now downstream retrieval quality, not route activation

## Proven vs Not Claimed
### Proven
- the old LLM route-hint path should not remain the default path
- the deterministic replacement activates non-fact routes live
- temporal / aggregate route activation alone does not rescue answer quality
- temporal-channel yield is still absent on most activated temporal questions

### Not Claimed
- this phase does not prove that deterministic routing is globally optimal forever
- this phase does not prove that graph yield is sufficient
- this phase does not restore fact-path parity

## Carry Forward
- `20.9` must now absorb the real post-routing blockers:
  - fact-path parity restoration
  - profile / identity retrieval recovery
  - temporal-channel candidate-yield recovery
  - candidate quality / evidence packing recovery on newly activated temporal and aggregate routes
  - deterministic classifier misroute audit must be explicit inside the route-policy / temporal-yield work
    - false-positive temporal routing and false-negative temporal routing must both be named
    - yield numbers must not be treated as meaningful until the misrouted cases are re-audited
  - if the deterministic gate hurts more cases than it helps, `20.9` must keep bounded multi-channel fusion / soft routing open as the explicit fallback policy alternative
  - aggregate counting remains an opportunity-cost warning, not a hard numerical ceiling
    - it may stay low-yield under the current architecture
    - but it must not dominate the phase mission or turn into a fake score target
