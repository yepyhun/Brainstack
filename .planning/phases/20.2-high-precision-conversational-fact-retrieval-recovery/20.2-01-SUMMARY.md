# Phase 20.2 Summary

## Outcome

Phase `20.2` executed the first **high-precision conversational fact retrieval** correction pass after the `9 / 15` post-`20.1` rerun.

This phase did **not** reopen the donor-first architecture.
It tightened the live retrieval path on the exact-fact failure class:

- transcript and continuity evidence now trims on cleaner boundaries instead of chopping answer-bearing tails
- conversational ranking now favors user-led, exact-value, answer-bearing turns more strongly
- the working-memory evidence rule now explicitly prefers concrete user facts over assistant suggestions or generic world priors
- fresher corrected numeric values can now auto-supersede stale exact values on the existing graph-state path
- bounded query decomposition now exists for true multi-entity questions, without turning into a second universal retrieval system

## Result

Phase `20.2` produced several real local fixes without reopening architecture work, but as a bundled change set it did **not** hold up as a verified net win on the real-path rerun:

- post-`20.1`: `9 / 15`
- post-`20.2`: `7 / 15`

What the bundle still demonstrably improved:

- packing fidelity is cleaner and more answer-preserving
- result / exact-value turns are easier to surface than nearby planning/setup turns
- user-authored facts are now favored more explicitly over assistant/generic priors
- the stale-vs-fresher numeric-value path is stronger on the existing graph route
- bounded multi-entity decomposition now exists, but stays sparse and capped

What this phase summary now records explicitly:

- `20.2` is not a completed success verdict
- the benchmark remains a diagnostic instrument, not the product goal
- the product goal is still a more accurate everyday kernel, not benchmark-maxing
- the immediate next step is no longer verify
- the immediate next step is pure split-pass regression forensics

What this phase does **not** yet claim:

- final-boss restoration
- `14 / 15`
- that decomposition alone solves the remaining benchmark gap
- that ranking bonuses and grounding overlays are all equally justified

- removing `digit` + quote bonuses together is catastrophically harmful in the current decomposition-off configuration

- whether the quote bonus is also materially important
- whether either bonus should survive unchanged long-term
- whether the stronger `authoritative` wording is itself the primary root cause of the collapse

Updated routing implication:

- do **not** remove `digit` + quote together again
- restore the decomposition-off baseline as the active forensic baseline
- split the bonuses apart next:
  1. quote-off only
  2. digit-off only
- only after that should the `user-led` weight be reconsidered
- keep the stronger-grounding question open as a hypothesis until a future wording-only A/B exists
