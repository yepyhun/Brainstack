# Phase 75 Plan

## purpose

Add bounded associative graph expansion and activation-style ranking on top of Phase 67/69 semantic seeds so Brainstack can find related memory through relations, aliases, context ids, and multi-hop evidence without opaque graph magic.

## core question

How can Brainstack adopt neural-memory/CerebroCortex-style associative recall while keeping expansion bounded, authority-aware, traceable, and evidence-preserving?

## GSD planning enforcement check

This phase must preserve:

- no heuristic sprawl: no graph-side phrase farms
- truth-first: distinguish similarity from authoritative truth
- inspectability: every expansion hop must be traceable
- product quality: improve long-range relation tracking without drowning current truth
- token priority: associative volume cannot overwhelm packet budgets

## shared north star and real-world proof gate

North star for Phase 70-77: Brainstack must become easier and safer for Hermes agents to use as a memory kernel without expanding Brainstack into runtime execution, workflow governance, or fake autonomy.

Every phase closeout must prove one of these:

- Hermes can use Brainstack more reliably in a real memory workflow
- or the phase is prerequisite work and proves no regression while naming the exact later phase that must deliver the user-visible improvement

Minimum proof shape:

- targeted tests for the changed surface
- at least one end-to-end Hermes/Brainstack interaction path or Brainstack packet/inspect proof using real provider/store code
- evidence ID/source/provenance proof where memory is involved, not only generated answer text
- no-regression check against Phase 66 golden gates once they exist
- bounded token/output sanity for any new tool, MCP, inspect, corpus, or trace surface

## architecture decisions to freeze before execution

### 0. conditional start gate

This phase must not start as behavior-changing implementation until Phase 67 and Phase 69 have proven:

- typed semantic seed availability
- graph recall mode honesty
- false-positive controls in Phase 66 golden cases
- inspect traces that can explain seed, hop, score, suppression, and final inclusion

If those prerequisites are not met, Phase 75 may only define contracts and read-only inspect traces. It must not feed associative candidates into final packet selection.

### 1. expansion bounds

Every run must define:

- max seed count
- max depth
- max candidate count
- max latency budget
- allowed shelves/channels

### 2. authority rule

Associative similarity cannot outrank authoritative current profile/operating/task truth unless explicit policy allows it.

### 3. trace rule

Each candidate must explain seed id, relation/hop path, score contribution, and suppression reason when dropped.

## phase-specific real-world proof examples

Closeout must include at least:

- relation or alias recall proof where associative expansion finds relevant durable evidence that direct lexical recall would miss
- false-positive proof where a superficially related memory is suppressed with an inspectable reason
- authority proof where current profile/operating/task truth outranks associative similarity volume
- disabled-inclusion proof if Phase 67/69 prerequisites are not met

## acceptance criteria

- related memory can be found through bounded relation/context expansion
- every expansion is traceable
- associative results do not drown authoritative truth
- unsupported relation recalls fail honestly
- behavior-changing packet inclusion remains disabled unless Phase 67/69 prerequisites and false-positive gates pass
