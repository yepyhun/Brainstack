# Phase 20.20 Context

## Why this phase exists

`20.19` established that the deployed Bestie runtime is no longer blocked primarily by missing graph/corpus dependencies:

- Bestie checkout `.venv` now imports `kuzu` and `chromadb`
- activation proof showed Brainstack can instantiate:
  - `KuzuGraphBackend`
  - `ChromaCorpusBackend`
- the enabled-stack isolated live baseline stayed at `3 / 5`
- `aggregate_total` passed, but the enabled-stack debug still showed:
  - active `semantic` and `graph` channels
  - yet `graph_rows = 0`
  - and `corpus_rows = 0`
  - while final packets remained transcript/continuity-dominant

So the main residual is no longer “backend unavailable”. It is now:

- enabled-stack graph/corpus -> final packet bridge
- native aggregate row surfacing on the live path
- enabled-stack live stability / variance

## Current product truth

- the enabled isolated live runtime can now truthfully report:
  - `semantic` active
  - `graph` active
- but the final packet is still usually dominated by transcript/continuity evidence
- the aggregate path already has a bounded native-aggregate prepend into the `matched` block; this phase is **not** inventing the first bypass slot from scratch
- `graph_rows = 0` by itself does **not** prove that native graph evidence was absent, because native aggregate rows currently surface through `matched`, not through the separate `graph_rows` block
- `aggregate_total` can pass on the enabled stack without proving that a native graph aggregate row actually won the surfaced packet
- live answer stability is not yet strong enough to call the enabled stack “settled”

## What this phase is not

- not broader aggregate planner expansion
- not another degraded-stack keyword/temporal micro-tuning pass
- not graph/corpus vendor experimentation
- not checkout-wide packaging or deployment system redesign
- not benchmark-shaped tuning

## Honest target

Make the enabled graph/corpus stack materially visible in the final live packet, prove that native aggregate evidence can actually surface on the live path, then re-baseline isolated live behavior again so the next phase is driven by enabled-stack truth rather than by dependency or fallback ambiguity.
