## workstreams

1. **Authority bootstrap after wipe**
   - ensure the first explicit style teaching yields one clean canonical revision
   - ensure the compiled behavior policy is present from the same authority lineage
   - if active style authority exists without compiled policy, auto-repair or fail closed

2. **Final-output enforcement convergence**
   - close the gap where typed invariants can be known and violated at the same time
   - no ordinary final output should silently ship when active invariants remain unenforced

3. **Owner-first implicit style recall**
   - make natural Hungarian and English rule questions route to style authority
   - keep transcript/session recall as supporting evidence, not the canonical owner

4. **Transcript persistence correctness**
   - repair the host-side transcript write path so continuity and recall do not sit on a broken session record seam

5. **User-surface hygiene**
   - keep operator traces, tool-call chatter, and internal interruption noise out of the user-facing chat stream

6. **Reminder timezone correctness**
   - make scheduled reminders align with intended user/local time semantics

7. **Integrated live proof**
   - prove the whole path on a wiped store:
     - clean rule teaching
     - clean rule recall
     - compliant ordinary reply
     - natural-language style recall
     - transcript persistence
     - no tool-trace leak
     - correct reminder timing semantics
