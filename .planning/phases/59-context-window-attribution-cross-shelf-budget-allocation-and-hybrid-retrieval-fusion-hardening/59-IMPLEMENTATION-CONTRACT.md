# Phase 59 Implementation Contract

## invariant

Phase 59 must improve Brainstack-owned token discipline and retrieval quality without cheating the attribution story.

The phase succeeds only if it can say two true things at once:

1. how much of context pressure belongs to Brainstack versus the wider Hermes stack
2. what Brainstack itself changed to become more bounded and more relevant

## required properties

- attribution comes before optimization
- Brainstack changes stay inside the source-of-truth repo
- allocation improvements are global enough to matter across shelves
- fusion improvements are general enough to matter across routes
- no quality claim is made from token counts alone
- no token claim is made from one synthetic benchmark alone

## prohibited outcomes

- declaring “Brainstack fixed context bloat” without separating host and provider contributions
- reducing context pressure by simply cutting evidence with no ranking improvement
- inflating retrieval complexity with query-specific heuristic farms
- smuggling a backend replacement into the phase and calling that the main fix
- regressing recall fidelity while claiming success from lower token usage
- solving product complaints with phase-local benchmark tuning

## proof expectation

Proof must include:

- whole-stack attribution evidence
- Brainstack-specific before/after packet evidence
- retrieval-fusion evidence
- install proof on `finafina`
- targeted regression coverage for retrieval and bounded-packet behavior

## output required

- a measured attribution note
- a clear allocator design summary
- a clear fusion-design summary
- a final result that distinguishes:
  - whole-stack context pressure
  - Brainstack-owned contribution
  - Brainstack-owned improvement

## anti-goals

- no new feature lane
- no donor churn
- no benchmark heroics
- no fake economy from deleting useful evidence
