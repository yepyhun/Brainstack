# Phase 35 Context

## problem statement

The current Brainstack is stronger at several integrity-heavy truth classes than before:

- behavior contract storage
- revision / supersession
- scoped personal continuity
- typed graph evidence boundaries

But the broader second-brain product still feels unbalanced.

The most useful sharpened reading is:

- Brainstack is becoming stronger at `how should I answer` truth
- than at `what is happening in the user's world and work right now` truth

That imbalance matters more than a single rule-regression.

The operating kernel is therefore lagging the response-oriented kernel:

- behavior and response truth already have stronger storage, projection, and enforcement surfaces
- operating truth is still too often reconstructed from continuity summaries, thin profile slices, and query-shaped routing choices
- the current system risks becoming a better rule keeper than a better second brain

## why this is the correct next phase

Phase `34` is about restoring synergy by reducing overreaching ordinary-turn behavior governance.

This phase is the natural follow-on:

- once Brainstack stops trying to over-govern the turn
- it needs to get stronger at what a second brain should really contribute

That means a better operating kernel:

- active work
- open decision
- current commitment
- next step
- external owner pointer

## accepted sharpened reading

- operating context is still too derived and session-summary-shaped
- retrieval is still too cue-first in places where owner-first logic would be more truthful
- truth classes remain too siloed
- the typed evidence direction is good, but still too narrow to support a strong operating substrate
- external owner pointers exist as a conceptual lane but are not yet carrying enough first-class truth to matter
- active work and open decisions are still too easy to confuse with continuity prose instead of durable operating records

This is not a call for a huge ontology or planner. It is a call for a small first-class operating-truth layer.

## phase boundary

This phase is about:

- operating truth
- owner-first retrieval
- operating-context maturation
- narrow typed evidence expansion for operating claims
- reducing the maturity gap between response truth and world/work truth

This phase is not about:

- more behavior-only work
- broad graph redesign
- broad multimodal redesign
- benchmark chasing
- building a pseudo-project-manager that exceeds Brainstack’s bounded kernel role

## expected proof shape

The proof should show that Brainstack gets better at present-world and present-work continuity:

- less fake forgetfulness about current work state
- better surfacing of active work / open decisions / commitments / next steps
- less dependence on the exact wording of the user query
- better operating-context rendering from first-class truth instead of mostly derived summary
- more explicit handling of external ownership instead of silent omission
