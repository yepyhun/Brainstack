# Phase 34 Context

## problem statement

The current Brainstack shape is stronger than before at durable truth, behavior-contract storage, correction capture, and scoped personal continuity. But the newest live conversation evidence surfaced a different class of problem:

- Brainstack can improve memory correctness and still make the assistant feel worse
- the issue is no longer only storage drift or recall failure
- the issue is that Brainstack can overstep from continuity support into ordinary-turn behavior governance

In practice this can look like:

- the model juggling live chat context plus recalled contract fragments plus injected behavior rules plus output repair logic
- recalled behavior material conflicting with what is already clearly present in the live conversation window
- the system feeling less fluid, less natural, or more self-contradictory even when some memory surfaces are technically “working”
- Brainstack paying the cost of strong authority framing even when the actual enforcement chain is only partial, budgeted, or validator-narrow
- partial or truncated behavior packets outranking richer live conversational evidence simply because they are rendered as internal contract blocks

That is a kernel-shape problem, not just another bug.

## why this is the correct next phase

Phase `33` improved genuine second-brain quality:

- personal-scope fallback
- short explicit correction capture
- better personal/preference retrieval depth
- tighter canonical truth paths

But `33` intentionally did not decide the broader product question:

- how much ordinary-turn behavioral control Brainstack should exert
- whether the current memory-context authority framing is too strong
- whether large behavior contracts belong in routine turn governance at all

This phase takes that next step directly.

It also sharpens one important product truth:

- the issue is not “any injection is bad”
- the issue is that the current hot path mixes:
  - durable factual memory
  - exact canonical behavior truth
  - every-turn behavior steering
- those need different authority and different budgets

## accepted sharpened reading

- the user may be partly right, but not absolutely right
- Brainstack is not “bad” simply because it influences the LLM
- Brainstack is overextended when it tries to govern too much of ordinary-turn behavior
- the fix is not to remove second-brain capabilities
- the fix is to restore a cleaner role split:
  - Brainstack owns durable truth and continuity
  - the LLM owns most local conversational behavior
  - only a small set of pinned invariants should routinely survive into ordinary-turn steering
- the pinned lane must be smaller than the canonical contract:
  - exact contract truth remains recallable
  - only a strict hard-rule subset belongs on the always-on hot path
- authority should correlate with guarantee:
  - strong factual truth can stay authoritative
  - partial behavior projection must stop sounding stronger than it really is

## phase boundary

This phase is about:

- role split
- authority tuning
- injection de-escalation
- synergy recovery
- narrowing the gap between Brainstack’s authority wording and its real enforcement surface

This phase is not about:

- broad graph redesign
- auth/runtime firefighting
- benchmark chasing
- deleting behavior memory entirely
- turning exact canonical behavior truth into vague chat-local hints

## expected proof shape

The expected proof is not benchmark-centered. It should show:

- smaller and more selective ordinary-turn memory injection
- preserved factual continuity and durable recall
- reduced conflict between live chat context and Brainstack guidance
- better natural-conversation behavior without loss of second-brain value
- explicit evidence that the hot path no longer acts like a response harness while exact contract recall still survives on demand
