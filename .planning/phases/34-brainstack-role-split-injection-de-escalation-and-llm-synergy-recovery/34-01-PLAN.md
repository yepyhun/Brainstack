# Phase 34 Plan

## goal

Re-balance Brainstack so it acts as a strong second-brain / continuity kernel for the LLM instead of a broad ordinary-turn behavior governor. The target is better synergy:

- Brainstack supplies durable truth, long-range continuity, commitments, and explicit high-value invariants
- the LLM keeps primary responsibility for local phrasing, short-range conversational flow, and immediate social reasoning
- the hot path stops paying large prompt and rigidity costs for behavior material that is not actually guaranteed or mechanically enforced
- the system stops presenting partial or budget-truncated behavior policy as if it had the same authority as durable factual memory

## scope

### in

- role-split clarification:
  - explicitly distinguish Brainstack responsibilities from LLM responsibilities
  - identify which current ordinary-turn memory injections are truly second-brain support and which are overreaching behavior control
- injection de-escalation:
  - reduce routine ordinary-turn behavior steering where it overloads or conflicts with live conversation context
  - preserve only a narrow, justified set of pinned ordinary-turn invariants
  - explicitly separate:
    - always-on hard invariants that are objectively checkable
    - exact archival behavior-contract recall
    - optional behavior guidance that should only surface when directly relevant
- behavior-memory split:
  - keep archival / exact behavior contract recall as a first-class lane
  - stop treating large behavior contracts as if they should routinely govern every ordinary turn
  - stop paying every-reply hot-path cost for large behavior contracts that mostly act as prompt pressure rather than durable second-brain help
- memory-context authority tuning:
  - rework authority framing so Brainstack supports the model’s live context rather than constantly competing with it
  - preserve truth-first handling for durable factual continuity without turning every recalled item into local-turn governance
  - specifically de-escalate authority wording where the current system says behavior material is non-optional even when the projection is budget-limited or only partially enforced
- pinned invariant lane design:
  - define the maximum acceptable always-on invariant set
  - require that every retained invariant be both product-valuable and mechanically supportable
  - exclude broad style, persona, and high-variance social behavior from the always-on lane
- retrieval and rendering role cleanup:
  - ensure personal/preference turns do not get solved by stripping evidence while keeping oversized behavior governance
  - make archival contract recall and ordinary-turn support visibly different lanes with different authority
- proof:
  - verify that the new shape improves conversational synergy without regressing durable second-brain capability
  - prove that Brainstack no longer wins authority simply by being injected more aggressively than the live conversation evidence

### out

- broad KG redesign
- broad multimodal envelope redesign
- unrelated auth/runtime cleanup
- benchmark-shaped tuning or special-casing
- removal of Brainstack’s durable truth / continuity role
- replacing exact canonical behavior truth with vague transcript search
- widening the always-on invariant lane by smuggling more style rules into it

## core principles this phase must preserve

- donor-first
- modularity / upstream updateability
- fail-closed upstream compatibility
- no benchmaxing
- truth-first / no "good enough"
- zero heuristic sprawl
- multimodal-first architecture

## architecture decision

### accepted target model

- Brainstack remains a memory / continuity kernel, not a replacement reasoning layer
- archival behavior memory and ordinary-turn behavior steering are treated as different products with different budgets and authority
- ordinary-turn behavior guidance is narrowed to a small justified invariant lane rather than a broad style-governance payload
- factual personal truth, commitments, task truth, identity, and graph continuity remain first-class Brainstack strengths
- local conversation context retains primacy for immediate turn-by-turn phrasing and interaction flow
- any behavior truth that is not retained in the narrow invariant lane remains canonical and recallable, but not automatically every-turn governance
- the system must prefer a smaller truthful always-on lane over a larger pseudo-authoritative lane that over-claims what is actually enforced
- the phase must be willing to prove the user only partially right:
  - keep what is genuinely synergistic
  - cut back what acts like a harness

### rejected model

- making Brainstack “smarter” by injecting even more behavior contract into every turn
- assuming all obedience failures are solved by stronger prompt governance
- replacing second-brain continuity with prompt-only chat-local memory
- building a shadow assistant-policy engine beside the LLM
- keeping strong authority framing while relying on budgeted projections and narrow final-output repair as the real enforcement surface
- solving synergy complaints by deleting behavior truth instead of narrowing its hot-path role

## workstreams

### 1. role and authority audit

- map the current ordinary-turn memory-context and behavior-policy seams
- classify each injected behavior/control surface as:
  - second-brain support
  - narrow invariant support
  - overreaching behavior governance
- freeze the accepted role split before implementation spreads
- explicitly inventory:
  - retrieval-rendered authority blocks
  - behavior-policy projection surfaces
  - output-validator enforced surfaces
  - host/runtime wording that upgrades recalled memory into quasi-system instructions

### 2. ordinary-turn de-escalation

- reduce or restructure intrusive behavior injection on ordinary turns
- keep only a narrow set of pinned invariants that are worth enforcing every turn
- ensure larger style / persona / communication contracts move to archival or explicit recall lanes instead of always-on governance
- define acceptance criteria for the pinned lane:
  - small enough to fit without budget games
  - objective enough to validate mechanically
  - valuable enough to justify every-turn cost
- treat partial or omitted contract projection as an argument for de-escalation, not for stronger framing

### 3. authority framing cleanup

- tighten the wording and structure of memory-context authority so durable truth remains useful without overshadowing live chat context
- preserve factual authority where needed
- de-emphasize or remove phrasing that turns recalled behavior material into quasi-system-prompt governance by default
- make sure factual truth and behavior guidance are not framed with the same strength when their guarantees differ
- reduce cases where a truncated or stale behavior packet outranks the richer live conversation window in practice

### 4. synergy-focused retrieval and rendering

- ensure the retrieval/rendering pipeline helps the model recover durable truth and continuity without flooding the turn with redundant or conflicting behavior payloads
- prefer smaller, more selective, more role-appropriate render surfaces
- avoid reintroducing starvation while still reducing over-injection
- explicitly avoid the bad tradeoff:
  - less transcript / graph / corpus evidence
  - more prepackaged behavior steering
- verify that personal and preference turns keep enough real evidence to support the LLM’s own reasoning instead of forcing reliance on canned governance blocks

### 5. proof

- add proof that compares the old overbearing behavior-governance shape against the new role-split shape on natural conversational flows
- validate on source, `hermes-final`, and rebuilt live runtime
- include natural-chat proofs where the expected improvement is:
  - less self-contradiction
  - less rule-recitation pressure on ordinary turns
  - fewer “Brainstack says X, live chat says Y” collisions
  - preserved exact recall when the user explicitly asks for the canonical contract

## validation ladder

- source:
  - targeted regression tests for role split, de-escalated ordinary-turn injection, and preserved second-brain truth continuity
  - tests for pinned invariant lane size and ordinary-turn payload narrowing
  - tests that archival behavior recall still returns exact canonical truth on demand
- `hermes-final`:
  - payload parity plus direct interpreter proof that the new injection shape is installed and active
  - proof that host/runtime framing no longer upgrades partial behavior payloads into stronger authority than intended
- live runtime:
  - rebuilt runtime smoke showing the model remains grounded in durable Brainstack truth without the previous “harness” effect
  - natural conversational smoke showing improved fluidity without loss of factual continuity

## anti-goals

- do not turn this into “delete behavior memory”
- do not solve the problem by ad hoc cue farms or narrow transcript-specific hacks
- do not regress the durable second-brain capabilities that actually justify Brainstack
- do not keep large always-on behavior payloads just because they are now stored more cleanly
- do not treat “more obedience wiring” as synonymous with “better synergy”
