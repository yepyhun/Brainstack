# Phase 33 Plan

## goal

Strengthen Brainstack as a general second-brain / continuous agent after the `32` hotfix by hardening personal scope continuity, making short explicit user corrections patch canonical truth, narrowing the gap between canonical personal truth and ordinary-turn behavior, and stopping self-starvation on personal/preference recall.

## scope

### in

- personal-scope hardening:
  - reduce brittleness from exact `agent_workspace` / adjacent runtime identity matching for personal truth
  - add a cleaner personal-scope fallback or lookup model where appropriate
- short correction -> canonical patch capture:
  - allow brief explicit user corrections to become durable canonical patches
  - do this without requiring a full structured rule-pack resend
- canonical truth -> ordinary-turn obedience convergence:
  - reduce the gap between “stored correctly” and “behaves accordingly”
  - focus on general personal-truth obedience, not a one-off ruleset rescue
- personal/preference retrieval depth:
  - reduce starvation in preference / self-model / personal-memory routes
  - keep enough transcript / graph / corpus evidence available for truthful recall

### out

- broad KG redesign
- broad multimodal envelope redesign
- benchmark-shaped special casing
- unrelated provider/auth/runtime cleanup

## core principles this phase must preserve

- donor-first
- modularity / upstream updateability
- fail-closed upstream compatibility
- no benchmaxing
- truth-first / no "good enough"
- zero heuristic sprawl
- multimodal-first architecture

## architecture decision

### accepted target model

- `33` is a general second-brain quality phase, not a rescue patch for one transcript
- personal truth must remain first-class and authoritative across more than one exact runtime surface
- short explicit user corrections need a bounded canonical patch path
- ordinary-turn behavior should converge toward authoritative personal truth without relying on fragile one-off prompts
- preference/personal routing must stay truthful without starving itself into fake forgetfulness

### rejected model

- special-case logic keyed to the `29`-rule story
- new marker/cue farms for narrow recall wins
- duplicate owners or shadow personal-memory systems
- bundling unrelated graph/runtime architecture work into this phase

## workstreams

### 1. personal-scope hardening

- inspect personal-truth lookup seams
- decide and implement the narrowest correct fallback / scope-stability model
- preserve donor-first storage ownership while reducing scope brittleness

### 2. short correction patch capture

- define what counts as an explicit short correction worthy of canonical patching
- add a bounded durable patch path for those corrections
- keep ambiguous chat chatter out of canonical mutation

### 3. obedience convergence

- inspect where canonical personal truth is still lost between storage and ordinary-turn behavior
- strengthen that bridge in a general way
- prefer authoritative convergence over special-case enforcement sprawl

### 4. personal retrieval depth

- inspect current starvation in personal/preference/self-model routing
- rebalance evidence budgets so the system can still prove continuity
- keep token discipline while avoiding fake “I forgot” outcomes

### 5. proof

- add regression proof that exercises the four workstreams together
- validate on source, `hermes-final`, and rebuilt live runtime

## validation ladder

- source:
  - targeted unit/integration tests for scope fallback, correction patching, obedience convergence seams, and retrieval-depth routing
- `hermes-final`:
  - parity proof on the installed payload and relevant host seams
- live runtime:
  - rebuilt runtime smoke proving that personal truth survives the expected session/runtime transitions and still influences ordinary-turn behavior
