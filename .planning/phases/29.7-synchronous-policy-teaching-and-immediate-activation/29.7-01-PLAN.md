# Phase 29.7 Plan

## goal

Make explicit policy teaching synchronous, durable, and immediately active in the very next reply.

## scope

### in

- explicit policy-teaching detection boundary
- synchronous parse / persist / compile path
- immediate activation before next answer

### out

- passive implicit preference mining from ordinary chat
- broader second-brain proactive state

## core principles this phase must preserve

1. donor-first
2. modular, thin integration
3. zero heuristic sprawl
4. multimodal-ready
5. token discipline
6. single durable owner per truth class
7. second-brain / proactive-agent fit
8. no dead code / no half-wired paths

## architecture decision

### accepted target model

- explicit policy teaching gets a dedicated, synchronous path
- Tier-2 remains enrichment and background recovery, not first owner for explicit rules
- successful explicit teaching updates both the raw archival contract and the compiled always-on policy before the next assistant answer
- activation is transactional and fail-closed

### rejected model

- waiting for idle window / batch accumulation on explicit rule teaching
- treating every preference-ish sentence as synchronous policy teaching

## workstreams

### 1. teaching boundary

- define the strict product boundary for explicit policy teaching

### 2. synchronous persistence

- persist the archival contract immediately
- compile the always-on policy immediately
- validate the compiled policy immediately
- if activation cannot complete cleanly, keep the previous active policy in force

### 3. immediate activation

- ensure the next reply uses the new compiled policy
- ensure ordinary-turn obedience changes immediately, not only explicit recall answers

### 4. fallback discipline

- keep Tier-2 as a backstop, not the primary owner
- surface a named gap on failed explicit teaching instead of claiming success

## validation ladder

1. explicit teaching path proof
2. immediate next-turn activation proof
3. no-regression proof for non-policy chat
4. fail-closed activation proof
5. source-vs-shim-vs-final verify-surface proof

## anti-goals

- no vague intent guessing
- no hidden file or skill persistence

## recommended model level

`xhigh`
