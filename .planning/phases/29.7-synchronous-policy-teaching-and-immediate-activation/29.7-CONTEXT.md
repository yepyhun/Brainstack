# Phase 29.7 Context

## problem statement

The current plain-chat path still treats explicit behavior teaching too indirectly.

If the user says "these are my rules, speak like this from now on", the system should not wait on delayed background enrichment before activating the new policy.

## why this phase exists

Even with a correct raw-vs-compiled policy split, the product will still feel broken if explicit teaching does not become active before the next answer.

This phase makes explicit behavior teaching a first-class synchronous write-through path.

## accepted evidence entering this phase

- Tier-2 defaults are delayed
- explicit `on_memory_write(target=\"user\")` is stronger than plain chat today
- the user experience expects explicit policy teaching to be immediate

## explicit non-goals

- no freeform broad Tier-1 semantic guessing engine
- no policy extraction from vague small talk
- no bypass around Brainstack ownership
## recommended model level

`xhigh`
