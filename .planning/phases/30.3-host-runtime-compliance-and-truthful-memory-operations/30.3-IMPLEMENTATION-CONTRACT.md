# Phase 30.3 Implementation Contract

## Required invariants

- the assistant may not claim durable success without commit evidence
- objective hard constraints may not rely only on model self-control
- reset/teardown may not silently discard explicit durable writes
- “not found” semantics must not overclaim what the lookup actually proved
- lookup routing for task-like asks must prefer structured truth lookups over transcript/session search fallbacks

## Minimum acceptable outcomes

- a hard-surface validator exists for the agreed mechanical constraints
- explicit durable writes produce receipts that gate success language
- reset/teardown has a durable write barrier for the agreed write classes
- lookup misses explain whether they came from:
  - no committed record
  - no owner
  - lexical/tool fallback miss
- verification proves the contract on source, shim, `hermes-final`, and live runtime

## Prohibited shortcuts

- broad regex-style behavior rewriting as the main engine
- fake success receipts
- leaving task-like queries on transcript search first
- claiming this phase solved task ownership itself

## Execute notes

- keep the final-answer validator tightly bounded to objective/mechanical constraints
- keep receipts and barriers inspectable
- use the existing trace surfaces rather than inventing parallel debug state

## recommended model level

`xhigh`
