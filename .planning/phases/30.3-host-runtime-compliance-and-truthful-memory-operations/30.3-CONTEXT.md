# Phase 30.3 Context

## Why this phase exists

The memory layer can now surface more truth than before, but the host/runtime layer still overclaims and under-enforces in ways a strict inspector will attack immediately:

- the assistant can claim it saved durable memory without committed evidence
- reset can happen too close to explicit durable user intent
- objective output constraints like em-dash/emoji/bold are still too dependent on model self-control
- lookup misses can be reported too strongly, as if they proved non-existence

## Inputs

- live Discord evidence showed:
  - em-dash violations after successful rule recall
  - false durable-memory confidence around tasks
  - reset immediately after a task-like write
  - task retrieval falling to `session_search` semantics
- the current provider already exposes enough trace/snapshot surface to support a stronger host/runtime contract

## Non-goals

- no task-owner architecture decision here
- no broad behavior-policy compiler redesign here
- no large post-generation rewriting system

## recommended model level

`xhigh`
