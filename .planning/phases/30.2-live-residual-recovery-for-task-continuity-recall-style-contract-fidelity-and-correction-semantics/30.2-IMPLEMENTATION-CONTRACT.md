# Phase 30.2 Implementation Contract

## Required invariants

- task/reminder ownership must remain explicit
- if Brainstack stays context-only for tasks, the product contract and retrieval path must reflect that consistently
- canonical scoped `style_contract` storage must be source-identifiable and fidelity-checkable
- no lossy `tier2_llm` row may silently outrank a stronger deterministic scoped contract without an explicit precedence rule
- compile coverage and ordinary-turn projection coverage must be separated
- active hard rules may not disappear behind a green “compiled_active” surface if they were omitted from the bounded ordinary projection
- structured plain-chat behavior teaching must be able to activate before the first post-teach answer if it is accepted as a valid contract
- same-session user correction semantics must be explicit:
  - chat-only
  - bounded session reinforcement
  - or explicit policy correction
- verification must include:
  - source
  - shim
  - `hermes-final`
  - live runtime

## Minimum acceptable outcomes

- the task residual is closed in one of two defensible ways:
  - Hungarian task recall after reset works from the intended continuity path
  - or the assistant no longer falsely claims durable task ownership it does not have
- the style residual is closed only if:
  - the stored canonical row is lossless for the tested contract
  - and its authoritative source is inspectable
- the projection residual is closed only if:
  - hard-rule omission is visible
  - ordinary-turn degraded status is visible when active hard rules do not fit
  - traces and snapshots stop overclaiming enforcement
- the correction residual is closed only if:
  - ordinary user correction behavior is formally defined
  - and live UAT matches that definition

## Prohibited shortcuts

- promoting tasks into a silent shadow owner just to make the demo pass
- verifying only route mode or packet composition
- relying on English-only summaries for Hungarian recall scenarios
- calling “policy active” a sufficient proof when the stored raw contract is already mutated
- calling compile coverage a sufficient proof when the ordinary-turn projection omitted active hard rules
- adding regex output policing as the main obedience mechanism

## Execute notes

- start from live DB and runtime evidence, not test assumptions
- compare deterministic scoped style-contract writes against Tier-2 writes for the same principal
- inspect continuity retrieval behavior with:
  - same-session queries
  - post-reset queries
  - Hungarian vs English wording
- inspect compile coverage vs actual projection coverage on the same policy snapshot
- treat pre-answer plain-chat activation as a first-class acceptance criterion, not a nice-to-have
- keep the phase narrow:
  - close these three residual classes
  - do not reopen broad architecture threads
