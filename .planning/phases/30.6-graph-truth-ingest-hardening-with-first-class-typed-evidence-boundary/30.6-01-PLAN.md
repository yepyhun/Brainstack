# Phase 30.6 Plan

## goal

Make graph-truth ingest pass through a first-class typed evidence boundary so Brainstack stops writing entity, relation, and temporal truth directly from raw text parsing on the live path.

## scope

### in

- define a first-class typed graph evidence / fact-candidate envelope
- make graph-truth admission accept only typed evidence
- replace raw `graph_text` authority flow in live ingest with typed graph evidence flow
- keep Tier-1 graph extraction conservative, bounded, and inspectable
- preserve graph retrieval and graph backend publication through a compatibility seam that consumes typed evidence instead of raw text
- prove fail-closed behavior when typed graph evidence is absent

### out

- broad KG expansion
- donor-helyetti graph platform rewrite
- corpus schema redesign
- transcript schema redesign
- broad multimodal ingestion platform work
- crash/partial-commit audit across every backend

## core principles this phase must preserve

- follow the canonical immutable project principles:
  - [IMMUTABLE-PRINCIPLES.md](../../IMMUTABLE-PRINCIPLES.md)
- especially:
  - donor-first
  - zero heuristic sprawl
  - fail-closed upstream compatibility
  - truth-first / no "good enough"
  - multimodal-first architecture

## architecture decision

### accepted target model

- transcript and corpus may remain raw text shelves
- graph truth becomes a typed-admission shelf
- the live provider no longer passes raw user/session text as graph authority
- instead, the shell produces bounded typed graph evidence items, and only those may mutate graph truth
- typed graph evidence carries the minimum authority shape needed for graph truth:
  - subject
  - predicate or attribute
  - object or value
  - temporal fields
  - confidence
  - evidence span or source span
  - source turn / source document
  - language
  - provenance class
- Tier-1 remains narrow and conservative
- richer evidence can come later from Tier-2 or explicit structured capture without changing the graph-truth authority boundary

### rejected model

- keep raw-text graph ingest and just add more regexes or multilingual cue lists
- push all graph hardening into Tier-2 while the live path still mutates graph truth from raw text
- rebuild corpus/transcript around typed evidence in the same phase
- broaden this into a general NER / KG / multimodal platform phase

## workstreams

### 1. typed evidence model

- define a first-class graph evidence / fact-candidate contract
- include typed provenance and temporal shape
- ensure the envelope is future-compatible with multimodal evidence instead of assuming text-only forever

### 2. graph admission boundary

- make graph-truth write APIs consume typed evidence, not raw text
- fail closed when the caller does not provide typed evidence
- make “no typed evidence” an explicit non-write state, not a silent fallback to raw parsing

### 3. live ingest path rewrite

- remove raw `graph_text` authority from turn/session ingest planning
- route live graph mutation through typed evidence production instead
- keep the boundary inspectable in provider traces / receipts / debug surfaces

### 4. conservative live extraction

- keep Tier-1 graph extraction narrow and bounded
- allow only high-confidence, well-typed graph evidence on the live path
- do not widen language-specific heuristics as a substitute for typed authority

### 5. graph compatibility and publication

- preserve existing graph retrieval / backend publication shape behind the new typed boundary
- ensure typed evidence still reaches SQLite/Kuzu publication seams cleanly
- keep graph conflicts / supersession semantics attached to typed evidence rather than implicit raw-text parsing state

### 6. observability and proof

- expose enough trace / debug state to show:
  - typed evidence produced
  - typed evidence rejected
  - graph write committed
  - graph write skipped because authority was missing
- prove source, shim, `hermes-final`, and live runtime behavior

## validation ladder

- targeted source regressions for typed graph evidence and fail-closed graph admission
- shim parity smoke
- `hermes-final` reinstall and typed-graph smoke
- rebuilt live runtime smoke on `hermes-bestie`

## anti-goals

- do not hide the problem under new regex or keyword expansions
- do not let raw text keep mutating graph truth “for compatibility”
- do not broaden the phase into generic benchmark chasing
- do not create a second graph truth owner
- do not flatten graph truth back into transcript or corpus retrieval blobs
