# Phase 30.6 Implementation Contract

## invariant

Graph-truth writes must be typed, bounded, inspectable, and fail-closed.

## required implementation properties

- graph truth accepts first-class typed evidence / fact-candidate records instead of raw text
- the live provider no longer treats raw `graph_text` as graph authority
- transcript and corpus may remain raw shelves without becoming graph authority
- Tier-1 graph extraction is conservative and emits typed evidence rather than direct graph mutations
- graph write APIs can distinguish:
  - typed evidence committed
  - typed evidence rejected
  - no typed evidence produced
- graph truth does not degrade open when typed evidence is missing
- typed evidence carries enough provenance to support future multimodal expansion
- existing graph retrieval / publication seams remain compatible behind the typed boundary

## prohibited outcomes

- more regex farms added to raw-text graph ingest as the main “fix”
- raw text still mutates graph truth on the live path “for compatibility”
- transcript or corpus being silently treated as graph authority
- typed evidence becoming just a cosmetic wrapper around the same old raw-text write
- widening the phase into generic KG ambition or broad multimodal platform work

## likely implementation seams

- provider turn/session ingest planning
- graph evidence dataclass / schema module
- donor graph adapter boundary
- graph write API in the store
- graph extraction seam
- graph debug / trace surface
- graph ingress tests and live smoke harness

## verify contract

- targeted source regressions for typed graph evidence and fail-closed graph admission
- shim parity smoke
- `hermes-final` reinstall and typed graph smoke
- rebuilt live runtime proof on `hermes-bestie`
