# Phase 30.6 Context

## problem statement

After `30.5`, the next architecturally similar authority gap sits in graph truth.

The live path still works like this:

- turn/session ingest keeps a raw `graph_text` field
- the donor graph adapter forwards raw text
- raw text is regex-parsed into graph candidates
- those candidates directly mutate graph truth

That means Brainstack's graph shelf still depends too heavily on raw-text parsing instead of a first-class typed truth boundary.

This is not just “some heurisztika exists”.

It means the graph-truth shelf can still drift away from the product promise of:

- explicit entities
- explicit relations
- temporal truth
- supersession
- conflict surfacing

because the authority boundary is still too soft.

## why this is the correct next phase

The README and donor registry give graph truth a serious role:

- relation tracking
- temporal state
- current vs previous truth
- conflicts

But the live ingest path still admits graph truth through raw text.

That violates:

- donor-first product correctness
- zero heuristic sprawl
- truth-first status discipline
- long-range accurate recall and relation-tracking
- multimodal-first architecture

## phase boundary

This phase is specifically about graph-truth ingest authority.

It is not:

- a broad KG phase
- a broad retrieval tuning phase
- a corpus redesign
- a transcript redesign
- a multimodal platform rewrite
- a crash-recovery / partial-commit audit phase

## accepted sharpened reading

The correct move is not to type everything in the system.

The correct move is narrower:

- transcript can stay transcript
- corpus can stay document-oriented
- graph truth must stop taking raw text as authority

So the new hard boundary belongs at graph admission:

- graph writes require typed graph evidence
- raw text may still exist as supporting material in other shelves
- typed graph evidence is the only authority allowed to mutate graph truth

## expected proof shape

The closing proof for this phase should look like this:

1. A plain user turn without acceptable typed graph evidence does not mutate graph truth.
2. A bounded, typed graph evidence item can mutate graph truth.
3. Session-end ingest follows the same authority rule.
4. Graph retrieval still works through the existing graph shelf / backend path.
5. Missing typed evidence yields explicit non-write behavior instead of degrade-open graph mutation.
6. The typed evidence envelope is future-compatible with multimodal evidence, even if this phase only lands text-produced evidence.
