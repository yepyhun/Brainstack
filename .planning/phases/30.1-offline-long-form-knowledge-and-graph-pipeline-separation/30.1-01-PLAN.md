# Phase 30.1 Plan

## goal

Separate long-form knowledge ingestion and offline graph construction from the chat-time memory kernel so both can improve without corrupting each other.

## scope

### in

- doc-to-graph pipeline shape
- evidence/provenance expectations
- separation from chat-memory retrieval

### out

- immediate behavior-policy fixes
- immediate Discord chat recall fixes

## core principles this phase must preserve

1. donor-first
2. modular, thin integration
3. zero heuristic sprawl
4. multimodal-ready
5. token discipline
6. single durable owner per truth class
7. second-brain / proactive-agent fit
8. no dead code / no half-wired paths

## architecture decision

### accepted target model

- separate offline long-form knowledge processing from chat-time memory
- preserve evidence, provenance, and temporal merge as first-class concerns
- prove the shape with a bounded evidence-backed pilot, not only abstract target language

### rejected model

- trying to stretch the live chat graph extractor into a book-scale KG engine

## workstreams

### 1. problem separation

- define chat memory vs long-form knowledge boundaries

### 2. pipeline target shape

- define offline extraction, normalization, merge, and provenance requirements
- define a minimum pilot shape:
  - documents
  - chunks
  - entity mentions
  - canonical entities
  - claims / relations / events
  - evidence spans
  - provenance and conflict candidates
- implement the first pilot as explicit claim-candidate + evidence-snippet binding, not as a fake live auto-extractor
- keep the pilot offline-only; no live Brainstack graph/corpus writes in this phase

### 3. integration contract

- define how offline knowledge may assist the live kernel without becoming its new owner
- keep the first proof bounded instead of trying to jump straight to book-scale rollout

## validation ladder

1. boundary proof
2. offline target shape
3. integration contract review
4. evidence-backed pilot proof
5. installed `final` payload parity and offline-only import proof

## anti-goals

- no all-in-one memory blob architecture
- no premature implementation sprawl

## recommended model level

`xhigh`
