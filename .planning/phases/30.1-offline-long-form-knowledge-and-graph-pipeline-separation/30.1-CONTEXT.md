# Phase 30.1 Context

## problem statement

Chat-time memory and long-form document / book knowledge are not the same problem.

Treating them as one "memory" problem creates architectural confusion.

## why this phase exists

The external review correctly warned that the current live graph path is too narrow for large-scale knowledge ingestion.

That warning is useful, but only if it becomes a separate, bounded roadmap thread rather than derailing chat-memory recovery.

## explicit non-goals

- no premature rewrite of the current chat-memory kernel
- no mixing large offline KG ambitions into ordinary-turn policy work
## recommended model level

`xhigh`
