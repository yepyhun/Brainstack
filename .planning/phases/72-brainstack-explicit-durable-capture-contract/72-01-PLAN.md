# Phase 72 Plan

## purpose

Add a deterministic, multilingual-safe explicit durable capture path for user-owned facts, profile updates, project state, procedures, and corrections without resurrecting Tier-1 keyword farms.

## core question

How can Brainstack capture explicit user memory intent reliably without using language-specific regex, broad live inference, or assistant-authored recap promotion?

## GSD planning enforcement check

This phase must preserve:

- no heuristic sprawl: explicit capture must be typed/tool-driven, not phrase-list driven
- multilingual by design: schemas cannot depend on English wording
- truth-first: rejected/no-op writes must be visible and honest
- donor-first: Hermes-native explicit truth remains host-owned where applicable
- non-governor: capture stores memory, not execution commands

## shared north star and real-world proof gate

North star for Phase 70-77: Brainstack must become easier and safer for Hermes agents to use as a memory kernel without expanding Brainstack into runtime execution, workflow governance, or fake autonomy.

Every phase closeout must prove one of these:

- Hermes can use Brainstack more reliably in a real memory workflow
- or the phase is prerequisite work and proves no regression while naming the exact later phase that must deliver the user-visible improvement

Minimum proof shape:

- targeted tests for the changed surface
- at least one end-to-end Hermes/Brainstack interaction path or Brainstack packet/inspect proof using real provider/store code
- evidence ID/source/provenance proof where memory is involved, not only generated answer text
- no-regression check against Phase 66 golden gates once they exist
- bounded token/output sanity for any new tool, MCP, inspect, corpus, or trace surface

## explicit scope model

### out of scope

- assistant recap promotion
- always-on broad extraction
- runtime task execution

## architecture decisions to freeze before execution

### 1. capture schema

Capture records must carry:

- content
- shelf target
- principal scope
- authority class
- provenance
- validity window
- correction/supersession target when applicable
- confidence and source evidence

### 2. explicitness rule

Only explicit user intent, explicit tool call, or typed producer evidence may create durable authority. Tier-2 can suggest but not silently create high-authority memory without policy.

### 3. multilingual rule

The schema must be language-neutral. Any natural-language adapter is a boundary adapter, not the core intelligence.
