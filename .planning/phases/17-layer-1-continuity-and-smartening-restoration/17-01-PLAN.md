# Phase 17 Plan

## Goal

Restore Layer 1 as a real Hindsight-shaped smartening layer.

The result of this phase is not “slightly nicer recall”.
It is a donor-faithful executive retrieval path that makes the agent feel materially smarter across sessions, without falling back to handwritten language heuristics.

## Scope

### In

- replace the current Tier-1-centered L1 behavior with a donor-first executive retrieval path
- restore Hindsight/TEMPR-style multi-channel recall logic as the primary L1 path
- define a stable retrieval-channel contract that L1 can use now and that later `18/19` can plug into without redesigning L1 again
- make `control_plane.py` and `retrieval.py` serve the donor-restored L1 instead of cosmetically rescuing weak recall
- add fast verification gates that measure actual smartening rather than only storage correctness

### Out

- no new handwritten trigger lists or regex growth
- no benchmark-first tuning loop
- no L2 backend migration in this phase
- no L3 backend migration in this phase
- no new service mesh or runtime zoo
- no “temporary bridge” that keeps the old Tier-1 path as the real center

## Planning Assumption

`17` comes before `18` and `19`.

That means this phase must do two things at once:

1. restore Hindsight-shaped executive retrieval behavior now
2. do it through a stable channel contract so later `Kuzu` and `Chroma` adoption in `18/19` does not require another L1 rewrite

This is not a temporary heuristic bridge.
It is the final L1 shape with initially available providers behind it.

## Execution Steps

1. Define the L1 retrieval-channel contract.
   - Create an explicit channel/result schema for L1 candidates.
   - At minimum support:
     - semantic
     - keyword / FTS
     - graph
     - temporal
   - Require each candidate to carry:
     - source shelf / source channel
     - confidence / score components
     - temporal basis
     - provenance handles
     - token / packing metadata needed by the recall packager

2. Isolate the Hindsight-shaped fusion core.
   - Add a dedicated L1 fusion module instead of scattering ranking logic across:
     - `control_plane.py`
     - `retrieval.py`
     - `usefulness.py`
   - Preserve donor shape:
     - multi-channel candidate gathering
     - fusion
     - bounded reranking / prioritization
   - The implementation target is a faithful TEMPR-style executive retrieval path, not a new Brainstack-specific ranking invention.

3. Rebind current evidence sources behind the channel contract.
   - Feed L1 from current shell-owned continuity/profile/transcript evidence where appropriate.
   - Feed graph signals through the current graph seam in a way that can later be replaced by the `18` Graphiti/Kuzu backend.
   - Feed corpus signals through the current corpus seam in a way that can later be replaced by the `19` MemPalace/Chroma backend.
   - The contract must be backend-agnostic, but the behavior must already be donor-first.
   - This step must also make an explicit early decision for the **semantic** channel:
     - if a bounded local embedding/vector path can be added inside `17` without violating the phase boundary, do it
     - otherwise, declare the semantic channel temporarily degraded and score/fuse it honestly instead of pretending it is present
   - An empty or fake semantic leg is not allowed; the phase must explicitly choose:
     - bounded interim semantic retrieval now
     - or honest degraded-fusion behavior until `19`

4. Remove Tier-1 heuristic centrality from live smartening behavior.
   - `tier1_extractor.py` may remain only as a bounded write-side helper where already justified.
   - It must stop being the effective intelligence center for follow-up behavior.
   - The live “what matters from the past?” path must come from the restored L1 retrieval engine.

5. Thin the local rescue layers.
   - Reduce `control_plane.py` to routing / budgeting / orchestration responsibilities.
   - Reduce `retrieval.py` to privacy / packaging / presentation responsibilities.
   - Keep `usefulness.py` only as bounded telemetry and modest tuning, not as a hidden replacement relevance engine.
   - Any logic that exists only to make weak recall look strong must be deleted or moved behind the new L1 core if it is genuinely part of donor-shaped retrieval.

6. Add donor-faithfulness tests, not just behavior snapshots.
   - Add focused tests around:
     - multi-channel candidate gathering
     - fusion ordering
     - temporal sensitivity
     - graph-assisted recall
     - token-budget-aware packing handoff
   - Use donor-shaped scenarios, not regex-shaped unit tests.
   - The goal is to prove that L1 behaves like a continuity/smartening engine, not like a transcript search helper.

7. Build the fast eval ladder for repeated implementation checks.
   - Gate A:
     - Brainstack-adapted kernel-memory acceptance scenarios
   - Gate B:
     - mini smartening suite
     - must check whether follow-ups carry the right thing forward
   - Gate C:
     - small LongMemEval subset as regression guard
   - Gate D:
     - final boss deferred to milestone-level restore gate, not every local iteration

8. Prove the live runtime shape.
   - Reinstall through the integration kit.
   - Confirm the running Bestie uses the new L1 path.
   - Run bounded live smoke focused on:
     - smarter follow-up behavior
     - less noisy retrieval
     - no regression into narrated memory internals
     - no reversion to heuristic-centered capture/recall

## Implementation Rules

- donor-first means donor-faithful behavior, not donor-flavored wording
- no new handwritten multilingual heuristics
- no “just enough until Kuzu/Chroma arrives” design shortcuts that would require another L1 rewrite
- no routing magic hidden in packaging layers
- no local scoring glue that becomes a substitute for restored donor behavior
- benchmark uplift may validate the work, but benchmark-targeting may not drive the implementation
- if the semantic channel is not yet fully donor-backed, the implementation must state that explicitly and fuse accordingly; it may not silently treat FTS as a fake semantic stand-in

## Output Shape Required From Phase 17

The implementation must leave behind:

1. a distinct L1 executive retrieval module
2. an explicit retrieval-channel contract
3. thinner shell routing/packaging layers
4. donor-faithful tests
5. a fast smartening-oriented eval ladder
6. live proof that the agent feels smarter, not just more verbose about memory
7. an explicit documented decision for the temporary-or-real semantic channel used in `17`

## Exit Gate

- the old Tier-1 heuristic path is no longer the effective center of L1 behavior
- L1 now uses a donor-faithful multi-channel executive retrieval path
- `control_plane.py` and `retrieval.py` are thinner, not fatter
- the implementation is compatible with later `18/19` backend restoration without another L1 redesign
- the semantic leg is real or explicitly degraded-by-design, not silently missing
- fast acceptance and smartening suites pass
- a small LongMemEval subset improves without becoming the main target
- live follow-up behavior is materially smarter in bounded smoke
