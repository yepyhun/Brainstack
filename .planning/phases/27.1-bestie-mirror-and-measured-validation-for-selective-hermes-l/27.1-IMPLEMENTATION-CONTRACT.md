# Phase 27.1 Implementation Contract

## Objective

Take the landed Phase `27` Brainstack source truth, mirror it narrowly into Bestie, and prove whether it produces measurable Bestie-side value.

## System doctrine this phase must preserve

- Brainstack source remains the source of truth
- Bestie is a mirror/integration target, not a second design surface
- no new donor-uptake work is allowed here
- no new host architecture slot is allowed here
- no claim of product win without measured evidence

## Workstream A: Source-to-mirror mapping

- diff the landed Phase `27` Brainstack source commit
- identify the exact Bestie mirror seam for:
  - bounded snapshot provenance
  - lifecycle/frontier state
- prove whether the mirror is clean or drifted

## Workstream B: Narrow Bestie mirror

- mirror only the landed Brainstack source slices
- keep the Bestie patch minimal and traceable back to Brainstack source
- stop if the patch turns into Bestie-specific redesign

## Workstream C: Runtime proof

- prove Bestie runtime now persists:
  - bounded snapshot provenance
  - lifecycle/frontier state
- prove the integrated runtime still passes its affected invariants

## Workstream D: Measured value proof

- run a bounded before/after comparison
- acceptable measured axes:
  - token discipline
  - retrieval targeting / usefulness
  - auditability / diagnostics
- if only one of those improves, say that precisely
- if none improve measurably, say that precisely

## Protected boundaries

### Anti-slop boundary

- no “it is cleaner internally therefore it is better” closeout
- no fuzzy success wording

### Mirror boundary

- Bestie must not become a second source of truth
- if Brainstack source and Bestie mirror diverge, that divergence must be called out

### Token boundary

- no ordinary-turn token growth without explicit proof that it is still worth it

### Truth boundary

- “correct mirror” and “measured product win” are different verdicts
- they must not be collapsed into one

## Minimum evidence required before calling Phase 27.1 done

- exact Brainstack-to-Bestie mirror mapping
- integrated Bestie runtime proof for the two landed slices
- measured comparison result
- explicit verdict:
  - measured win
  - correct mirror, no proven product lift yet
  - not worth keeping
