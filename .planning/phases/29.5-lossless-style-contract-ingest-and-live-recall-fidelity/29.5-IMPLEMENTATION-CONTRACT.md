# Phase 29.5 Implementation Contract

## phase type

Forensics-first bounded deployed-path correctness repair.

## required final architecture

- one canonical principal-scoped detailed style-contract owner row:
  - `preference:style_contract`
- one compact operational communication lane for ordinary turns
- no second detailed owner in corpus, files, skills, or persona artifacts
- explicit deployed-path access to the canonical detailed lane
- lossless preservation of taught section headings and rule lines inside the canonical detailed lane
- later corrections must supersede the prior canonical wording in that same lane

## minimum evidence required before calling the phase done

- a failing reproduction proves the current deployed-path blocker
- a targeted fix removes that blocker without reopening unrelated retrieval seams
- the live DB shows the canonical row contains the full current taught contract, not a shortened subset
- a post-reset explicit recall works in the deployed runtime, not only in isolated local probes
- ordinary turns still exclude the full detailed style contract

## explicitly disallowed outcomes

- claiming success based only on DB shape while Discord/live recall still fails
- corpus or transcript becoming the hidden detailed owner again
- heuristic transcript parsing added as the primary fix
- prompt policing or formatting guardrails pretending to solve memory
- leaving behind dead code from prior lossy style-contract experiments

## cleanup requirement

- any abandoned partial detailed-lane logic must be removed, not merely bypassed
- no dormant alternative detailed style owners may remain active

## compatibility requirement

- existing compact communication slots must continue to work
- explicit style-contract recall must keep using the canonical detailed lane
- live runtime install parity must remain intact

## regression-audit requirement

- confirm that the repaired style-contract ingest does not disturb:
  - ordinary compact communication contract projection
  - direct identity slot recall
  - logistics memory boundaries
  - the repaired oracle runner gate from `29.4`

## code hygiene requirement

- no broad refactor unless the proven collapse point demands it
- no dead helpers
- no duplicate renderers for the same style-contract payload

## handoff requirement

The final summary must say explicitly:

- where the style-contract was being shortened
- why that seam was the real culprit
- what the canonical repaired path now is
- what residual, if any, still belongs to model-behavior rather than memory storage

## recommended model level

`xhigh`
