# Phase 61.1 Architecture Decision

## frozen decision

Do not roll back to `v1.0.16`, and do not restore task/operating cue heuristics.

Instead, keep `structured_understanding` as the preferred typed-understanding seam, but reduce its ordinary-kernel blast radius in two bounded ways:

1. explicit availability-bounded degraded mode
   - query understanding degrades to fact route with null structured lookups
   - capture understanding degrades to no-capture mode
   - degraded responses are explicit and non-misleading

2. remove unnecessary duplicate route-understanding work on the read path
   - if `control_plane` already produced route payload, `executive_retrieval` must not perform a second understanding call

## accepted rationale

- the project's no-heuristic rule remains intact
- live evidence showed real structured-understanding failures in production
- the correct response is architectural stabilization, not heuristic restoration and not timeout theater

## explicit anti-decisions

- no cue-list fallback
- no phrase-table rescue
- no provider-specific branching in Brainstack kernel logic
- no transcript/tool fallback disguised as successful memory understanding
