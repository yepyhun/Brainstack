# Phase 61.1 Implementation Contract

## execution intent

Stabilize the `v1.0.17` Brainstack kernel transition in place. Preserve the no-heuristic direction, but sharply reduce the number of ordinary-kernel outcomes that depend on successful live structured-understanding calls.

## required architecture discipline

### 1. no heuristic rollback

Forbidden:

- cue lists
- keyword farms
- locale-specific phrase detection
- exact-phrase rescue paths
- deterministic routing tables based on wording

### 2. structured understanding stays a seam, not the whole kernel

Allowed:

- keeping `structured_understanding.py` as the typed-understanding seam
- using it where explicit semantic interpretation is genuinely required

Required:

- execution must identify where ordinary read behavior can remain correct without fresh understanding success
- execution must remove or narrow mandatory use where a fact-safe degraded mode is possible

### 3. degraded mode must be explicit and safe

If structured understanding fails, times out, or returns unusable output:

- the kernel may fall back to bounded fact-mode retrieval
- it must not fabricate task or operating claims
- it must not silently escalate to transcript dumps
- it must not present tool-driven search as if Brainstack memory had succeeded

### 4. ordinary write authority must stay disciplined

Preferred durable sources:

- explicit native write seams
- compact operating-truth consolidation
- already-typed durable records

If any automatic capture path remains dependent on structured understanding, the execution must justify why that path is still necessary and how failure behaves safely.

### 5. no provider hacks

Forbidden:

- provider-specific control flow in Brainstack kernel logic
- model-name exceptions as the main correctness path
- timeout inflation presented as stabilization

Runtime/provider configuration may still be used operationally, but it is not the architectural answer for this phase.

### 6. no Phase 62 scope bleed

This phase is not allowed to absorb:

- heartbeat truth modeling
- pulse/evolver live-state authority
- cron capability-truth decisions

Those remain Phase 62.

## required design decisions to freeze before execution

- which current call sites are:
  - mandatory
  - degradable
  - removable from ordinary path
- what the fact-safe degraded Brainstack packet looks like
- what the answer path may and may not do after degraded retrieval
- what release freeze condition is required before Phase 62 proceeds

## accepted change shapes

- reducing mandatory structured-understanding calls on ordinary read surfaces
- making route handling degrade safely to fact mode
- rebiasing authority toward typed durable records and compact operating truth
- adding tests that prove failure does not become misleading pseudo-success

## rejected change shapes

- restoring old `v1.0.16` routing logic
- keyword-based recap/task/operating classifiers
- broad new host seams
- transcript search acceleration masquerading as memory stabilization
- another large architecture rewrite beyond what is needed to stabilize the seam

## proof obligations for the later execute step

- show which call sites were narrowed or left mandatory and why
- show that structured-understanding failures no longer produce misleading task/operating behavior
- show that the stabilized kernel still avoids heuristic routing
- show that the resulting state is a better base for Phase 62, not a detour away from it

## inspector note

The execute step must be able to answer both of these directly with code evidence:

- why this is not a hidden heuristic rollback
- why this is not just timeout/provider containment dressed up as architecture
