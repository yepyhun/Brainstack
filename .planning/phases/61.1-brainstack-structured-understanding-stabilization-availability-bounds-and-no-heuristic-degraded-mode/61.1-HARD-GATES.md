# Phase 61.1 Hard Gates

## 1. no-heuristic-rollback gate

- no cue tables, keyword farms, locale-specific phrase routing, or exact-phrase rescue paths are reintroduced in Brainstack code

## 2. stabilization-not-reversion gate

- the implementation does not use `v1.0.16` as the functional recovery target
- any retained pre-`v1.0.17` logic must be justified as structural code reuse, not heuristic rollback

## 3. degraded-mode gate

- structured-understanding failure must produce a bounded, explicit degraded mode
- degraded mode must not fabricate task or operating truth
- degraded mode must not pretend transcript/tool fallback is successful Brainstack recall

## 4. ordinary-read blast-radius gate

- ordinary Brainstack read behavior must no longer depend on successful structured-understanding calls more than is strictly necessary
- the closeout must state what dependency remains and why

## 5. no-provider-hack gate

- the phase must not claim success through provider pinning, timeout inflation, or provider-specific branching alone

## 6. no-scope-bleed gate

- Phase 62 live-state authority work must remain separate
- no heartbeat / pulse / evolver truth modeling is smuggled into this phase
