# Phase 14.2 Context

## Problem

Live Discord proving showed that Brainstack durability improved, but the runtime still has three correctness gaps:

- fresh communication preferences are not applied strongly enough in the very next replies
- `skill_manage` is still available as a side path for user-profile style memory, which breaks Brainstack-only ownership
- identity and preference records can drift into duplicates such as `User` vs `Tomi` and heuristic vs Tier-2 preference variants

## Agreed Architectural Direction

- do not fully ban `skill_manage`
- keep `skill_manage` for reusable workflows and procedures
- make Brainstack the only owner of personal profile, style, identity, and relationship memory
- harden the runtime so the wrong ownership path is rejected, not merely discouraged
- improve prompt-time application of fresh user communication preferences without adding a new runtime
