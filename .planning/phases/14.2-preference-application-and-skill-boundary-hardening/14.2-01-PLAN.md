# Phase 14.2 Plan

## Goal

Harden Brainstack-only runtime behavior so personal profile/style memory stays exclusively in Brainstack, fresh communication preferences influence the next replies more reliably, and user identity/preference records stop fragmenting into parallel truths.

## Workstreams

### W1. Fast Preference Application

- identify why freshly written preference records are not shown strongly enough in the next prompt block
- add a bounded same-session preference boost so recent user communication preferences and identity signals are surfaced without waiting for broad semantic matching
- keep token discipline bounded

### W2. Scoped `skill_manage` Boundary

- preserve Hermes procedural skill learning
- block or reject `skill_manage` calls that attempt to store user identity, profile, communication style, or personal preference memory in Brainstack-only mode
- keep the installer payload and host patches aligned with the new runtime contract

### W3. Identity And Preference Normalization

- reduce duplicate personal-memory records where possible
- converge graph writes away from split aliases like `User` vs `Tomi`
- make future writes cleaner without adding a new layer or runtime

### W4. Verification

- add targeted tests for:
  - same-session preference boost
  - skill boundary rejection in Brainstack-only mode
  - canonical user identity graph behavior
- reinstall into the Bestie runtime
- run bounded live/runtime verification

## Exit Gate

- freshly corrected style rules influence the next replies materially better than before
- `skill_manage` still works for procedures, but not for personal profile/style memory
- identity/preference duplication is reduced and canonical handling is clearer
- installer carry-forward is preserved
