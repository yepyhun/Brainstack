# Phase 36 Context

## problem statement

The latest Brainstack shape is no longer primarily failing at truth ownership.

After `34` and `35`, the stronger remaining problem is packet assembly quality:

- Brainstack can now own more truth correctly
- but the turn-time working-memory packet can still feel noisy, duplicative, and overly directive

This is not only a UX complaint.

The current architecture still shows the following pattern:

- Brainstack retrieval renders many explicit sections
- system substrate can already inject memory-bearing sections before the working-memory block
- Hermes host wrapping adds another high-authority memory boundary
- continuity, recent continuity, and transcript can all describe nearly the same turns
- profile, graph, and operating truth can all restate related facts with insufficient cross-section collapse
- operating truth has improved, but continuity-derived fallback still remains strong enough to keep the packet busier than it should be

That means the system can improve truth integrity and still underperform as a second-brain hot path.

## why this is the correct next phase

Phase `34` narrowed the behavior harness.

Phase `35` strengthened the operating substrate.

But neither phase was primarily about the final assembly rule:

- how many shelves make it into the packet
- in what order
- with what authority wording
- and with what cross-section dedupe

That remaining assembly problem is now the main product drag:

- it wastes tokens
- it makes the packet feel louder than it should
- it can cause owner-backed truth and fallback prose to coexist unnecessarily
- it lets the model see too many overlapping memory explanations instead of one clean memory substrate

## accepted sharpened reading

- the user is right that Brainstack can still “get in the way” even after real progress
- that does not mean the storage and ownership work was wrong
- it means the final hot-path packet still has an unresolved assembly problem
- the missing fix is not more extraction and not more compliance wiring
- the missing fix is:
  - owner arbitration
  - render collapse
  - cross-section dedupe
  - wrapper simplification

This phase should therefore improve the packet itself, not just the underlying shelves.

It must treat the packet as the combined hot-path memory surface:

- system substrate additions
- working-memory additions
- host/runtime boundary framing

## phase boundary

This phase is about:

- low-noise packet assembly
- owner-driven rendering
- section collapse
- deduplicated factual rendering
- truthful but narrower host/runtime memory framing

This phase is not about:

- replacing Brainstack with transcript search
- deleting exact recall
- weakening fail-closed truth semantics
- new graph/behavior/platform subsystems
- heuristic farms or test-shaped packet hacks

## expected proof shape

The proof should show:

- fewer overlapping sections in ordinary turns
- fewer duplicate facts across shelves
- fewer layered meta-instructions
- better token efficiency without silent truth loss
- preserved exact contract recall
- preserved owner-backed factual truth
- preserved operating-truth precedence where explicit operating records exist
- packet-quality improvement across the full combined hot path, not just one render helper

The proof should also become more measurable than before. It should capture at least:

- section count
- combined character footprint
- semantic-key repetition
- cross-surface duplicate facts
- wrapper-note multiplicity

The key proof is not just “shorter”.

It is:

- quieter
- cleaner
- more truthful
- more owner-consistent
