# Phase 36 Plan

## goal

Close the remaining architect gap after `34` and `35`:

- Brainstack now has better truth ownership
- Brainstack now has a better operating substrate
- but the turn-time memory packet is still assembled too noisily

This phase should make the working-memory packet:

- quieter
- less duplicated
- more owner-driven
- less governance-shaped
- more token-efficient

The target is not “shorter prompt at any cost”.

The target is:

- one coherent memory packet
- built from the most authoritative owners first
- rendered with less duplication and less layered meta-instruction
- while preserving exact recall, durable truth, and fail-closed honesty

For this phase, `packet` means the combined hot-path memory payload the model actually sees:

- system substrate / system-prompt memory additions
- working-memory block additions

The problem is not isolated to one render function. It is the combined packet footprint and combined authority shape.

## scope

### in

- packet assembly collapse:
  - inspect and simplify the final combined packet assembly order
  - remove or merge sections whose owner role is redundant
  - stop treating section concatenation as the default integration strategy
  - treat system substrate and working-memory rendering as one packet-quality surface, not two unrelated optimizations
- owner arbitration before rendering:
  - decide likely authoritative owner(s) first
  - then render only the minimal needed shelves for that turn
  - move away from “render many shelves, let the model sort it out”
- cross-section deduplication:
  - deduplicate repeated truths across:
    - profile
    - graph
    - continuity
    - transcript
    - operating truth
  - require dedupe to use owner identity, stable keys, record identity, or provenance-linked identity where available
  - do not introduce phrase-matching farms as the main dedupe strategy
- wrapper collapse:
  - remove double-authority layering between Brainstack retrieval framing and Hermes memory-manager framing
  - keep one truthful, proportionate memory-context boundary instead of multiple overlapping internal notes
  - make the remaining boundary consistent across system substrate and working-memory payloads
- continuity collapse:
  - collapse the current multiple continuity surfaces toward one bounded representation for ordinary turns
  - keep exact transcript evidence available where needed, but stop routinely rendering both continuity and transcript as overlapping retellings of the same turns
- operating-truth precedence:
  - when explicit operating truth exists, it should outrank continuity-derived paraphrase in the rendered packet
  - continuity fallback should stay available only where a structured owner miss actually happened
- contract lane discipline:
  - preserve the exact canonical contract lane for explicit recall/debugging
  - preserve only the small ordinary-turn invariant lane for routine hot-path use
  - do not let the full contract re-enter the packet through adjacent sections or host framing
- token discipline:
  - prove that the quieter packet saves meaningful hot-path space by removing duplication rather than by deleting needed truth
  - measure packet quality on the combined hot path, not just one sub-block in isolation

### out

- broad behavior-policy redesign
- broad graph redesign
- broad operating-truth redesign
- new benchmark-shaped tuning
- any solution that relies mainly on more keyword routing
- transcript stuffing as a substitute for owner arbitration
- weakening truthful memory semantics just to make the packet shorter
- hiding missing truth behind aggressive collapse or omission

## core principles this phase must preserve

- donor-first
- modularity / upstream updateability
- fail-closed upstream compatibility
- no benchmaxing
- truth-first / no "good enough"
- zero heuristic sprawl
- multimodal-first architecture

## architecture decision

### accepted target model

- Brainstack should assemble working memory through owner arbitration, not section accumulation
- the combined packet should expose the minimum distinct shelves needed for the current turn
- the same underlying truth should not appear multiple times just because several shelves can reach it
- the host/runtime should contribute one memory boundary, not a second quasi-system instruction layer on top of Brainstack’s own guidance
- continuity, transcript, and operating truth should have explicit precedence and fallback rules rather than being routinely rendered side by side
- exact archival recall must survive
- ordinary-turn support must stay narrow
- token savings must come from collapsing redundancy, not from dropping truth or inventing heuristics
- packet quality should be evaluated across both:
  - system substrate injection
  - working-memory injection

### rejected model

- solving the packet problem with more render-specific heuristics
- keeping duplicate sections because “more evidence feels safer”
- adding new owner classes instead of cleaning up the current render path
- replacing owner arbitration with another budget-shaping layer
- using section collapse to obscure stale or missing data

## workstreams

### 1. packet assembly audit

- map the exact ordinary-turn combined assembly order from:
  - retrieval
  - operating context
  - system substrate rendering
  - host memory wrapper
- classify every rendered section as:
  - distinct owner-backed truth
  - bounded support
  - redundant render
  - duplicated wrapper/governance
- freeze the accepted target packet shape before implementation spreads

### 2. owner arbitration and precedence

- define turn-time owner arbitration explicitly before rendering
- specify precedence for the main personal-memory surfaces:
  - operating truth
  - task truth
  - profile truth
  - graph truth
  - continuity
  - transcript
  - corpus
- require that continuity and transcript behave as fallback/support owners, not routine parallel retellings of the same content
- require that operating truth replace continuity-derived prose where it has an explicit committed record

### 3. render-path collapse and dedupe

- implement bounded cross-section dedupe based on stable owner identity and record identity where possible
- remove or merge overlapping meta-sections whose semantics do not justify separate render space
- converge continuity / recent continuity / transcript overlap toward a smaller number of distinct packet surfaces
- make duplicate factual renders across profile and graph impossible for the same underlying fact unless there is a clear conflict or temporal reason
- add cross-surface overlap suppression so the same truth does not appear once in system substrate and again in working-memory evidence without a distinct owner reason

### 4. host/runtime boundary cleanup

- collapse the remaining double-note / double-authority framing between Brainstack and Hermes host seams
- keep exactly one truthful memory-context boundary
- ensure host wording does not upgrade recalled memory beyond the guarantee level already established in Brainstack
- keep the runtime donor-friendly and installer-friendly

### 5. proof

- prove lower-noise packet assembly on source, `hermes-final`, and rebuilt live runtime
- prove fewer rendered sections and fewer duplicated facts on ordinary turns
- prove that exact contract recall still works
- prove that operating truth is surfaced ahead of continuity fallback when present
- prove that token reduction comes from collapse and dedupe, not from silent loss of truth
- add packet-quality regressions that measure the combined prompt surface, including:
  - section count
  - combined character footprint
  - repeated semantic-key count
  - cross-surface duplicate count
  - duplicate wrapper-note count

## validation ladder

- source:
  - targeted tests for owner arbitration, cross-section dedupe, wrapper collapse, and continuity/transcript collapse
  - tests for preserved exact recall and fail-closed honesty after packet simplification
  - tests for distinct-owner conflict cases where dedupe must not erase meaningful disagreement
  - packet-quality regressions for combined system substrate + working-memory footprint
- `hermes-final`:
  - payload parity plus direct interpreter proof that the quieter packet shape is actually installed
  - proof that only one host/runtime memory boundary remains
  - proof that combined packet duplication metrics stay within the new bounded target shape
- live runtime:
  - rebuilt runtime smoke showing fewer duplicated sections and lower packet noise on natural personal / continuity / operating queries
  - proof that the packet stays truthful, exact recall still works, and no new heuristic farm was introduced
  - proof that packet collapse survives the actual host/runtime assembly path, not just the source-side render helper
