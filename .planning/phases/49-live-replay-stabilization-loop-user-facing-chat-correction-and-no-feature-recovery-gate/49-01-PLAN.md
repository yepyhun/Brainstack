# Phase 49 Plan

## goal

Recover dependable live Bestie chat quality by running a closed-loop replay-and-correction phase against real failing conversations until the docker/live runtime stops leaking internal blockers, tool traces, routing drift, transcript loss, and reminder-timezone mistakes.

## scope

### in

- build a bounded replay pack from real failing Bestie conversations
- define user-facing acceptance metrics for live chat stability
- fix cross-layer defects only where they block replay-pack success:
  - Brainstack authority bootstrap and final-output compliance delivery
  - blocked-output handling on the host path
  - natural style-recall routing on the live path
  - transcript persistence on the gateway path
  - user-surface tool-trace containment
  - reminder timezone correctness
- rerun the whole replay pack after each correction
- finish with one combined docker/live proof pass

### out

- new memory owners, new shelves, or new retrieval capabilities
- graph value expansion or multimodal feature growth
- benchmark-specific prompt shaping
- broad heuristic farms for free-form intent recognition
- unrelated product polish not needed for replay-pack recovery

## workstreams

1. Replay pack and acceptance gate
- collect a small bounded pack of real failing conversations from the current live incidents
- define one replay runner surface and one results artifact
- lock user-facing metrics:
  - `blocked_user_leak_count = 0`
  - `tool_trace_leak_count = 0`
  - `style_recall_success = pass`
  - `authority_ready_when_active = pass`
  - `final_output_compliance = pass`
  - `transcript_persist_errors = 0`
  - `reminder_timezone_correct = pass`

2. Authority and enforcement convergence on the live path
- keep active style authority and compiled policy converged on the real runtime path
- eliminate the case where blocked validation turns into a user-facing internal blocker message
- require internal regenerate-or-rewrite behavior before anything user-facing ships

3. Owner-first style recall on natural questions
- make natural rule/obedience questions hit style authority on the live path
- keep the detection bounded and owner-first rather than broad cue farming

4. Gateway transcript and user-surface stabilization
- close transcript persistence defects on the host path
- ensure internal tool progress and internal blocker text never leak into ordinary chat

5. Reminder scheduling correctness
- prove reminder creation, listing, and delivery are timezone-correct on the live path
- prevent UTC/local display drift from reappearing in ordinary chat behavior

6. End-to-end replay loop
- execute the full replay pack after every correction batch
- no local green result is accepted if the combined replay pack still fails
- finish with docker/live replay proof and one concise operator artifact

## deliverables

- replay pack artifact with the bounded real-conversation cases
- replay runner and results artifact for source, host, and docker/live surfaces
- corrected host/plugin/runtime behavior for the live chat failure chain
- one final verification summary covering the full replay pack

## validation ladder

1. targeted source/host tests for each corrected defect
2. replay-pack run on local source/host surfaces
3. installed payload parity where plugin files are touched
4. docker/live replay run against the actual Bestie runtime path
5. one final combined replay report showing all acceptance metrics green
