# Phase 16.2 Module Recovery Contract

## Classification Legend

- `keep local` = justified Brainstack shell responsibility
- `thin down` = keep the seam, reduce local intelligence or scope
- `donor-recenter` = later phase must restore donor-shaped strength here
- `stop growing` = no more local expansion in this direction

## Explicit Stop-Growth Rules

These patterns are now explicitly disallowed:

- expanding `tier1_extractor.py` with more language-specific preference heuristics
- adding more local ranking rules to `control_plane.py` to fake donor recall strength
- adding more local packaging tricks in `retrieval.py` to make weak retrieval look strong
- calling L3 “good enough” because documents are stored, while retrieval and packing remain weak

## Recovery Shape For Later Phases

### Phase 17

- restore L1 smartening donor-first
- keep transcript durability and shell boundaries
- remove Tier-1 heuristic centrality as the primary L1 behavior
- restore Hindsight/TEMPR-style retrieval/fusion behavior instead of inventing a new local retrieval engine
- treat L1 as executive retrieval intelligence over donor-backed retrieval channels, not as another storage engine

### Phase 18

- restore L2 graph usefulness donor-first
- keep temporal/provenance/safety shell value
- reduce graph-side local approximation as substitute for graph strength
- target an embedded Graphiti-shaped graph backend, with Kuzu as the default recovery target unless evidence disproves it

### Phase 19

- restore L3 corpus power donor-first
- keep bounded packing and ownership shell value
- reduce nominal corpus shelf behavior that lacks real retrieval advantage
- target MemPalace-style raw corpus retrieval as the primary restored strength, with Chroma as the default embedded corpus backend target

## Non-Negotiable Rule

If later work improves complexity or code volume but still leaves donor-strength weak, it fails this contract.
