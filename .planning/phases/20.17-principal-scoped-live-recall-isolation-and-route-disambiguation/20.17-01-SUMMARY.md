## Result

Phase `20.17` is execution-complete in source and deployed Bestie, but **not** cleanly at gate on live answer-only score.

What is now clearly fixed:

- principal-scoped live recall isolation is restored
  - unrelated other-principal rows are no longer the dominant live packet failure
- noun-`order` recall requests stay on the fact route instead of temporal
- reset-boundary `session_summary` rows now preserve principal scope through the gateway finalize path
- deployed live truth is now explicitly recorded with `semantic` / `graph` degraded
  - current `20.17` gains therefore live on the active `keyword + temporal` stack, not on graph/corpus-backed runtime enablement

What is **not** yet solved:

- the live isolated answer-only score is still unstable / low (`2 / 5` on the latest repeated run)
- one real structural residual remains on the active degraded live stack:
  - `temporal_order` still fails because same-principal prior events are only partially surfaced for a low-overlap ordering query
  - the localized packet now surfaces `dentist` and `bike brake pads`, but still drops the third event (`tax accountant`)
- the remaining `coffee_order` and `gift_context` failures are not the same leak/misroute class
  - their packets already contain same-principal evidence
  - so those failures now read more like live answer-level packet use / synthesis weakness than principal leakage

## What This Phase Does Not Claim

- that deployed Bestie now has graph/corpus-backed live retrieval enabled
- that the reset-boundary live eval is fully green
- that the remaining live failures are all one class
- that broader aggregate expansion should resume immediately

## Explicit Pending Gaps

- same-principal low-overlap temporal recall under degraded live stack
  - current `keyword + temporal` live runtime can still under-surface a full three-event chain for generic ordering recall
  - this is the strongest remaining structural live blocker after `20.17`
- answer-level packet use / synthesis split
  - `gift_context` and `coffee_order` packets are materially better than their final answers
  - the next thread should distinguish:
    - structurally insufficient packet
    - versus structurally sufficient packet but weak answer synthesis
- graph/corpus live enablement remains separate
  - `semantic` and `graph` are still degraded in deployed Bestie
  - graph/corpus-backed runtime enablement must stay visible as its own later step, not be silently assumed from the phase 20.11-20.15 source work

## Next Thread Decision

Do **not** resume blind aggregate broadening from the old `20.16` planning direction.

The next honest thread should instead target:

- same-principal temporal/session-summary surfacing for low-overlap ordering recall on the active degraded live stack
- plus an explicit packet-sufficiency vs answer-synthesis split for the remaining fact-style live failures
