Recover real retrieval quality on the fixed canary using the `20.10` harness stack, beginning with a pre-change patched-runtime baseline and then fixing the highest-signal named retrieval classes.

## Workstream A: Baseline Instrument Reading

- pass runtime-sync verification
- run patched-runtime `--retrieval-only --canary`

- no retrieval-code commit should land before this artifact exists

## Workstream D: Bounded Proof

Required order:
1. post-change retrieval-harness canary
2. post-change canary answer-only
3. larger proof only if justified by those results

This workstream must not:
- default back to full `15`-question live proof as inner loop
- present a larger proof run as mandatory if canary evidence is still poor

## Quality Standard

No component is “good enough”.

Each must be stated as one of:
- correct for the current phase contract
- explicitly pending with named uncertainty
- wrong and fixed now

## Minimum Evidence Required

The phase must leave behind:
- a pre-change canary retrieval baseline
- a post-change canary retrieval artifact
- a post-change canary answer-only artifact
- an explicit statement of dominant remaining blocker if the canary still fails to improve materially
