# Phase 67 Plan

## purpose

Add a typed semantic retrieval substrate over durable Brainstack evidence so paraphrased recall can work from real stored memory instead of keyword overlap, transcript stuffing, or remote hot-path classification.

## core question

How can Brainstack retrieve semantically relevant durable records across paraphrases while keeping authoritative rows as source of truth and avoiding keyword farms, language-specific routing, or a second truth owner?

## GSD planning enforcement check

This phase must preserve:

- no heuristic sprawl: semantic recall must not be implemented as cue lists or language phrase tables
- Brainstack non-governor boundary: the index helps retrieval only; it does not decide execution or approval
- source-of-truth discipline: durable typed rows remain authoritative, the index is derived
- token discipline: semantic retrieval must be bounded and traceable through Phase 65 inspect output

## shared north star and real-world proof gate

North star for Phase 65-69: Brainstack must become a more reliable memory kernel in user-visible terms, not merely gain more internal machinery.

Every phase closeout must prove one of these:

- the memory kernel is measurably better for a real recall/state/policy scenario
- or the phase is pure prerequisite work and proves no regression while naming the exact later phase that must deliver the user-visible improvement

Minimum proof shape:

- targeted tests for the changed surface
- at least one end-to-end Brainstack packet or inspect proof using real store/retrieval code
- evidence ID/source/provenance proof where memory is involved, not only answer text
- no-regression check against Phase 66 golden gates once they exist
- bounded token/output sanity for any new inspect, doctor, or trace artifact

## accepted evidence

- profile and graph paraphrase recall can fail even when durable evidence exists
- current graph recall is lexical seed based and cannot carry the full semantic memory burden
- continuity/transcript fallback can answer some literal cases but is not a clean durable memory substrate
- Phase 66 will define golden cases that need durable semantic recall rather than route heuristics

## primary goals

- define one typed semantic evidence document contract for durable shelves
- index durable records with stable keys, shelf identity, owner, provenance, authority, timestamps, and supersession metadata
- keep authoritative durable rows as source of truth
- retrieve semantically related records across paraphrases without query-specific cue lists
- make index health and retrieval traces visible through Phase 65 doctor/inspect
- detect index drift when embedder, normalizer, chunker, schema, or derived-index version changes, and refuse to present stale derived results as fresh truth

## explicit scope model

### Brainstack-owned scope

- derived semantic evidence document model
- index update/delete/supersession handling
- search API over typed durable evidence
- integration into existing retrieval without mandatory remote hot-path understanding
- tests against Phase 66 golden paraphrase gaps

### out of scope

- replacing all shelf-specific storage
- making semantic index authoritative truth
- broad prompt stuffing
- adding a new runtime memory layer
- importing a donor memory framework as the core architecture

## anti-goals

- no renamed keyword routing
- no locale dictionaries or phrase farms
- no remote LLM dependency in ordinary query hot path
- no duplicate durable truth outside Brainstack rows
- no opaque ranking that cannot be inspected

## architecture decisions to freeze before execution

### 1. semantic document contract

Define a typed derived document with:

- stable `evidence_key`
- shelf and record ID
- principal scope
- authority class
- provenance class
- content excerpt or normalized representation
- timestamps and supersession/currentness metadata
- modality metadata for future non-text records

### 2. source-of-truth rule

The index stores derived search material only. Returned results must resolve back to current durable rows before being used in a packet.

### 3. update lifecycle

Every indexed shelf needs explicit behavior for:

- insert
- update
- supersede
- delete
- backfill
- degraded index unavailable state

### 3.1 index fingerprint and drift rule

Derived semantic index rows must record enough fingerprint material to detect stale index state:

- embedder identity/version/config
- text normalizer version
- chunker/sectioner version where applicable
- semantic document schema version
- source shelf schema/version marker when available

If the fingerprint does not match the active runtime, doctor/inspect must mark the index stale or degraded. Retrieval may either refuse stale derived hits or label them explicitly as stale candidates that cannot become authoritative packet evidence until refreshed.

### 4. backend choice

Prefer existing Brainstack semantic/corpus infrastructure only if it can satisfy typed provenance, bounded search, idempotency, and inspectability. If not, introduce a narrow Brainstack-owned index seam rather than forcing corpus semantics to carry unrelated memory truth.

### 5. retrieval integration

Semantic evidence retrieval must feed existing packet selection as traceable candidates. It must not bypass authority ordering, temporal validity, or packet budgets.

## workstreams

### 1. durable shelf inventory

Map profile, task, operating, graph, continuity, transcript, and corpus records into possible semantic evidence documents.

### 2. document and index API

Implement the typed evidence document model and the index write/search API with clear unavailable/degraded states.

### 3. fingerprinting, write-path hooks, and backfill

Add idempotent derived-index updates from durable writes, persist index fingerprints, and provide bounded backfill/rebuild for existing stores.

### 4. retrieval integration

Add semantic evidence candidates into inspectable retrieval without changing authority semantics or turning it into prompt stuffing.

### 5. proof against golden gaps

Run Phase 66 paraphrase cases and convert fixed semantic recall cases into gates.

## execution sequencing

1. Freeze the document contract and source-of-truth rule.
2. Implement index API and health reporting.
3. Add write-path/backfill integration shelf by shelf.
4. Integrate bounded semantic evidence search into retrieval.
5. Add/update Phase 66 golden tests.
6. Run targeted and broader regression rings.

## deliverables

- typed semantic evidence document contract
- derived index implementation or narrow index seam
- index fingerprint/drift detection
- idempotent update/backfill behavior
- retrieval integration with inspect traces
- golden eval proof for paraphrase recall improvements

## lightweight process refinements

### terminology reuse

Use the Phase 65 terminology lock for index and retrieval status. Do not introduce alternate words for the same states unless the plan explicitly justifies them.

### expected-red ownership

Phase 67 must consume the Phase 66 expected-red cases it owns and close each one as:

- fixed and promoted to hard regression gate
- still red with explicit architectural reason
- moved to a later phase only with a written dependency reason

### phase handoff gate

Closeout must state:

- which semantic/paraphrase cases moved from expected-red to gate
- which typed shelves are indexed
- which shelves are deliberately not indexed yet
- whether Phase 68 is blocked by any index lifecycle issue

## acceptance criteria

- paraphrased durable-memory queries can retrieve the correct evidence ID where Phase 66 requires it
- stale semantic index state is detected and cannot silently masquerade as current memory
- semantic index results resolve back to current authoritative rows
- index unavailable/degraded state is visible through doctor/inspect
- no keyword farm or locale-specific route logic is introduced
- no mandatory remote LLM call is added to the ordinary hot path
