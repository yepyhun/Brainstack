# phase 29.1 context

## title

canonical communication contract and durable identity capture hardening

## problem

Phase 29 repaired the reset-time seam that was dropping communication rules at the scoped retrieval / active-contract boundary.

That fix restored recall after reset, but the post-fix live state still shows two weaker structural gaps:

- the communication contract is durable, but not stored in one clean canonical shape
- explicit age recall still appears to come from transcript fallback instead of a durable identity row

The problem is no longer "it forgot everything".
The problem is that the durable personal-memory shape is still uneven:

- some communication truth lives in canonical slot rows
- some communication truth still lives in older summary-style or duplicated preference rows
- explicit age is visible in transcript evidence, but not in `profile_items`

## north star

For one principal, Brainstack should hold one coherent personal-memory shape:

- explicit core identity facts land as durable identity rows
- communication rules land in a bounded canonical contract shape
- transcript recall stays a fallback and evidence source, not the main owner of either of those truths
- retrieval can rebuild the active contract from canonical durable truth without depending on free-text accidents

## why this matters

If the contract shape stays fragmented, the system remains harder to audit, migrate, and prove correct.

If age only survives in transcripts, the system is still depending on continuity fallback for a basic durable identity fact.

Both gaps matter because they weaken the "single owner, durable truth, bounded retrieval" story even if the live UX looks mostly better.

## settled constraints

- preserve Brainstack as the single owner of personal memory on this axis
- preserve donor-first updateability
- do not solve this with persona files, skill files, or prompt stuffing
- do not turn this into a runtime style-policing phase
- do not bring back broad handwritten Tier-1 profile inference into the live ingest path

## anti-goals

- do not hardcode the full `29`-rule Humanizer list into a brittle always-on guardrail system
- do not create a second owner for communication truth by storing one free-text mega-contract blob and separate atomic rows without clear ownership
- do not treat transcript fallback as "good enough" for explicit durable identity facts
- do not patch retrieval first if the real gap is still in extraction / canonicalization
- do not expand this into unrelated donor or host-runtime work

## investigated findings

### live state findings

- the live deployed database contains `15` active identity/preference rows for the principal
- the communication lane is partly canonical:
  - `preference:ai_name`
  - `preference:message_structure`
  - `preference:formatting_style`
- but it is also fragmented by older variants and summaries:
  - `preference:style:humanizer`
  - `preference:writing_style`
  - `preference:language_preference`
  - `preference:preference:communication_style`
  - `preference:preference:style`
  - `preference:preference:formatting`
- there is no live `identity:age` row
- transcript evidence does contain repeated `19 éves` / `19 years old` statements

### source-code findings

- live ingest no longer uses handwritten Tier-1 profile inference:
  - `brainstack/extraction_pipeline.py` explicitly sets `profile_candidates=[]`
- the old Tier-1 extractor still has a deterministic age rule:
  - `brainstack/tier1_extractor.py` can emit `identity:age`
- the active Tier-2 extractor already supports:
  - stable slot aliases for `identity:age`
  - bounded transcript fallback for several communication slots
- the active Tier-2 extractor does **not** have equivalent deterministic transcript fallback for age
- retrieval already knows how to render the canonical communication slot family into the active contract:
  - language
  - Bestie name
  - Humanizer style
  - emoji usage
  - newline rule
  - pronoun capitalization
  - dash rule

## current seam reading

This follow-up does not currently read as a retrieval failure.

The stronger reading is:

- communication gap:
  - canonicalization / compatibility gap
- age gap:
  - durable extraction gap

## candidate minimum repair surface

- `brainstack/tier2_extractor.py`
  - bounded deterministic hardening for explicit age capture
  - bounded canonical communication-slot shaping
- `brainstack/db.py`
  - if needed, bounded migration/backfill helpers for older fragmented communication rows
- `brainstack/retrieval.py`
  - consume the canonical slot family cleanly and compatibly, without depending on noisy legacy variants
- tests
  - real-world flow
  - phase 21 communication-contract tests
  - phase 24 scoped-profile correctness tests

## required outcome

This phase is complete only when the project can show:

- one bounded canonical communication-contract storage shape
- one durable `identity:age` path for explicit user self-statements
- one compatibility story for older rows that does not create a second owner
- one explicit proof that transcript fallback is now secondary on this axis, not primary

## final verdict shape

The closeout for this phase should end in one explicit verdict:

- what became canonical durable truth
- what still remains intentionally summarized or projected
- what migration/backfill was required
- what was ruled out as unnecessary overengineering
