# phase 29.1 plan

## goal

Harden Brainstack so the communication contract lives in one bounded canonical durable shape and explicit age facts land as durable identity truth, without turning the fix into a brittle runtime guardrail system.

## scope

### in

- canonical communication-contract storage shape
- explicit age durable capture
- bounded compatibility/backfill for fragmented legacy communication rows
- retrieval / active-contract projection proof for the canonical shape
- live-state verification against the current deployed DB and runtime

### out

- style-fidelity hardening through prompt guardrails
- full `29`-rule behavior policing
- persona / skill / note file fallbacks
- broad rollback to live Tier-1 handwritten inference
- donor refresh work

## validation method

- one live-state audit of current communication rows and identity rows
- one source-path audit for extraction, canonicalization, and retrieval
- one explicit proof of why age is not durable today
- one bounded repair design that names:
  - canonical owner
  - compatibility path
  - no-touch zones
- one post-fix validation ladder that would prove:
  - canonical communication rows exist
  - `identity:age` exists
  - reset-time contract assembly still stays clean and bounded

## decision rules

- if a gap can be explained by durable data shape alone, do not add runtime guardrails
- if a communication rule is already covered by the canonical slot family, do not create a second row family for the same meaning
- if a durable fact is explicit and low-ambiguity, transcript fallback alone is not acceptable
- if a proposed canonical shape creates dual ownership between atomic rows and a mega-summary blob, reject it
- if the smallest correct fix is extractor plus compatibility migration, stop there

## north star

One principal, one durable personal-memory shape:

- atomic identity truth for explicit core facts
- bounded canonical communication truth
- transcript evidence as support, not primary ownership

## core principles this phase must preserve

1. donor-first
   - prefer the settled Brainstack shape and donor-aligned seams over inventing a new local memory pattern
2. modular, thin integration
   - keep the repair on the smallest correct Brainstack seam
   - do not smear the fix across host/runtime/prompt layers unless evidence forces it
3. no heuristic sprawl
   - do not build a brittle rule engine for style obedience
   - only bounded deterministic extraction/canonicalization is acceptable
4. single durable owner
   - Brainstack remains the only durable owner of communication and identity truth on this axis
   - no side file, prompt artifact, or parallel summary blob may co-own the same truth
5. native-system synergy, not replacement
   - preserve the existing native/runtime responsibilities
   - do not solve a Brainstack data-shape gap by replacing native behavior elsewhere
6. durable only where durable value is real
   - durable storage is for reusable truth
   - transcript fallback and runtime shaping remain support layers, not the primary memory owner

## anti-goals

- no brittle Humanizer police layer
- no giant `29`-rule free-text contract blob as the new truth owner
- no semantic dependence on one exact wording of the rules
- no host/runtime prompt hacks standing in for missing durable truth
- no broad schema explosion for every stylistic nuance

## hypothesis -> falsification -> repair decision

1. communication canonicalization seam
   - hypothesis:
     - the live communication gap is mainly a fragmented durable shape, not missing data
   - falsification:
     - prove the live DB already contains only the canonical slot family with no meaningful duplicate variants
   - repair decision:
     - if not falsified, add bounded canonicalization plus compatibility/backfill

2. age durable-write seam
   - hypothesis:
     - explicit age is absent because the active extractor path has no deterministic age fallback after Phase 17 removed live Tier-1 profile inference
   - falsification:
     - prove that explicit age already lands durably on the live Tier-2 path and the gap is later in reconciliation or retrieval
   - repair decision:
     - if not falsified, harden extraction at the bounded explicit-identity seam

3. migration seam
   - hypothesis:
     - older fragmented communication rows can be migrated or compatibly projected without creating a second owner
   - falsification:
     - prove that the older rows are either irrelevant noise or already transparently consumed by the current canonical path
   - repair decision:
     - if not falsified, land a bounded migration/backfill plan

4. retrieval projection seam
   - hypothesis:
     - retrieval already has enough logic to render a clean contract if canonical slot rows exist
   - falsification:
     - prove that retrieval would still need noisy legacy summaries even after canonicalization
   - repair decision:
     - if not falsified, keep retrieval changes minimal and compatibility-focused

## protected boundaries

### anti-guardrail boundary

- do not repair missing durable truth by adding output policing

### ownership boundary

- Brainstack remains the only owner of personal communication truth

### schema boundary

- do not explode the schema to one field per microscopic stylistic preference unless evidence forces it

### compatibility boundary

- old fragmented rows may be migrated or compatibly projected, but they must not remain a permanent parallel authoritative lane forever

## stop conditions before any patch

- stop if the design introduces dual ownership for communication truth
- stop if the only age fix requires broad handwritten language-specific inference again
- stop if the proposed canonical contract depends on one exact English phrasing of the rules
- stop if the phase starts solving LLM obedience instead of durable truth shape

## regression net

- no loss of current reset-time communication recall
- no loss of scoped principal isolation
- no reintroduction of persona / skill / side-file memory ownership
- no growth into a brittle runtime style-enforcement subsystem
- no donor-host drift outside the bounded Brainstack seam

## compatibility / donor-risk notes

- Phase 17's "no handwritten live Tier-1 profile inference" rule should stay intact
- a bounded deterministic explicit-identity fallback inside Tier-2 is acceptable if it does not become a broad semantic rule engine
- the communication contract should stay donor-friendly:
  - atomic durable rows
  - compact projection
  - no giant local-only prompt artifact

## validation ladder

1. prove current live DB fragmentation and missing `identity:age`
2. prove the exact extraction gap for age
3. define the canonical communication slot family
4. define bounded migration/backfill for old fragmented rows
5. prove retrieval / prompt block can rely on the canonical family
6. execute only the smallest repair surface

## required working style

- correctness-first
- no overfit to one noisy live transcript
- no giant schema redesign without evidence
- no "just summarize it harder" shortcuts
- preserve the core principles above exactly, not approximately
- if the best answer is a smaller canonical family plus one explicit identity fix, stop there

## recommended model level

- `high`
- reason:
  - this phase is architectural hardening of durable truth shape, not a trivial edit
  - it still has a bounded fix surface and does not need the full live-forensics depth of Phase 29

## execute verdict

- closed

### what was proven

- the bounded communication slot family is now sufficient for the active communication contract builder
- explicit self-declared age now has a durable Brainstack-owned capture path
- the historical gap was closed with a one-shot versioned compatibility migration, not an always-on sweep
- the deployed `hermes-bestie` runtime now carries both migration markers and a live `identity:age` row for the tested principal

### what was ruled out

- prompt-strengthening as the primary fix
- a broad handwritten Tier-1 rollback
- a giant Humanizer blob as a second owner
- any Graphiti-side direct transplant for this seam

### final repair shape

- shared contract helper for slot normalization and bounded communication expansion
- deterministic Tier-2 explicit-age fallback using the same helper family
- versioned compatibility migrations:
  - `canonical_communication_rows_v1`
  - `explicit_identity_backfill_v1`
