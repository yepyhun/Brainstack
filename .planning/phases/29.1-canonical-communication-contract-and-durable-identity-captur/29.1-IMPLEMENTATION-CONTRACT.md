# phase 29.1 implementation contract

## objective

Turn the remaining post-Phase-29 memory weakness into one bounded data-shape repair:

- canonical communication-contract storage
- durable explicit age capture

without drifting into prompt guardrails, schema sprawl, or dual ownership.

## system doctrine this phase must preserve

- Brainstack remains the single owner of personal memory on this axis
- atomic durable truth outranks transcript fallback for explicit core facts
- transcript remains evidence and fallback, not the primary owner when durable truth should exist
- donor-first updateability must stay intact
- the fix must stay smaller than the problem

## core principles this phase must preserve exactly

1. donor-first
   - reuse the settled Brainstack shape and donor-aligned seams before adding any local mechanism
2. modular, thin integration
   - keep the fix inside the smallest correct Brainstack-owned seam
   - do not spread the repair into host/runtime prompting unless the evidence forces it
3. no heuristic sprawl
   - no fragile style-policing engine
   - only bounded deterministic extraction or canonicalization is allowed
4. single durable owner
   - Brainstack is the only durable owner for this truth
   - prompt artifacts, side files, or summary blobs may not become parallel owners
5. native-system synergy, not replacement
   - preserve existing native/runtime responsibilities
   - do not compensate for missing durable truth by replacing adjacent native behavior
6. durable only where durable value is real
   - explicit core facts and reusable communication truth deserve durability
   - transcript evidence remains support, not the final home

## non-negotiables

- no persona-file fallback
- no skill-file fallback
- no runtime style-enforcement layer as the primary fix
- no giant free-text contract blob as a second owner of communication truth
- no rollback to broad handwritten Tier-1 semantic/profile inference

## workstream a: live truth inventory

- inventory the current principal-scoped communication rows
- classify canonical rows versus fragmented legacy rows
- prove that `identity:age` is absent while transcript evidence is present

## workstream b: canonical communication contract shape

- define the bounded canonical slot family
- decide whether a compact pattern payload is needed at all
- make atomic slots the authoritative lane

## workstream c: explicit age durable capture

- prove the exact extractor seam that loses age today
- design a deterministic, low-ambiguity capture path for explicit self-declared age
- keep the logic bounded to explicit identity facts

## workstream d: compatibility and migration

- define how older fragmented communication rows are migrated, backfilled, or compatibly projected
- ensure the end state does not preserve a permanent dual-authority model

## workstream e: retrieval and proof

- prove the active contract builder can rely on the canonical family
- define the regression net and live validation path

## acceptable outputs

- one canonical communication-contract storage shape
- one durable `identity:age` path
- one bounded compatibility/backfill story
- one explicit no-go on anything that would add maintenance cost without durable value

## unacceptable outputs

- "transcript fallback is enough"
- "just make the prompt stricter"
- "store the whole 29-rule list as one giant text blob and hope"
- "add a runtime checker for every style slip"
- any fix that makes Brainstack plus a prompt artifact co-own the same personal truth

## protected boundaries

### anti-band-aid boundary

- do not use output obedience machinery to hide missing durable truth

### ownership boundary

- the authoritative communication memory lane must stay inside Brainstack durable data

### schema boundary

- the canonical shape must be bounded and explainable
- if a compact pattern payload is used, it must support the atomic slot family rather than replace it ambiguously

### compatibility boundary

- older fragmented rows may be supported during migration
- they must not remain an unbounded parallel memory lane

## minimum evidence required before calling phase 29.1 done

- one live inventory proving the current fragmented communication shape
- one live inventory proving absence or presence of `identity:age`
- one source-path proof of where age is currently lost
- one named canonical communication slot family
- one named durable age capture seam
- one explicit statement of what was intentionally not changed

## current best repair shape to prove or reject

- canonical owner:
  - bounded communication slot family:
    - `preference:communication_style`
    - `preference:response_language`
    - `preference:ai_name`
    - `preference:emoji_usage`
    - `preference:message_structure`
    - `preference:pronoun_capitalization`
    - `preference:dash_usage`
    - optional compact pattern payload only if needed for the broader Humanizer set
- explicit identity hardening:
  - deterministic explicit age capture inside the Tier-2 extraction seam or an equivalent bounded Brainstack-owned seam
- compatibility:
  - bounded migration/backfill from older fragmented communication rows into the canonical family
- projection:
  - retrieval and active-contract rendering should prefer canonical rows and only use legacy compatibility paths as temporary support

## recommended model level

- `high`
- this phase needs enough reasoning depth to keep the fix bounded, architecture-aligned, and migration-safe, but it is still narrower than the Phase 29 live forensic split

## execute closeout

### accepted implementation shape

- `brainstack/profile_contract.py`
  - shared slot normalization
  - bounded communication expansion
  - deterministic explicit-age derivation from explicit self-statements only
- `brainstack/tier2_extractor.py`
  - reuses the shared helper for communication and age shaping
- `brainstack/db.py`
  - versioned one-shot compatibility migrations instead of open-ended sweep logic
- `brainstack/retrieval.py`
  - projects the bounded communication contract family cleanly

### rejected alternatives

- runtime style obedience guardrails
- a full Humanizer rules engine
- broad Tier-1 profile inference revival
- graph-side or donor-side ownership expansion for a profile-store problem

### deployed proof required and achieved

- rebuilt runtime image
- force-recreated running container
- verified migration code exists in the running container
- verified migration markers exist in the live DB
- verified live durable `identity:age`
- verified active prompt block still reconstructs the communication contract correctly
