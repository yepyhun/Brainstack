# Phase 20.9 Implementation Contract

## Purpose

Recover live post-routing retrieval quality after `20.8` by fixing fact-path parity, route-policy misrouting, structural-route yield/quality, and the named profile / identity miss class with comparable proof against the `20.8` live truth.

## Required Workstreams

### Workstream A. Fact-Path Parity Restoration

- This workstream is first and must stay first.
- It must compare the current live fact path against:
  - the `20.8` comparable live fact behavior
  - the older stronger decomp-off reference behavior
- It must name where the live fact path differs:
  - packet shape
  - evidence ordering
  - rendering / packing
  - budget or runtime assumptions
- It must not be collapsed into “probably the route-policy issue.”

### Workstream B. Route-Policy Decision And Misroute Audit

- This workstream must explicitly re-audit deterministic classifier decisions before reading temporal yield as route-quality truth.
- It must name:
  - false-positive temporal routes
  - false-negative temporal routes
- It must separate three questions:
  - is the selected mode right
  - does the chosen route yield candidates
  - are the yielded candidates useful
- If the deterministic gate hurts more cases than it helps, bounded multi-channel fusion / soft routing must remain the explicit fallback policy alternative.
- Any such fallback must stay bounded and donor-first-compatible; it must not turn into unbounded run-everything retrieval.

### Workstream C. Activated Structural Route Yield, Candidate Quality, And Packing

- This workstream begins only after route-policy misroutes are separated from yield truth.
- It must separately measure and, if needed, repair:
  - temporal-channel yield
  - aggregate/graph yield
  - candidate relevance
  - evidence packing quality
- It must not flatten these into a single “route quality” label.
- Aggregate counting remains an opportunity-cost warning only:
  - it may stay weak under the current architecture
  - but it must not dominate the phase mission or masquerade as the only meaningful success criterion

### Workstream D. Profile / Identity Retrieval Recovery

- The `5d3d2817`-style miss class must remain named explicitly.
- This workstream must treat profile / identity retrieval as its own class, not as generic fact noise.
- It must separate:
  - candidate discovery
  - ranking
  - packing / final context handoff

### Workstream E. Comparable Live Reruns

- The phase must end with comparable answer-only and split reruns.
- The comparison anchor is `20.8`, not only older pre-routing phases.
- The summary must state clearly whether improvements came from:
  - fact-path parity
  - route-policy correction
  - structural-route yield
  - candidate quality / packing
  - profile / identity recovery

## Protected Boundaries

### Scope Boundary

- This phase must not retry `20.8` route-hint reliability work from scratch.
- This phase must not reopen decomposition, overlay tuning, or prompt-wording games.

### Heuristic Boundary

- This phase must not grow a broad multilingual keyword/alias routing table.
- Deterministic route-policy changes must stay explainable and bounded.

### Route Policy Boundary

- Bounded multi-channel fusion / soft routing is allowed only as an explicit fallback policy decision with evidence.
- Unbounded all-channels-parallel dumping is disallowed.

### Aggregate Boundary

- Aggregate-counting difficulty must be treated as opportunity-cost risk, not as a hard numerical ceiling and not as the dominant target function.

### Proof Boundary

- This phase must not launder weak results into “partial success.”
- If the post-fix reruns remain at the `20.8` floor, the summary must state restoration insufficiency plainly.

## Minimum Proof Required Before Calling Phase 20.9 Done

- A fact-path parity diff exists and names the dominant parity loss.
- A deterministic classifier misroute table exists.
- The route-policy decision is explicit:
  - deterministic gate retained
  - or bounded multi-channel fallback selected
- Activated temporal / aggregate routes are measured with yield, candidate quality, and packing separated.
- The profile / identity miss class is reported separately from generic retrieval failures.
- Comparable answer-only and split reruns against `20.8` are recorded.
- If the answer-only score remains `<= 2 / 15` and split `retrieval_correct` remains `<= 3 / 15`, the phase summary must explicitly declare that live restoration remained insufficient.
