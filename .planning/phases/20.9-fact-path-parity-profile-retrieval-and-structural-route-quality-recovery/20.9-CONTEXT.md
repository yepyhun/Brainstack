# Phase 20.9: Fact-Path Parity, Profile Retrieval, And Structural Route Quality Recovery - Context

**Gathered:** 2026-04-14
**Status:** Ready for planning

## Why This Phase Exists

`20.8` solved the narrow route-hint reliability gate:

- the old LLM route-hint path was retired
- the deterministic route-hint path became the default
- structural routing activated live again

But the live quality did not recover:

- answer-only: `2 / 15`
- split raw answer: `5 / 15`
- split `retrieval_correct`: `3 / 15`
- split `both_correct`: `3 / 15`

`20.8` also proved that activation is not enough:

- activated `temporal` routes: `6`
  - temporal-channel nonzero: `2`
  - temporal-channel zero: `4`
- activated `aggregate` routes: `2`
  - graph-channel nonzero: `1`
  - graph-channel zero: `1`

So the next blocker is not route-hint reliability anymore. The next blocker is downstream retrieval quality:

- weak live fact-path parity versus the older stronger decomp-off path
- named profile / identity misses
- deterministic classifier misroutes
- partial temporal / aggregate channel yield
- weak candidate quality and evidence packing even when structural routes activate

## What This Phase Is

This phase is a **live restoration phase** for the post-routing stack.

It exists to answer these bounded questions:

- why the live fact path is still far below the older stronger baseline
- which deterministic classifier decisions are wrong enough to distort route-quality reading
- whether the active temporal / aggregate paths fail mainly at:
  - route policy
  - channel yield
  - candidate quality
  - evidence packing
- how the named profile / identity miss class should be recovered without collapsing it into generic retrieval noise

## What This Phase Is Not

This phase is **not**:

- a retry of `20.8` route-hint reliability work
- a return to LLM route-hint prompt tweaking
- a decomposition comeback phase
- a `20.2` overlay / wording / bonus tuning phase
- a multilingual keyword-table routing phase
- a benchmark-score laundering phase
- an aggregate-counting optimization phase as the main target

If the deterministic gate harms more cases than it helps, that must be decided explicitly here. But this phase must not hide behind new route-hint speculation once the route-policy evidence is already on the table.

## Entry Conditions

- `20.8` is execute-complete.
- The deterministic route-hint path is live in the comparable benchmark path.
- Route activation is restored:
  - `deterministic_route_hint`: `14 / 15`
  - `default`: `1 / 15`
  - requested/applied modes:
    - `fact`: `7`
    - `temporal`: `6`
    - `aggregate`: `2`
- The current measured blocker is downstream retrieval quality, not dead routing.
- The known carry-forward guards are already recorded:
  - deterministic classifier misroutes must be audited explicitly
  - bounded multi-channel fusion / soft routing remains an allowed fallback if the deterministic gate hurts more than it helps
  - aggregate counting remains an opportunity-cost warning, not a hard score ceiling

## Target End State

- The phase names the dominant downstream blocker layer explicitly:
  - fact-path parity
  - route-policy misrouting
  - temporal / aggregate channel yield
  - candidate quality
  - evidence packing
  - profile / identity retrieval
- The deterministic gate is either retained with evidence or displaced by a bounded multi-channel fallback with evidence.
- Live fact-path parity is materially recovered or explicitly explained as still insufficient.
- The named profile / identity miss class is either improved or narrowed to a clearly documented residual blocker.
- A comparable answer-only plus split rerun exists against the `20.8` truth, not only against older pre-routing phases.
