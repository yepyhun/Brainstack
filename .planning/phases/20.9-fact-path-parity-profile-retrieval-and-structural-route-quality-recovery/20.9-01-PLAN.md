# Phase 20.9 Plan

## Goal

Recover live post-routing retrieval quality by restoring fact-path strength, auditing deterministic route-policy errors, recovering structural-route yield/quality, and fixing the named profile / identity miss class without reopening route-hint reliability work from scratch.

## Scope

### In

- fact-path parity restoration against the strongest known decomp-off behavior
- deterministic classifier misroute audit
- explicit route-policy decision:
  - retain bounded deterministic gate
  - or move to bounded multi-channel fusion / soft routing
- temporal / aggregate channel-yield recovery on already activated routes
- candidate quality / evidence packing recovery on activated structural routes
- named profile / identity retrieval recovery
- comparable answer-only plus split reruns against `20.8`

### Out

- LLM route-hint prompt-contract retry as the main solution
- new decomposition work
- `20.2` overlay tuning
- broad proof/judge recalibration
- aggregate-counting maximization as the main score target

## Planning Assumption

`20.8` already proved that route activation can be restored without restoring answer quality.

`20.9` should therefore treat downstream retrieval quality as the main problem, while still allowing route-policy replacement if the deterministic gate misroutes enough cases to poison the structural-route measurements.

## Execution Steps

1. Freeze the `20.8` baseline.
   - Use the live comparable `20.8` answer-only and split reports as the only comparison anchor.
   - Preserve the per-question route, channel-yield, and failure-layer view before changing anything.

2. Workstream A: restore fact-path parity first.
   - Compare the current live fact-path packet shape, rendering, budgets, and candidate ordering against the older stronger decomp-off path.
   - Identify where the live fact path lost useful evidence or ranking strength.
   - Apply only general fact-path fixes, not benchmark-specific exceptions.

3. Workstream B: audit route policy before reading route quality.
   - Re-audit deterministic classifier decisions question by question.
   - Name both:
     - false-positive temporal routes
     - false-negative temporal routes
   - Do not interpret temporal-channel yield as route-quality truth until the misrouted cases are separated.
   - If the deterministic gate hurts more cases than it helps, compare it against a bounded multi-channel fusion / soft-routing fallback.

4. Workstream C: recover structural-route yield and downstream retrieval quality.
   - For activated `temporal` and `aggregate` routes, measure separately:
     - channel yield
     - candidate relevance
     - evidence packing quality
   - Fix the dominant failing layer instead of treating “route quality” as one blob.
   - Keep aggregate counting under opportunity-cost discipline; do not let it become the dominant target function.

5. Workstream D: recover the named profile / identity miss class.
   - Audit the `5d3d2817`-style misses as their own retrieval class.
   - Separate:
     - candidate discovery
     - ranking
     - packing / context handoff
   - Fix only the general class, not one-off benchmark wording.

6. End with comparable live reruns.
   - Run answer-only and split using the same comparable benchmark path as `20.8`.
   - Compare directly against `20.8`, not against stale pre-routing runs only.

## Implementation Rules

- Do not reopen route-hint reliability from scratch.
- Do not reintroduce LLM route-hint prompt tweaking as the default answer.
- Do not let route-policy work collapse into keyword-table benchmaxing.
- Do not let structural-route work hide the dominant layer; name whether the blocker is:
  - selection
  - yield
  - candidate quality
  - packing
- Do not let aggregate-counting difficulty dominate the phase mission.
- Do not let the summary claim restoration success if the comparable reruns stay at or below the weak `20.8` floor without a clear explanation.

## Output Shape Required From Phase 20.9

- a fact-path parity diff against the `20.8` live fact path and the older stronger reference behavior
- a deterministic classifier misroute table with named false positives and false negatives
- an explicit route-policy decision:
  - bounded deterministic gate kept
  - or bounded multi-channel fusion / soft routing chosen
- a structural-route table that separates:
  - route activation
  - channel yield
  - candidate quality
  - evidence packing
- a named profile / identity retrieval report
- comparable answer-only and split rerun results against `20.8`

## Exit Gate

- The dominant downstream blocker layer is named explicitly rather than hidden under generic retrieval language.
- Fact-path parity is materially improved, or the phase explicitly says why restoration remained insufficient.
- Deterministic classifier misroutes are either reduced or ruled against by an explicit route-policy decision.
- Activated temporal / aggregate routes are evaluated with selection, yield, candidate quality, and packing separated.
- Profile / identity retrieval remains named and either improves or is narrowed to a clearly documented residual blocker.
- If the comparable answer-only score remains `<= 2 / 15` and split `retrieval_correct` remains `<= 3 / 15`, the phase must explicitly declare restoration insufficient rather than implying partial success.
