# Phase 29.8 Plan

## goal

Expand the behavior-policy ontology and compiler so nuanced communication rules can survive compilation without turning the always-on lane into prompt bloat.

## scope

### in

- ontology design for behavior policy dimensions
- compiler behavior from raw contract to compiled policy
- bounded prompt projection rules

### out

- explicit teaching path mechanics
- second-brain routines/contacts/place modeling

## core principles this phase must preserve

1. donor-first
2. modular, thin integration
3. zero heuristic sprawl
4. multimodal-ready
5. token discipline
6. single durable owner per truth class
7. second-brain / proactive-agent fit
8. no dead code / no half-wired paths

## architecture decision

### accepted target model

- richer first-class policy dimensions
- compiled output remains bounded and ordinary-turn safe
- compiler output is coverage-aware and no-silent-drop

### rejected model

- flattening all nuance into 4-6 slogans
- compensating with raw long-form prompt injection

## workstreams

### 1. ontology audit

- list which user behavior dimensions deserve first-class compiled representation

### 2. compiler design

- define how archival rules map into policy slots / structures
- define a bounded named-outcome model for each taught rule
  - active
  - conditional
  - recall-only
  - conflict
  - unsupported
  - ambiguous

### 3. bounded projection

- define which compiled policy pieces are always-on
- define projection caps and priority rules
- keep raw contract out of ordinary-turn prompts

### 4. revision discipline

- make later corrections recompute cleanly
- preserve language-neutral semantics so Hungarian long-form rule packs remain first-class inputs

## validation ladder

1. ontology sufficiency review
2. compiler coverage / no-silent-drop proof
3. bounded projection proof
4. correction/recompile proof

## anti-goals

- no ontology explosion
- no token-unbounded policy block

## recommended model level

`xhigh`
