# Phase 29.8 Implementation Contract

## phase type

Schema and compiler expansion.

## required final architecture

- a richer compiled behavior ontology than the current tiny compact slot family
- stable archival-to-compiled transformation rules
- bounded always-on projection rules

## minimum evidence required before calling the phase done

- important rule classes no longer collapse silently into generic slogans
- compiled policy remains bounded
- later corrections trigger deterministic recompute

## explicitly disallowed outcomes

- open-ended heuristic rule mining
- unbounded prompt growth
- ontology drift without tests

## cleanup requirement

- remove superseded compact-only shortcuts that the richer compiler replaces

## compatibility requirement

- earlier 29.x durable storage and sync-teaching work must still function

## code hygiene requirement

- no duplicate compilers
- no parallel incompatible policy DTOs
## recommended model level

`xhigh`
