# Phase 20.13 Context

## Why This Phase Exists

- `20.12` closed the question of whether a bounded temporal semantic rerank layer should exist.
- A bounded temporal semantic rerank layer now exists in code.
- An env-gated external scorer path now exists in code.
- Jina v5 TEI earned preferred external evaluation status through local evidence.
- The targeted `gpt4_7f6b06db` proofs showed that stronger scoring alone does not fix the residual temporal miss.
- The fixed canary with the Jina-gated scorer showed no obvious fact/profile/aggregate collateral regression.

## Current Truth Entering 20.13

- `20.12` is execution-complete at gate.
- The current semantic layer is no longer hypothetical:
  - bounded hybrid temporal-event reranking exists
  - stronger external scoring can be evaluated without a full backend swap
- The residual blocker is now upstream of the scorer:
  - the desired trip chain is still not reliably present in the bounded temporal-event pool that the reranker receives
- This means the main problem is now better read as:
  - temporal-event candidate availability
  - temporal-event generation reliability
  - temporal-event pool assembly visibility
- This is not a reason to reopen:
  - route-hint work
  - benchmark-specific lexical tuning
  - "maybe a reranker still does not exist" debate

## What This Phase Is

- a follow-up capability phase focused on upstream temporal-event candidate availability / generation reliability
- a source-of-truth phase for where the desired event chain drops out:
  - generation
  - persistence
  - bounded pool assembly
  - or pre-rerank filtering
- a bounded implementation phase for one generic upstream fix if the source-of-truth reading is clear enough
- a continuation of donor-first memory-kernel work, not a LongMemEval solver phase

## What This Phase Is Not

- not another `20.12` scorer-tuning loop
- not a full corpus-wide embedder migration
- not a vendor-lock decision
- not a return to route-hint, lexical cues, or query-shaping
- not aggregate-counting redesign
- not a benchmark-row-specific trip heuristic phase

## Entry Conditions

- runtime-sync verification already exists and must continue to gate live proof
- the fast canary / retrieval-only workflow from `20.10` remains the inner loop
- the `20.12` bounded temporal rerank layer and env-gated external scorer hook must be preserved unless a defect forces a targeted correction
- Jina v5 TEI remains a preferred external evaluation candidate, but it is not the focus of this phase

## Target End State

- the project can say, with evidence, where the desired temporal-event chain drops out before reranking
- at least one bounded upstream fix exists if the failure point is clear enough
- the bounded temporal-event pool seen by the reranker is materially more reliable for the `gpt4_7f6b06db` class
- the phase leaves behind reusable upstream visibility and reliability improvements for structured temporal continuity, not a benchmark-specific patch
