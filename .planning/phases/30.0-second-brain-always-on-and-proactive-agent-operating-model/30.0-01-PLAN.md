# Phase 30.0 Plan

## goal

Define and implement the operating model that makes Brainstack a true always-on second-brain and proactive-agent kernel without breaking ownership, token discipline, or donor-first modularity.

## scope

### in

- always-on personal operating context
- proactive continuity operating rules
- durable-vs-transient separation for real second-brain data

### out

- large offline document graphing
- replacing Chron or other native owners

## core principles this phase must preserve

1. donor-first
2. modular, thin integration
3. zero heuristic sprawl
4. multimodal-ready
5. token discipline
6. single durable owner per truth class
7. second-brain / proactive-agent fit
8. no dead code / no half-wired paths

## architecture decision

### accepted target model

- broaden the raw-vs-compiled pattern beyond behavior style
- keep proactive behavior bounded, explicit, and owner-disciplined

### rejected model

- turning Brainstack into a shadow scheduler
- turning every transient note into durable operating memory

## workstreams

### 1. operating memory classes

- define stable vs transient classes for second-brain use
- define an explicit operating-context matrix:
  - behavior policy
  - stable profile
  - project / active work context
  - open decisions
  - commitments / reminders as external-owner pointers
  - transient session state
- implement the first slice as a derived projection from existing owners, not as a new durable store
- exclude communication-policy duplication from this matrix; compiled behavior policy remains the only ordinary-turn behavior owner

### 2. proactive activation model

- define when Brainstack may drive proactive continuity
- define trigger, urgency, user-value, suppression, and owner boundaries
- keep the first live rule conservative: resume active work when the user re-engages vaguely and the current session still carries concrete work/decision context

### 3. boundary preservation

- preserve Chron and other native owners

### 4. product proof

- prove the model improves real continuity, not just internal elegance

## validation ladder

1. memory class matrix
2. proactive activation contract
3. owner-boundary proof
4. product UAT

## anti-goals

- no silent owner takeover
- no proactive noise engine

## recommended model level

`xhigh`
