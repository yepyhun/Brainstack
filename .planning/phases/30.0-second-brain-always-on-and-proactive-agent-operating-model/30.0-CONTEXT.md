# Phase 30.0 Context

## problem statement

The user's product goal is larger than communication-style obedience.

They want Brainstack to become an always-on second-brain kernel that can support proactive, stateful agent behavior without fragmenting ownership.

## why this phase exists

Once behavior policy is trustworthy, the same architectural lessons need to be applied to broader personal operating memory:

- routines
- contacts
- places
- recurring tasks
- proactive continuity

## explicit non-goals

- no second scheduler beside Chron
- no shadow CRM or todo database hidden in Brainstack
- no undisciplined proactive spam
## recommended model level

`xhigh`
