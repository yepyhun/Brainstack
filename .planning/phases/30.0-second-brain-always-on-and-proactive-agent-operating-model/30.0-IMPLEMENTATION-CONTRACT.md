# Phase 30.0 Implementation Contract

## phase type

Product architecture expansion.

## required final architecture

- a clear always-on second-brain operating model
- explicit proactive-agent boundaries
- no shadow ownership over reminders, scheduling, or other native systems
- the first operating-context slice is a derived projection from existing owners, not a new durable truth class
- compiled behavior policy remains the only ordinary-turn behavior owner

## minimum evidence required before calling the phase done

- stable memory classes are defined and enforced
- proactive behavior is bounded and user-beneficial
- owner boundaries stay explicit

## explicitly disallowed outcomes

- proactive behavior that is only impressive on paper
- shadow reminders or duplicate task truth

## cleanup requirement

- remove any exploratory proactive paths that do not survive the final contract

## compatibility requirement

- must preserve earlier behavior-policy architecture

## code hygiene requirement

- no hidden sidecar owner
- no dead proactive flows
## recommended model level

`xhigh`
