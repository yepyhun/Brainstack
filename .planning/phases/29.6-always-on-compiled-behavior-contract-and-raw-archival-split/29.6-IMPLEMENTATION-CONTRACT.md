# Phase 29.6 Implementation Contract

## phase type

Architecture phase with bounded implementation.

## required final architecture

- one archival behavior-contract owner for exact, literal durable policy truth
- one compiled always-on behavior-policy owner for ordinary-turn execution
- no third owner in files, skills, corpus, or transcript shadow paths

## minimum evidence required before calling the phase done

- the raw-vs-compiled split is explicit in code and storage ownership
- ordinary turns consume the compiled policy, not the raw archival block
- exact recall still comes from the archival owner
- the compiled lane is token-bounded

## explicitly disallowed outcomes

- raw contract and compiled policy collapsing back into one ambiguous lane
- output-policing marketed as policy architecture
- half-wired compiled lane that exists only in local probes

## cleanup requirement

- remove any superseded path that still tries to make one lane serve both purposes

## compatibility requirement

- current detailed style recall must remain possible
- compact contract behavior must not silently disappear during the transition

## code hygiene requirement

- no dead DTOs
- no duplicate compilers
- no hidden fallback owner
## recommended model level

`xhigh`
