# Phase 29.6 Plan

## goal

Introduce a Brainstack-owned dual-layer behavior-policy model:

- raw archival behavior contract
- compiled always-on behavior policy

## scope

### in

- behavior-policy data model
- ownership split between raw and compiled layers
- ordinary-turn policy injection path
- explicit recall path for the raw archival contract

### out

- synchronous teaching path details
- broader second-brain operating model
- KG/document pipeline redesign

## north star

After this phase, the system should no longer rely on a single lane to serve both:

- exact recall of the taught contract
- everyday obedience in normal replies

## core principles this phase must preserve

1. donor-first
   - reuse Brainstack/Hermes seams instead of inventing a side subsystem
2. modular, thin integration
   - add one explicit behavior-policy layer, not diffuse hacks across prompt code
3. zero heuristic sprawl
   - no regex rule engine as the primary policy mechanism
4. multimodal-ready
   - the policy carrier must not assume text-only forever
5. token discipline
   - raw contract stays on-demand; compiled policy stays bounded and ordinary-turn friendly
6. single durable owner per truth class
   - one archival owner, one compiled owner, no duplicate authorities
7. second-brain / proactive-agent fit
   - this must improve always-on continuity rather than only solving one recall demo
8. no dead code / no half-wired paths
   - no dormant dual-write or dual-read leftovers

## architecture decision

### accepted target model

- keep one raw archival contract as the exact durable source
- derive one compiled policy object for ordinary-turn behavior
- keep both principal-scoped and Brainstack-owned
- make ordinary-turn obedience depend on compiled policy injection, not on the query being routed to `style_contract`
- keep `style_contract` as an exact-recall lane for the raw contract, not as the primary obey-path
- once compiled policy is active, legacy compact communication runtime paths lose authority and become fallback/migration material only

### rejected model

- shoving the raw long-form contract into every normal prompt
- keeping obedience dependent on explicit style-contract recall
- solving behavior with post-hoc output policing

## workstreams

### 1. owner split

- define archival owner semantics
- define compiled owner semantics
- define update and supersession contract between them
- define the cutover rule that prevents dual runtime authority

### 2. runtime injection seam

- identify where compiled policy belongs in ordinary-turn prompt construction
- ensure it is always-on but bounded
- prove ordinary prompts obey without requiring a behavior-recall phrasing from the user

### 3. recall separation

- preserve exact recall from the archival lane
- ensure the compiled lane is not mistaken for the archival source

### 4. bridge validation

- prove the new split does not regress compact ordinary turns
- prove it does not create a second shadow owner
- prove principal isolation for compiled policy activation

## validation ladder

1. schema/ownership proof
2. ordinary-turn injection proof
3. exact archival recall proof
4. no-dual-owner proof
5. principal-isolation proof

## anti-goals

- no heuristic patch pile
- no prompt bloat
- no file-based side memory

## recommended model level

`xhigh`
