# Phase 10.1 Context

## Goal
Make Brainstack repeatably installable into a fresh upstream Hermes checkout, so regular Hermes updates do not require manual copying or uncertain runtime wiring.

## Why This Phase Exists
- The user will keep updating upstream Hermes.
- Brainstack currently exists in the development Hermes checkout, but the fresh Docker/Discord Bestie runtime did not contain `plugins/memory/brainstack`.
- That creates a real operational failure mode:
  - the launcher can correctly start Hermes Gateway
  - Discord can appear to work
  - but Brainstack may not actually be installed or active
- Manual copying is not acceptable because it will recur after every upstream update and invites half-wired states.

## Architecture Decision
Use a **Brainstack Integration Kit** rather than making Brainstack API-first or maintaining a long-lived Hermes fork.

### Chosen Direction
- Brainstack remains a Hermes-native memory provider for the primary runtime path.
- Brainstack's store/core should stay API-ready, but API-first is deferred.
- The Brainstack repository becomes the install/update source of truth.
- A target Hermes checkout can be refreshed from upstream, then Brainstack is re-applied by script.

### Rejected For Now
- Long-lived Hermes fork as the primary strategy.
- Full API-first memory server as the first production path.
- Manual runtime patching after each upstream update.
- Blind patch scripts that modify host files without compatibility checks.

## Required Install/Update Workflow
Target operator flow:

```bash
git clone https://github.com/NousResearch/hermes-agent hermes-agent-latest
git clone https://github.com/yepyhun/Brainstack Brainstack

python Brainstack/scripts/install_into_hermes.py hermes-agent-latest --enable --doctor
```

Regular update flow:

```bash
python Brainstack/scripts/update_hermes_with_brainstack.py hermes-agent-latest --pull --reinstall --doctor --docker-rebuild
```

The desktop launcher can later wrap this flow, but the script must be usable by a human or another agent first.

## Required Behavior
- Copy Brainstack plugin payload into `plugins/memory/brainstack/`.
- Copy required helper files into explicit target paths only.
- Patch or validate config so Hermes builtin memory and user profile are off.
- Enable Brainstack as the intended memory provider using the current Hermes plugin convention.
- Validate Docker/Discord gateway launch config separately from terminal chat mode.
- Emit a machine-readable install manifest showing what changed.
- Fail closed when the target Hermes checkout lacks required provider/lifecycle surfaces.

## Compatibility Policy
- Safe copying is allowed.
- Config patching is allowed when the config file shape is recognized.
- Host code patching is allowed only if:
  - the target anchors match exactly
  - the patch is small and named
  - the doctor command proves the resulting live path
- If anchors or lifecycle hooks moved upstream, the installer must stop and report the incompatibility instead of guessing.

## Anti-Half-Wire Requirements
The doctor command must detect at least these cases:
- Brainstack files copied but plugin config not enabled.
- Brainstack enabled but Hermes builtin memory/user profile still active.
- Docker launcher starts an interactive terminal chat instead of `gateway run --replace`.
- Docker compose mounts the wrong Hermes home/config directory.
- Brainstack plugin exists in one checkout but the desktop launcher points at another checkout.
- Required lifecycle/provider interfaces are absent or no longer importable.

## Non-Goals
- Do not build a full standalone Brainstack API server in this phase.
- Do not solve upstream donor auto-merge.
- Do not make three peer memory runtimes.
- Do not migrate old Bestie memories or unrelated Tomi tool folders.
- Do not include secrets in generated manifests, logs, docs, or README output.

## Output Expectation
This phase should leave Brainstack with a practical integration kit:
- simple enough for a human or agent to run
- strict enough to catch bad installs
- compatible with regular upstream Hermes refreshes
- honest about when manual host adaptation is required
