# Phase 20.1 Implementation Contract

## Purpose

This document turns the Phase `20.1` corrective diagnosis into a concrete execution contract.

The goal is to prevent two failure modes:

- a shallow benchmark polish pass that leaves the real fidelity gap open
- an overgrown rewrite that reopens the architecture instead of repairing the evidence path

## Required Workstreams

### Workstream A. Conversational semantic publication

Required output:

- continuity summaries and/or transcript turns published into the semantic backend
- stable metadata attached:
  - session id
  - turn number
  - date or timestamp
  - source / kind
- semantic retrieval returning conversational evidence through the existing executive retrieval contract

Hard rule:

- this may not remain corpus-only semantics
- if semantically related conversation turns still cannot surface, the phase fails

### Workstream B. Temporal rendering restoration

Required output:

- retrieved conversational evidence rendered with explicit date/session labels where available
- temporal questions receiving visibly chronological evidence
- no hidden dependence on implicit chronology only the store knows

Hard rule:

- temporal improvement must be visible in the prompt evidence itself
- not merely in internal metadata

### Workstream C. Graph quality gate

Required output:

- a truth-ingress quality gate that blocks junk state rows from entering rendered graph truth
- explicit suppression of the known legacy junk family
- proof that the graph backend can stay healthy without truth-context pollution

Hard rule:

- this may not be solved by hiding rendered graph rows after they already polluted truth state
- the bad rows must be prevented or disqualified before they can act like truth

### Workstream D. Transcript widening

Required output:

- transcript availability widened beyond the current one-snippet bottleneck
- raw turn text surviving often enough to carry exact numbers, names, and thresholds
- bounded transcript inclusion that still respects token discipline

Hard rule:

- transcript access must improve detail recall materially
- not merely increase transcript count on paper

### Workstream E. Adaptive evidence packing

Required output:

- packing logic that prefers fewer, better, longer snippets when retrieval confidence is strong
- better survival of answer-bearing suffixes and exact values
- reduced pointless clipping of correct evidence

Hard rule:

- this may not degrade into “dump more text”
- the new packing must improve information density, not just output length

### Workstream F. Evidence-priority prompt contract

Required output:

- a narrow instruction clarifying that:
  - specific, non-conflicted recalled facts outrank generic model priors
- explicit avoidance of blind memory supremacy

Hard rule:

- no “always trust memory”
- no benchmark-only wording that would be embarrassing as a general runtime rule

### Workstream G. Corrective eval ladder

Required output:

- Gate A:
  - fast publication / retrieval / rendering contract tests
- Gate B:
  - targeted scenario tests for:
    - lexical mismatch
    - temporal ordering
    - exact detail
    - graph junk suppression
    - evidence-priority behavior
- Gate C:
  - a bounded benchmark-derived subset focused on the actual failure families
- Gate D:
  - explicit rerun recommendation for Phase `20`

Hard rule:

- the ladder must attribute gains by lever
- not just say “score went up somehow”

## Protected Boundaries

### Architecture boundary

- `20.1` may not reopen the donor-first architecture
- `20.1` may not demote `Kuzu` or `Chroma`
- `20.1` may not turn the shell back into a replacement brain

### Heuristic boundary

- no multilingual regex growth as the main fix
- no new handwritten semantic alias tables as the primary solution
- any narrow gating logic must remain bounded and obviously subordinate to donor-shaped retrieval

### Prompt boundary

- prompt fixes must stay fidelity-oriented
- prompt fixes may not become benchmark gaming
- prompt fixes may not override conflict handling or provenance discipline

### Runtime boundary

- keep non-blocking provider behavior
- do not turn synchronous turn handling into a heavy publication job
- preserve cross-store consistency guarantees already restored in earlier phases

## Minimum Proof Required Before Calling Phase 20.1 Done

All of the following must be true:

1. conversational turns are semantically retrievable through the donor-aligned path
2. temporal evidence is rendered clearly enough for ordering-heavy questions
3. graph junk no longer appears as truth in the rendered graph context
4. raw transcript detail is materially more available than before
5. exact-detail carry-through is materially stronger than under the old clipping path
6. the prompt contract expresses correct evidence priority without blind memory bias
7. bounded benchmark-derived evidence shows a real improvement over the flat Phase `20` path
8. the phase leaves a clean recommendation on whether to rerun Phase `20`

If any one of these is missing, the phase is not done.
