# Phase 20.1 Plan

## Goal

Close the benchmark-exposed memory-fidelity gap left after the donor-first recovery track by improving conversational semantic retrieval, temporal readability, graph-truth quality, exact-detail carry-through, and evidence-priority behavior without regressing the restored `L1` / `L2` / `L3` architecture.

## Scope

### In

- semantic indexing of continuity and/or transcript history into the existing semantic backend
- date-rich rendering of retrieved conversational evidence
- graph ingress quality gating before junk can surface as truth context
- wider transcript evidence access where raw detail matters
- adaptive evidence packing that preserves answer-bearing detail
- narrow prompt-contract clarification for evidence priority
- bounded eval ladder focused on the benchmark-exposed failure families

### Out

- new donor selection
- shell redesign
- benchmark-specific hacks
- handwritten multilingual heuristic growth
- full milestone closeout
- another broad architecture proof phase

## Planning Assumption

The best corrective route is **not** a new architecture layer.
It is a fidelity repair on top of the now-correct donor-first stack.

That means:

- Phase `20.1` should change the quality of evidence retrieval and rendering
- not the core layer split
- not the cross-store ownership model
- not the donor alignment already restored in `17` / `18` / `19`

## Execution Steps

1. Restore semantic retrieval on real conversation history.
   - Decide the publish shape for continuity summaries and/or transcript turns into `Chroma`.
   - Keep session, turn, and date metadata attached.
   - Make the semantic leg return conversational evidence, not only corpus documents.
   - Prove that semantically related but lexically different user history can now surface.

2. Make temporal evidence explicitly readable in the prompt.
   - Surface session/date labels on retrieved conversational evidence.
   - Ensure temporal-order questions receive evidence that is legible to the model without requiring hidden inference about chronology.
   - Preserve existing provenance/cross-store behavior.

3. Add a graph-truth quality gate.
   - Stop legacy graph ingress from turning arbitrary user phrases into truth rows.
   - Gate or narrow the old regex path instead of adding more ad hoc rules.
   - Prove that junk `role=...` style rows no longer surface in `Graph Truth`.

4. Widen transcript evidence for exact-detail access.
   - Raise transcript availability above the current narrow default.
   - Ensure raw answer-bearing turns can compete with lossy summaries.
   - Keep the result bounded; do not flood the prompt with transcript spam.

5. Replace shallow clipping with adaptive evidence packing.
   - Shift from flat trimming toward “fewer but better and longer” evidence snippets when retrieval quality is higher.
   - Preserve numbers, names, thresholds, exact titles, and concrete detail-bearing suffixes.
   - Prove that exact-detail carry-through improves on detail-heavy retrieval cases.

6. Clarify evidence-priority behavior in the prompt contract.
   - Add a narrow rule that specific, non-conflicted recalled facts outrank generic prior knowledge.
   - Explicitly avoid a blind “always trust memory” instruction.
   - Prove that this changes model behavior only in the intended direction.

7. Build the bounded Phase 20.1 eval ladder.
   - Gate A:
     - fast retrieval/rendering contract tests
     - conversation semantic publication tests
     - graph junk suppression tests
   - Gate B:
     - targeted conversational scenarios covering:
       - lexical mismatch
       - temporal ordering
       - exact detail
       - graph contamination resistance
       - memory-vs-prior-knowledge behavior
   - Gate C:
     - small benchmark-derived subset focused on the failure families exposed in Phase `20`
   - Gate D:
     - rerun recommendation threshold for a fresh Phase `20` verdict

8. Keep attribution clean for rerouting.
   - The phase output must let us say whether improvement came from:
     - conversational semantic retrieval
     - temporal rendering
     - graph cleanup
     - detail preservation
     - prompt evidence-priority behavior
   - If the phase still fails, the corrective routing must be obvious instead of fuzzy.

## Implementation Rules

- donor-first still means donor-first
- `20.1` may improve memory fidelity; it may not quietly replace donor intelligence with new local hacks
- conversation-history semantics are first-class, not optional polish
- graph cleanup must happen at ingress / truth qualification, not by simply hiding rows later
- transcript widening and evidence packing must work together; neither may become raw context spam
- prompt behavior changes must remain narrow, conditional, and evidence-based
- benchmark-derived cases may steer the ladder, but they may not become the definition of success

## Output Shape Required From Phase 20.1

The implementation must leave behind:

1. source changes that repair the recorded fidelity gaps
2. a bounded eval ladder for the six corrective levers
3. explicit evidence that conversation history is now semantically retrievable
4. explicit evidence that graph junk no longer pollutes truth context
5. explicit evidence that exact-detail carry-through improved
6. a written rerun recommendation for whether Phase `20` should be executed again

## Exit Gate

All of the following must be true:

1. conversational history participates in semantic retrieval through the donor-aligned path
2. temporal ordering questions receive clearly date-rich evidence
3. graph truth context is materially cleaner and no longer admits the known junk pattern family
4. transcript/raw-turn detail is available often enough to support exact-detail questions
5. adaptive packing preserves more answer-bearing detail than the old shallow clipping path
6. the prompt contract now gives correct evidence priority without blind memory bias
7. bounded benchmark-derived proof is materially stronger than the flat Phase `20` result
8. the phase can say honestly whether a fresh Phase `20` rerun is now justified
