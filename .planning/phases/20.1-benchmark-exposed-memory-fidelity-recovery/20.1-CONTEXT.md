# Phase 20.1 Context

## Discuss Outcome

Phase `20.1` is now explicitly locked as a **corrective memory-fidelity phase**.

This phase is **not** allowed to:

- reopen the donor architecture debate
- redesign the shell again
- fall back to benchmark-only prompt tricks
- grow multilingual regex heuristics
- declare success because bounded real-world proof already improved

Phase `20.1` exists because the donor-first recovery track is now structurally sound, but the final benchmark path is still too weak on the exact memory-fidelity questions that matter.

## What “Corrective Fidelity Recovery” Means

This phase must improve the real benchmark path by fixing the specific places where the current recovered system still loses or distorts useful memory.

The accepted corrective levers are now:

1. conversational semantic indexing into the semantic backend
2. date-rich temporal rendering for retrieved conversational evidence
3. graph extraction quality gating against junk truth rows
4. transcript evidence widening for exact-detail access
5. adaptive evidence packing instead of shallow flat clipping
6. a narrow evidence-priority rule in the prompt contract

These are not six independent feature ideas.
They are one fidelity chain:

- retrieve the right conversation evidence
- make time legible
- suppress graph junk
- expose raw detail
- stop clipping away answer-bearing content
- tell the model how to use specific recalled facts against generic priors

## Why This Phase Exists

Phase `20` ended with:

- real-world proof improved
- restored gate reruns passing
- donor-first architecture intact
- final-boss benchmark still flat at `3 / 15`

The current diagnosis is:

- conversational history is still too lexical
- temporal evidence is still too implicit
- graph truth can still be polluted by legacy ingress
- raw exact detail is still underrepresented
- useful evidence can still be clipped away
- the prompt contract still does not clearly prioritize specific, non-conflicted recalled fact over generic model prior

So Phase `20.1` is not a phase for “more architecture”.
It is a phase for **making the recovered architecture actually hit the benchmark-exposed memory tasks**.

## Brainstack Job vs Donor Job

### Brainstack job

- publish conversational history into the right retrieval paths
- render retrieved evidence in a way the model can actually use
- gate low-quality graph ingress before it pollutes truth context
- pack evidence without clipping away the answer
- express the correct evidence-priority rule in the memory contract
- preserve cross-store consistency and donor-first boundaries

### Donor job

- `Chroma` should provide real semantic retrieval over conversation history, not only corpus docs
- `Kuzu` / Graphiti-shaped L2 should provide graph truth without junk-state contamination
- `L1` should fuse the improved signals without another conceptual redesign

## Hard Rule: No “Cheap Pass”

Phase `20.1` does not pass if:

- the benchmark gets a little luckier
- the prompt gets more forceful but the evidence stays weak
- more snippets are dumped into context without better detail preservation
- graph junk is merely hidden instead of blocked
- the semantic leg still mostly ignores conversation history

This phase passes only if memory fidelity actually improves.

## Hard Rule: Prompt Fixes Must Stay Honest

The accepted prompt-side correction is:

- specific recalled facts with clear provenance and no open conflict outrank generic prior knowledge

The forbidden prompt-side correction is:

- “always trust memory”

That distinction is mandatory.

## Eval Shape

Phase `20.1` needs bounded proof, not another giant benchmark marathon.

The phase should prove:

- conversational semantic hits now surface the right turns
- temporal ordering questions receive visibly date-rich evidence
- graph junk no longer occupies truth context
- exact-detail answers survive the evidence-packing path
- the model follows specific non-conflicted recalled fact over generic prior knowledge

Then it should rerun a bounded benchmark slice strong enough to decide whether Phase `20` can be rerun honestly.
