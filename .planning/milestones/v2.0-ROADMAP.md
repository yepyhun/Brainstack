# Milestone v2.0: Hermes Brainstack

**Status:** COMPLETE
**Completed:** 2026-04-10
**Phases:** 1-10
**Total Phases:** 13

## Overview

This milestone shipped the first full Hermes Brainstack baseline:

- one Hermes-facing composite memory provider
- stable `profile`, `continuity`, `graph_truth`, and `corpus` views
- a working-memory control plane optimized for token savings
- host-level native-memory displacement
- bounded RTK and My-Brain-Is-Full-Crew integrations
- transcript hardening and donor-boundary refresh workflow

No formal milestone audit artifact existed at archive time, so this archive records the shipped scope directly from executed phase summaries and verification artifacts.

## Key Accomplishments

1. Shipped a real Hermes-facing composite memory provider instead of a benchmark-only kernel.
2. Proved native memory displacement with contract-matrix, wiring-audit, and real-world scenario evidence.
3. Added graph-truth and corpus memory as first-class shelves rather than bolted-on heuristics.
4. Put a working-memory control plane in front of recall to improve token efficiency and provenance behavior.
5. Preserved raw transcript evidence without reintroducing a second context engine.
6. Reworked donor-inspired internals into explicit, refreshable local seams instead of hidden baked-in logic.

## Key Decisions

- Brainstack remains the only Hermes-facing external memory provider.
- Hindsight, Graphiti, and MemPalace are donor patterns/baselines behind Brainstack seams, not peer runtimes.
- Native Hermes memory and user-profile behavior stay functionally displaced on the Brainstack path.
- RTK and My-Brain-Is-Full-Crew remain bounded sidecar/shell additions, not competing memory cores.
- Full one-click donor auto-update is deferred; the shipped path is a middle-ground refreshable architecture.

## Issues Resolved

- Hidden dependency on Hermes native memory responsibilities was surfaced and covered.
- Half-wired integration risk was reduced through explicit invariants and negative-path tests.
- Transcript retention was strengthened without letting prompt noise explode.
- Donor-inspired logic is now much easier to audit than in the original baked-in state.

## Issues Deferred

- Full automatic upstream donor update system is not built yet.
- LLM-based implicit preference extraction is not part of this milestone.
- Formal milestone audit automation and requirements traceability checkboxes were not introduced before archive.

## Technical Debt

- Refresh remains review-first rather than fully automated.
- `PROJECT.md` still describes the architecture at a high level, not a full operator handbook.
- A future milestone should convert requirements tracking into a more explicit completion/traceability format before the next archive cycle.
