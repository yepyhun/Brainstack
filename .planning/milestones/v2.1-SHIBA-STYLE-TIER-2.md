# Milestone v2.1: Shiba-Style Tier-2 Inference (Definition)

This document formally captures the architectural definition of "Shiba-style Tier-2 inference" referenced in `STATE.md`, based on an explicit audit of the Shiba Memory source code (`cli/src/extraction/targeted.ts`) and architectural constraints.

## 1. What "Shiba-Style" Means for Us
Brainstack explicitly **DOES NOT** integrate the physical Shiba Memory codebase (no new PostgreSQL, pgvector, NodeJS, or external sentence-transformer dependencies). 
We already have the infrastructure (Chroma for embeddings, Kuzu for graph, and an LLM `auxiliary_client`). 
"Shiba-style" means implementing Shiba's **advanced prompting logic and mathematical scoring** entirely within our existing `brainstack/tier2_extractor.py` and `reconciler.py` architecture.

## 2. Core Intelligent Extractions (The "What")
The Tier-2 extractor must be expanded to include the deep behavioral analysis that makes Shiba powerful, specifically:

- **Implicit Preference Extraction (`extractPreferences`):**
  Instead of just listening for explicit "I prefer X", the LLM must scan a batch of conversation and look for *behavioral signals*: "What did the user accept silently vs. what did they repeatedly reject or edit?" This yields high-value profile building.
  
- **Correction Rules (`extractCorrection`):**
  When a user corrects the AI, it shouldn't just save the raw event. It must extract a generalized `rule` so the AI fundamentally adjusts its behavior for the future.

- **NLI Contradiction Checks (`checkContradiction`):**
  Any newly extracted fact must run through a targeted Natural Language Inference (NLI) LLM check against existing related facts to determine if it truly updates (supersedes) the old fact or sits alongside it.

- **ACT-R Forgetting Curve:**
  Memories must slowly decay in weight unless they are accessed frequently or recently, mimicking human memory logistics.

## 3. Strict ROI and Token-Cost Mandate (The "How")
Because we lack a "free" local LLM and rely on `auxiliary_client` (which incurs token API costs), we **CANNOT** run these intelligent extractions on every conversation turn.

- **Debounced / Scheduled:** These heavy extractions must be executed conditionally.
- **Session-End Batching:** Tasks like `extractPreferences` must strictly happen at *Session End* or designated downtime. This maximizes the contextual ROI while collapsing the token cost into a single, comprehensive batch operation rather than 50 incremental, expensive calls.
- **Targeted NLI:** `checkContradiction` must only trigger when there is high semantic overlap on core Identity/Preference slots, keeping input token windows small (~150-300 tokens) to ensure it remains a cheap boolean check.

## 4. Current Reading After Phase 23

This milestone remains valid as a **later capability uplift**, but it is **not** the immediate next priority after Phase `23`.

What Brainstack already has in partial form:

- session-end / scheduled Tier-2 execution
- durable profile and communication-contract promotion
- correction-to-rule behavior on the communication axis
- reconciliation / supersession logic in partial form
- usefulness / weighting / telemetry in partial form

What still appears meaningfully incomplete relative to this Shiba-style target:

- broader implicit preference extraction outside the current communication-contract emphasis
- more general correction -> durable rule extraction beyond the current narrow communication lane
- targeted contradiction / supersession checks on identity / preference updates
- richer memory decay / weighting sophistication

Priority note:

- these Shiba-style gaps should **not** be scheduled ahead of the current post-Phase-23 correctness follow-up:
  - principal-scoped durable profile isolation
  - proactive continuity carry-through hardening
- reason:
  - the first item is a correctness / scope-hygiene problem
  - making Tier-2 extraction more aggressive before that fix could promote more wrong or cross-principal durable profile state

Recommended sequencing:

1. fix principal-scoped durable profile isolation
2. harden proactive continuity carry-through
3. then re-evaluate a focused Shiba-style Tier-2 uplift phase
